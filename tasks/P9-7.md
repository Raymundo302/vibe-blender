## Brushes
- Cursor circle follows the mouse projected on the surface (use the existing
  GPU pick to find the hit point + element; radius adjustable with F +
  mouse-move + confirm, Blender-style, or simply [ ] keys — pick one, document).
- **Inflate**: verts within radius (world-space distance to hit point) move
  along their OWN vertex normal by strength × falloff(d/radius) per stroke
  step; smooth falloff (3t²-2t³ inverted). Ctrl inverts (deflate).
- **Grab**: on stroke start capture verts within radius + their falloffs;
  while dragging, translate captured verts by the pointer's world-space delta
  (screen-plane mapping like the G operator) × falloff.
- Stroke = pointerdown → move (apply per event with spacing) → pointerup;
  ONE undo entry per stroke (MeshEditCommand.capture around the whole stroke).
- Strength: fixed default 0.5, Shift+F or a status hint optional.
- Works on the BASE mesh (like edit mode; modifiers hidden or shown as-is —
  match edit mode's existing behavior).

## Acceptance criteria
1. Unit: falloff shape (0 at radius, max at center, monotone); inflate on a
   cube's face-center vert moves it outward along its normal; grab moves
   captured verts by delta × falloff and leaves out-of-radius verts alone.
2. e2e: subdivided plane (use Ctrl+D subdivide or a parametric sphere),
   enter the sculpt tool, one inflate stroke at a known screen point →
   center verts' Y rose, corners unchanged; Ctrl+Z restores ALL of it in one
   step; grab stroke moves verts laterally; mode/tool exits cleanly (Tab back
   to object mode, G still grabs objects).
3. All e2e suites (flock) + npm test + npx tsc --noEmit pass.

### Mode decision (as built by the previous worker — kept)
Sculpt is a **TOOL toggle inside Edit Mode**, NOT a third Scene mode. Rationale
documented in `sculptBrushes.ts`: Blender's edit mesh IS the base mesh the spec's
brushes touch, and a `'sculpt'` Scene enum would not have "stayed tiny" (mode
plumbing threads through Scene/Renderer/topbar/save-load). Instead a module-level
`sculptState` singleton (`{ tool, radius, strength }`) gates the brush — the same
pattern the codebase already uses for `proportional`, `snapState`, `xrayState`.
`Scene.ts` was left untouched. The topbar chip reads `Sculpt · inflate/grab` while
a brush is active and reverts to `Edit Mode · <elem>` / `Object Mode` otherwise, so
the toggle is honest.

### Design choices (as found)
- **Falloff**: smooth `3t²−2t³` with `t = 1 − d/radius` (the same curve
  proportional-editing uses). d=0→1, d=radius→0, d=radius/2→0.5, monotone.
- **Inflate**: each dab pushes verts within radius along their OWN area-weighted
  vertex normal by `strength × falloff`. Ctrl at press → deflate (sign flip).
  Dabs are spaced along the drag (`spacing = radius × 0.3` in world units) so one
  gesture accumulates smoothly without over-applying at a single spot.
- **Grab**: on stroke start, captures the in-radius verts + their falloffs + their
  start positions + the world hit point; each move projects the pointer onto the
  view plane through that hit point (`rayPlane`, the G-operator's screen-plane
  mapping) and translates captured verts by `delta × falloff`.
- **One undo entry per stroke**: `SculptStrokeOperator` snapshots the mesh at
  press and pushes a single `MeshEditCommand.fromSnapshots` in `confirm()`. The
  stroke owns the pointer (LMB press → moves → release confirms), mirroring the
  gizmo-drag path via the `sculptStroke` flag in `InputManager`.
- **Radius adjust**: chose the `[` / `]` keys (× 1/1.15 and × 1.15, clamped
  0.02..50) over the F+mouse+confirm option — simpler, no extra modal. Documented
  in the status hint and helpOverlay.
- **Cursor preview**: reuses the existing `renderer.editPreviewLines` hook (no new
  pass) — `buildBrushCircle` builds a line-loop in the plane perpendicular to the
  camera forward, drawn on hover and during a stroke, cleared on miss / stroke end
  / mode exit.
- **Base mesh + strength**: brushes edit the edit-object base mesh like edit mode;
  fixed default strength 0.5.

### Clean exits (verified)
`clearSculptTool()` fires on Tab out of edit mode; toggling the same brush key off
sets `tool='none'` and clears the cursor. After exit the chip reads `Object Mode`
and plain **G grabs whole objects** again.
