# UR16-3 — Render tab in Properties + transparent background + GPU pacing

Builds on UR16-1/2 (in tree).

## A. New Properties tab: **Render** (icon: a small camera-back/printer glyph
consistent with the P8-5 set; sits above the Object tab)
Rows (propRow layout from UR16-2):
- **Engine**: CPU | GPU — THE canonical control (render window + Ctrl+F12
  read/write the same viewPrefs value; their local selects stay but sync).
- **Samples**: the F12 default spp (render window's field syncs).
- **Resolution X/Y**: MOVES here from the Camera tab's "Format" section
  (scene.renderSettings — same storage). Camera tab keeps a one-line pointer
  "Resolution → Properties ▸ Render".
- **Output**: still format (PNG) — placeholder select for now — and animation
  format (WebM/MP4-when-supported/PNG-seq; syncs with Ctrl+F12's modal).
- **Transparent background** (NEW feature): checkbox on scene.renderSettings.
  When on, F12/Ctrl+F12 renders skip the world backdrop and output ALPHA:
  CPU tracer writes alpha = accumulated coverage (primary-ray miss = 0),
  GPU kernel same, Save PNG preserves alpha (premultiplied-vs-straight:
  pick straight, document), PNG-seq zip frames carry it; WebM gets background
  composited black (MediaRecorder has no alpha — document in a tooltip).
  Viewport raytraced mode ignores it (viewport always shows the world).
- **Limit GPU load** (checkbox, default off): Ray's Beelink mouse stutter —
  when on, F12/anim GPU renders use smaller per-batch spp and yield between
  batches (target ≤ 8ms GPU work per rAF; reuse/extend viewportRay's
  adaptBatch). Tooltip: "Keeps the desktop responsive on weaker GPUs; renders
  a bit slower." (The 5090 won't care; the Vega does.)

## B. Serialization
renderSettings gains { transparent?: boolean } (+ whatever Output needs) —
version-tolerant like UR5-5 did; old scenes default off.

## Acceptance criteria
- tsc clean; units green (serializer; alpha coverage math).
- e2e (extend render-engine suites + new checks): (1) Render tab present with
  all rows; engine/samples/resolution round-trip to the render window and
  camera-tab pointer exists; (2) transparent ON: F12 GPU render of a floating
  cube → corner pixel alpha 0, cube pixel alpha 255 (readback probe), Save
  PNG re-decoded keeps alpha; CPU engine same; PNG-seq frame alpha verified;
  (3) transparent OFF unchanged vs baseline; (4) Limit GPU load ON: F12 still
  completes and per-batch spp is capped (probe the exposed batch stat);
  (5) donut + gpu-parity quick scenes still green (parity suite renders with
  transparent OFF — confirm no default change).
- Screenshots: Render tab + a transparent render over a checker (alpha
  visualized), LOOKED at. Free-text report, FIRST LINE STATUS:. NEVER
  idle-wait on long runs.
