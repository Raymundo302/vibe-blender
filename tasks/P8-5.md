## Format v3 (sceneJson.ts)
- Write version 3; accept 1, 2, 3 (SUPPORTED_VERSIONS). v1/v2 objects default
  kind 'mesh', materialId null; missing materials/activeCamera → empty/null.
- Serialize per object: `kind`, `light` (full LightData) and `camera` (full
  CameraData) when present, `materialId` (null allowed). Scene-level:
  `materials` array (id/name/baseColor/metallic/roughness/emissive/
  emissiveStrength — numbers rounded through num()), `activeCameraId` (null ok).
- Loader: rebuild with scene.addLight/addCamera/add so kinds are real
  (preserve names/order; addLight/addCamera take an optional data payload).
  CRITICAL: object ids are regenerated on load, so remap the SAVED
  activeCameraId to the REBUILT camera's id (match by object list index).
  Materials: re-insert captured Material objects into scene.materials
  directly, restore the id counter so future addMaterial() doesn't collide
  (same private-counter pattern buildMesh uses for meshes), and remap NOTHING
  for materialId — material ids are stored verbatim, so keep them verbatim.
- Validation: same fail() style — every new field validated before mutation;
  unknown light type / negative power / bad color arrays → readable errors.
- Determinism: stable key order, num() rounding; round-trip
  serialize→apply→serialize must be byte-identical (test it).

## Outliner
Each row gets a kind glyph before the name: ▢ mesh, 💡 light, 🎥 camera
(plain text span, class `outliner-kind`). Follow existing row build code.

## Join guard (InputManager Ctrl+J only)
Ctrl+J currently joins selectedObjects into the active object. With lights or
cameras in the selection: silently skip non-mesh objects; if the ACTIVE object
is non-mesh, setStatus('Join needs a mesh active object') and do nothing.
Small, surgical edit — nothing else in InputManager.
