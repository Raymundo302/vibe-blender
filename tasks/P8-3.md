# P8-3 — Material properties tab

Read `tasks/P8-CONVENTIONS.md` first.

## Goal
A "Material" properties tab (🎨) managing the scene material library and the
active object's assignment. The rendered viewport (Z-cycle to 'rendered')
already draws mesh objects with scene.materialOf(obj) — this task is ONLY the
data UI.

## Files (yours alone)
- `src/ui/materialTab.ts` — fill the stub (already imported by main.ts)
- NEW unit test file for pure helpers
- NEW `e2e/p8-material.mjs` — your own suite (harness pattern from ship.mjs)
- NEW `src/ui/materialTab.css` — imported from materialTab.ts, `material-tab-` prefix

## Behavior
- Empty state unless the active object is a MESH object.
- Slot row: a select listing scene.materials by name + "(None)"; picking
  assigns obj.materialId (undoable). A "New" button creates a material via
  scene.addMaterial() AND assigns it (one undo step for both).
- Below the slot, when a material is assigned, edit ITS fields: name (text),
  baseColor (color input), metallic (0..1 slider), roughness (0..1 slider),
  emissive color + emissiveStrength (number ≥ 0). Edits affect every object
  sharing the material — that's correct (library semantics).
- Undo: one command per committed change; restores prior field value (and for
  New: removes the material again + restores prior assignment). NOTE
  scene.removeMaterial exists; re-adding on redo must keep the SAME id — if
  that's awkward with addMaterial's id counter, have your command re-insert the
  captured Material object into scene.materials directly and say so in Result.
- Live in rendered mode.

## Acceptance criteria
1. Unit: assign/new/edit commands round-trip under undo/redo (id stability
   asserted).
2. e2e (p8-material.mjs): New material on the default cube via the UI, set baseColor
   red-ish + roughness 0.2, `renderer.shadingMode = 'rendered'`, screenshot;
   assert the center pixel is clearly red-dominant via the existing screenshot
   helper (read pixels through canvas.toDataURL in-page or compare PNG sizes
   like P3-3 did — pixel read preferred; the GL context uses default
   preserveDrawingBuffer=false, so read pixels IMMEDIATELY after a
   requestAnimationFrame callback, or set a temporary
   `__app.renderer.render(...)` call before reading).
3. Undo returns the cube to grey in a follow-up screenshot/pixel read.
4. All e2e suites + npm test + npm run build pass.

## Result
Built the Material properties tab (🎨, id `material`, title "Material").

Files:
- `src/ui/materialTab.ts` — filled the stub. `MaterialTab` registers via
  `registerPropertiesTab` (mirrors modifierTab). Empty state ("Select a mesh
  object") unless the active object is a MESH. Slot row = a `(None)`-prefixed
  select over `scene.materials` + a "New" button. Below it, when a material is
  assigned, edit its library fields: name (text), baseColor (color), metallic &
  roughness (0..1 range sliders with a numeric readout), emissive (color),
  emissiveStrength (number ≥ 0, negatives clamped to 0). Edits mutate the shared
  library entry (library semantics — reaches every object using the material).
  Rebuild is signature-gated and skipped while a panel input is focused (no
  focus/caret churn during edits), same discipline as modifierTab/ObjectTab.
  Field edits use focus→input(live preview)→change(commit) so rendered mode
  updates live while dragging a slider, with one undo command per committed
  change. Also exported small `rgbToHex`/`hexToRgb` helpers.
- Undo commands (exported from materialTab.ts, kept there since the spec lists
  only these four files as mine):
  - `AssignMaterialCommand` — before/after `materialId` round-trip.
  - `NewMaterialCommand.perform(scene, obj)` — creates via `scene.addMaterial()`
    AND assigns, one undo step. **id stability:** undo calls
    `scene.removeMaterial(id)` then restores the prior assignment; redo
    **re-inserts the captured Material object directly into `scene.materials`**
    (guarded against double-insert) rather than calling `addMaterial` again, so
    the id counter can't hand out a fresh id — exactly the approach the spec
    permitted.
  - `MaterialEditCommand` — one field before/after; clones RGB triples so
    snapshots never alias the live material.
- `src/ui/materialTab.css` — `material-tab-` prefixed styles, imported from the tab.
- `src/ui/materialTab.test.ts` — 9 unit tests: color-helper round-trip, and
  assign/new/edit commands round-tripping under undo/redo with the New id
  stability + prior-assignment restore + no-double-insert asserted.
- `e2e/p8-material.mjs` — drives the UI: New material on the default cube, paint
  baseColor red-ish + roughness 0.2 through the inputs, add a bright off-axis key
  light, `shadingMode='rendered'`, then read the center pixel via
  `gl.readPixels` immediately after a forced `renderer.render(...)` (the GL
  context is `preserveDrawingBuffer=false`). Asserts the center pixel is
  red-dominant, then undo restores the default grey (pixel de-saturates), redo
  re-applies red (same material id), undo-New removes the material + clears the
  slot, and a slot-select assign round-trips through undo.
