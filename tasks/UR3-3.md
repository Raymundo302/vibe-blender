# UR3-3 — Edge Slide (GG) guide lines tangent to the rails

## Goal
During GG edge slide, draw guide lines through each sliding vert EXTENDED along its
slide rails (tangent to the edges it slides on) — Blender's vertex/edge-slide
visualization. Today the slide is invisible until verts move.

## Files
- EDIT `src/core/operator/Operator.ts` — alongside the existing optional
  `axisIndicator?()` hook (line ~60), add:
  `guideSegments?(): { a: Vec3; b: Vec3 }[] | null;` — WORLD-space line segments.
- EDIT `src/render/Renderer.ts` — a `guideSegments` field (like `axisIndicator`,
  line ~151) rendered in the same pass/style family as the axis-indicator guide
  line (~line 794): thin line, a light neutral accent color distinct from the
  X/Y/Z axis colors (suggest the theme's highlight grey ~0.8, slight alpha or
  dashed if the axis guide is dashed — MATCH the axis guide's implementation
  technique). ⚠️ Reuse the axis guide's near-plane clamp — SwiftShader DROPS
  lines with an endpoint behind the eye instead of clipping (documented lesson,
  CLAUDE.md batch 1).
- EDIT `src/input/InputManager.ts` — `syncAxisIndicator()` (~line 400) also syncs
  `renderer.guideSegments = this.activeOp?.guideSegments?.() ?? null;` and the
  operator-end path clears it (~line 422 area). Also call the sync from
  `onPointerMove` if it isn't already (rails don't change during a slide, so
  syncing once at start + on key is enough — verify and do the minimal correct
  thing).
- EDIT `src/tools/edgeSlide.ts` — implement `guideSegments()`: for each sliding
  vert, for each non-null rail (a and b), a segment from the vert's START position
  (`this.before`) extended along the rail: from `base - dir*0.5*length` to
  `base + dir*(1.5*length)` i.e. through and PAST the far vert by half a rail
  length, so the tangent line visibly extends beyond the edge. Rails/positions are
  LOCAL space — transform to world with the edit object's world matrix (parent
  aware; see how the edit cage / editOverlay gets its world matrix).
  Cache the array at `start()` (rails are fixed for the slide's lifetime).

## Out of scope
- No changes to slide math, clamping, or numeric input.
- No guides for other operators (the hook is generic; only EdgeSlide implements it).

## Acceptance criteria
- `npx tsc --noEmit` clean; `npm test` green. Unit-test the segment math (pure
  part): given a vert with rails a/b, guideSegments spans the expected extended
  endpoints; single-rail and zero-rail verts handled.
- e2e: extend `e2e/edit.mjs` OR a small new suite: enter edit mode on a cube,
  select a vert, G then G, screenshot-probe that guide-line pixels exist along
  the expected screen direction (sample a strip; compare against pre-GG frame —
  see how the locked-axis indicator was verified, and remember single-pixel
  probes are banned: count a region). Escape cancels → guides gone.
- Suites green on SwiftShader; run shading-adjacent checks with unique `E2E_PORT`.
- If the spec is ambiguous or wrong, stop and report rather than improvising.
