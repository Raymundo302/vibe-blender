# P6-5 — Proportional editing (O)

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Blender's proportional editing for edit-mode G/R/S: with **O** toggled on,
transforming a selection also moves nearby UNSELECTED verts with smooth
falloff. Mouse wheel DURING the modal adjusts the radius; a circle overlay
shows it at the pivot.

## Files
- `src/tools/editTransform.ts` — the only tool file to touch: weight map
  computed at start (weight = smooth falloff `3t²−2t³` of distance to the
  NEAREST selected vert, t = 1 − d/radius, clamped; selected verts weight 1);
  apply deltas scaled by weight. Radius lives in a module-level setting object
  exported for the overlay + persistence across invocations (default 2).
- `src/input/InputManager.ts` — O key toggles (status message), wheel while a
  proportional modal is active adjusts radius (guard: only when the active op
  is an edit transform with proportional on — add a narrow hook, do NOT break
  camera zoom otherwise).
- `src/render/Renderer.ts` — reuse `editPreviewLines` to draw the radius circle
  (48-segment polyline in the plane facing the camera at the pivot) while the
  modal runs — set/cleared by the operator like loop cut does. If loop-cut-style
  reuse conflicts, STOP and report.
- `e2e/edit.mjs` appended.

## Acceptance criteria
1. Unit (add to `src/tools/editTransform.test.ts`): falloff weights (d=0 → 1,
   d=radius → 0, d=radius/2 → 0.5, beyond → 0); a G-delta applied with
   proportional moves an unselected vert by weight×delta.
2. e2e: edit mode, select ONE corner vert, O on, G, move, confirm → the
   diagonally-opposite vert moved 0 but an adjacent vert moved a nonzero amount
   less than the selected one; Ctrl+Z restores; O off → adjacent vert stays put.
3. All four e2e suites + npm run build + npm test pass.
