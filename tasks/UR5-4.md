# UR5-4 — Select modes: Box / Circle / Lasso, W cycles

## Goal
Edit-mode box select (B, `src/tools/boxSelect.ts`) is the only area-select tool.
Add **Circle** (brush-paint select) and **Lasso** ("random squiggle" — freehand
loop), with a persistent app-level *select mode* that **W cycles**
(Box → Circle → Lasso → Box, status line announces). B keeps starting whatever
kind of area select the current mode says (i.e. B = "start area select", mode
decides the shape — simple mental model; document it in the F1 overlay).

## Design
- `selectMode: 'box' | 'circle' | 'lasso'` — module-level state next to whatever
  owns similar prefs (grep for how xrayState is kept; same pattern is fine). Not
  persisted (session-only, like xray).
- **Circle select** (`src/tools/circleSelect.ts` + tests): modal like Blender's C:
  a circle cursor of radius r follows the pointer (SVG/DOM overlay like knife's
  preview); LMB-drag paints elements inside the circle into the selection
  (Ctrl-drag or MMB-drag deselects — pick ONE deselect gesture and document);
  wheel adjusts r (default 30 CSS px, clamp 5–200); Esc/Enter/RMB ends the tool.
  Membership per element mode: vert = projected point in circle; edge = either
  endpoint in circle; face = any vert in circle (matches boxSelect's membership
  spirit — reuse `projectToScreen` and the elementIndexMaps helpers).
  Selection commits ONE undo entry on exit for the whole paint session (capture
  before-state at start, compare at end; no entry when unchanged).
- **Lasso select** (`src/tools/lassoSelect.ts` + tests): LMB-drag draws a
  freehand polyline (preview overlay like knife); on release the loop closes and
  elements whose projected test point (vert pos / edge midpoint / face centroid)
  is INSIDE the polygon (even-odd ray cast — pure, unit-testable function
  `pointInPolygon(pt, poly)`) join the selection; Shift extends, plain replaces
  (match boxSelect's modifier semantics — read its code). One undo entry.
- **W**: cycles the mode + status "Select: Circle" etc. Choose the InputManager
  branch consistent with existing single-key handling; W currently unused —
  verify with a grep and report if not.
- Box select code itself: untouched except where B's dispatch moves behind the
  mode switch.
- Toolbar (UR3): update the edit-mode Select entry — one button whose icon/label
  reflects the current mode and whose click starts it; W and the button stay in
  sync (toolbar update() already polls per frame).
- Scope: EDIT MODE only (all three element modes). Object-mode area select is a
  separate future item — do not add it.

## Occlusion rule
Box select today selects through geometry (projection-only membership — verify
by reading the code). Circle and lasso follow the SAME rule as box select does
today, whatever it is — consistency over cleverness. Note what you found in your
report.

## Files
circleSelect.ts, lassoSelect.ts (+ tests), InputManager.ts (W + B dispatch),
toolbar.ts (select entry), helpOverlay entry for W, e2e below. Nothing else.

## Acceptance criteria
- tsc clean; npm test green (pointInPolygon: convex/concave/self-touching loop
  cases; circle membership per element mode; one-undo-entry contracts).
- e2e (new `e2e/select-modes.mjs`, unique E2E_PORT): (1) W cycles the three
  modes (status text); (2) circle: paint across a cube's near verts → selected,
  wheel changes radius state; (3) lasso: draw a loop around the cube's screen
  extent → all verts selected, plain lasso elsewhere replaces selection;
  (4) each commits exactly one undo entry, Ctrl+Z restores the prior selection;
  (5) box select via B still works (regression).
- If ambiguous or wrong, stop and report.
