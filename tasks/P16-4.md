## Result
Status: BUILT — all 5 items implemented, tsc clean, 653 unit tests pass, new
`e2e/p16-quickwins.mjs` green (16/16), frozen suites (p10-themes,
p10-collections, p10-camera2, edit) + smoke + workspace all pass under flock.

1. **Theme picker toggle-close** (`src/ui/themePicker.ts`) — FIXED. Root cause:
   the picker's capture-phase `pointerdown` outside-handler fired on the 🎨
   button's own pointerdown, closing the picker; the ensuing `click` then
   re-opened it, so it could never toggle shut. Fix: the ThemePicker now holds
   the anchor and `onOutsidePointer` ignores pointerdowns inside it, letting the
   anchor's click reach `openThemePicker` with the picker still open → it
   closes. e2e drives REAL pointer events (open / toggle-shut / re-open).

2. **M → New Collection = ONE undo** (`src/core/undo/collectionCommands.ts` +
   `src/ui/collectionMenu.ts`) — DONE. Added `CreateCollectionAndMoveCommand`
   composing the existing create + move primitives (create-then-move on redo,
   move-back-then-remove on undo), preserving each primitive's invariants
   (stable collection id, per-object previous assignment). `moveToNew()` now
   pushes the single composite. Unit test + e2e both assert one Ctrl+Z returns
   the object AND removes the collection.

3. **Lock-to-view framing distance** (`src/input/InputManager.ts` camera-rig) —
   FIXED. The rig pose already reproduced the camera exactly, but the pivot
   depth came from the world-origin projection clamped to 1..50 — for a camera
   framing empty space this pins the pivot too close, so the first orbit swings
   disproportionately (the "teleport" feel). `configureRigFromCamera` now takes
   an optional `preferredDistance`; `ensureCamRig` passes the live viewport
   OrbitCamera distance, so the locked fly continues at the SAME framing depth
   the user was navigating at. Bare unit tests (no arg) keep the old
   origin-projection fallback. e2e proves first-orbit motion now SCALES with
   viewport distance (d4=1.90 vs d16=7.61 ≈ 4×) and there is no teleport at
   entry.

4. **Proportional-edit default radius** (`src/tools/editTransform.ts`) — DONE.
   Default radius 2 → 1.0 world unit (half the startup cube's extent). The
   radius already lives in module-level state that `adjustRadius()` mutates, so
   the last-used value is remembered for the session; added a `__proportional`
   debug handle for e2e. Unit test asserts the 1.0 default; e2e asserts default
   + persistence across a transform.

5. **.hdr (RGBE) parsing for World** (`src/core/scene/worldData.ts` +
   `src/ui/worldTab.ts`) — DONE. Added `parseRgbe(bytes)` (Radiance header +
   `-Y H +X W` resolution, new-format adaptive RLE, old-format RLE, and flat
   uncompressed scanlines; RGBE→linear-float via `2^(E-136)`). `decodeHdriDataUrl`
   now detects the `#?` magic in the decoded data-URL bytes and routes .hdr
   files through the parser (browser-image path unchanged for PNG/JPEG). The
   worldTab `<img>` preview is hidden for non-`data:image` URLs to avoid a
   broken-image glyph. Unit tests use a hand-built RLE fixture (base64
   round-trip through `decodeHdriDataUrl`) plus exponent-0=black and
   bad-magic-throws cases.
