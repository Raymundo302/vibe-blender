# UR16-6 — Per-texel PNG alpha in the tracers (transparent pixels render black)

## Ray's report (2026-07-12)
Image planes with PNG transparency: the transparent texels render BLACK in the
traced render instead of not rendering at all (ray should pass through).

## Root-cause first (likely, verify)
- The UR16-4 GPU image atlas packs RGBA8 but the kernel's texture sampling
  probably consumes .rgb only — no per-texel alpha cutout in the GPU path, so
  transparent texels shade as black color. Check whether the atlas resample
  (bilinear 256²) also premultiplies or zeroes RGB under alpha=0 (black
  fringes even after the cutout fix — resample should use straight alpha and
  bleed edge colors, or cutout at a threshold before filtering; document).
- CPU tracer: UR8-3 added alpha<0.5 pass-through gated on material.alphaBlend
  — check it works for these planes when loaded fresh (eager decode from
  UR16-4 supplies texImage now) and that the UR16-1 stochastic material-alpha
  path composes rather than conflicts.
- Rendered-live raster: UR8-3 blended path exists — verify these planes take
  it (alphaBlend flag set by the transparent-raster path; a plain PNG picked
  via Image ▸ Diffuse/Emit may NOT set alphaBlend even when the PNG has
  alpha — if so, detect alpha presence at decode and set it; that may be the
  actual bug for Ray's case).

## Fix (ALL renderers and ALL shading modes agree — Ray follow-up)
Not just Rendered/traced: matcap, studio, and wireframe draw image/html planes
via the Always-Textured path (UR8-3 texturedPass) — transparent texels must be
transparent THERE too (blended, showing the grid/objects behind), and in the
hidden-line/wire fills. If suspect 2 (alphaBlend flag unset for plain
PNG-with-alpha picks) is confirmed, fixing the flag at decode likely covers
these modes via the existing blended path — verify each mode explicitly.
Texture alpha multiplies the material alpha at the hit point:
`aTex(point) × alphaSocket(point)` → the existing pass-through machinery
(CPU stochastic/cutout, GPU cutout, raster blend). Transparent texels = ray
continues; NO black. Emit planes: transparent texels emit nothing AND pass
through (a cutout TV sprite works). NEE emitter sampling: a sampled point
with aTex≈0 contributes nothing (skip or zero radiance — document choice).
