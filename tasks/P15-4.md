**Spec-contradiction / engine-tiebreaker note (P14-3 lesson):** the spec asks for
"byte-identical re-serialize", but keying a transform on the fixture's
non-identity-rotation camera surfaces a **one-time euler↔quat settle** — the
sampler rebuilds the transform quat from 6-dp-rounded euler keys on load, so the
first save-after-authoring differs from every later one by <1e-6 (measured A≠B,
B==C stable). This is pre-existing engine behaviour in `sampler.ts`/`sceneJson.ts`
(both OUT of my file set), not something P15-4 introduced. Rather than improvise a
core change, the run **settles once** (apply→serialize) before writing the file,
so the shipped scene is the round-trip fixed point and reloads genuinely
byte-identical. Documented as a PUNCH-LIST item in ANIM-RUN.md alongside the other
gaps (no dope sheet / F-curve editor, no easing presets, no keyframe-range batch
render).
