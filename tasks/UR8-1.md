# UR8-1 — Text geometry engine (pure core, no integration)

## Goal
The from-scratch pipeline that turns (text, font, style params) into an
EditableMesh: canvas glyph rasterization → contour trace → simplify →
triangulate (with holes) → extrude. NEW FILES ONLY under `src/core/text/` +
tests — zero integration (UR8-2 wires the object kind). Do not touch any
existing file except adding exports if a barrel exists (it doesn't — don't).

## Pipeline modules (each pure & unit-tested unless canvas-bound)
1. `raster.ts` (canvas-bound): draw one glyph (or word) with
   `ctx.fillText` at EM_PX=192 px/em into an offscreen canvas; return the
   alpha bitmap + metrics (advance via measureText, ascent/descent from
   TextMetrics). Font availability probe: `fontAvailable(name)` via
   `document.fonts.check('16px "<name>"')`.
2. `trace.ts` (pure): marching squares over a thresholded bitmap → closed
   contours (px coords); classify outer vs hole by signed area + containment.
   Unit-test on hand-built bitmaps (disc → 1 contour; ring → outer + hole).
3. `simplify.ts` (pure): Douglas-Peucker, tolerance param (default 0.75 px).
4. `triangulate.ts` (pure): polygon-with-holes → triangles. Bridge each hole
   to its outer via a mutually-visible vertex pair (max-x hole vertex →
   raycast, standard algorithm), then ear clipping. Unit tests: square,
   concave L, square-with-hole (donut letter "o"), two holes ("8"),
   degenerate slivers don't crash.
5. `layout.ts` (pure given per-word advance widths): line breaking + alignment.
   `layoutText(words, opts)` → per-word (line, xOffset):
   - wrap OFF → one line;
   - wrap ON at `wrapWidth` (em units): greedy word wrap;
   - align 'left' | 'center' | 'right' | 'justify' (justify = distribute the
     leftover into the gaps of every line EXCEPT the last; single-word lines
     left-aligned). lineHeight param (default 1.25 em).
6. `buildTextMesh.ts`: assemble — for each glyph, contours (scaled px → em →
   world via `size`), positioned per layout; produce ONE EditableMesh:
   - front cap (triangulation), back cap (reversed winding) at z = ±thickness/2
     (Z-up world: text lies in the XZ plane facing −Y? NO — match how image
     planes orient (XY footprint facing +Z); build in local XY facing +Z,
     thickness along +Z; the object transform handles the rest);
   - side walls quads along every contour;
   - `style: 'face' | 'outline' | 'both'`:
     'face' = caps + walls; 'outline' = an OUTLINE BAND only — each contour
     offset outward by `outlineWidth` (default 0.02 em… make it a param,
     offset via per-vertex contour normal) minus the original → hollow letter
     shell (caps ring + walls), no fill; 'both' = face mesh PLUS the band.
   - **Per-face tints** (P9 faceTints — grep how Scatter writes them) mark
     face-vs-outline faces so UR8-2 can color them independently in one mesh:
     face faces tinted `faceColor`, outline faces `outlineColor` (pass colors
     in; default white/black).
   - thickness=0 → caps only (flat text), no degenerate walls.
- Determinism: same inputs → identical mesh (vertex order stable) — unit-test.
- Performance guard: trace/triangulate cached per (font, char, EM_PX) in a
  module-level Map (glyph geometry reused across the string; layout/extrude
  is the cheap per-build part). "Thickness animation re-extrudes cached
  contours" must be < ~2ms for a 30-char string — measure & report.

## Acceptance criteria
- tsc clean; npm test green — every pure module unit-tested WITHOUT canvas
  (trace/simplify/triangulate/layout take arrays); raster.ts covered by a tiny
  e2e (new `e2e/text-geom.mjs`, unique port): in-browser build "Ab o8" at two
  fonts (monospace + a probed system serif), assert: mesh non-empty, hole
  counts (o=1, 8=2, b=1) survive to triangulation (probe via exported debug
  counts), 'outline' style yields no interior fill tris (probe: ray through
  the middle of 'o' — no hit… simpler: face count face-style > outline-style),
  thickness=0 has no side walls, per-face tints present in 'both'.
- Report the per-glyph cache hit timing numbers.
- If ambiguous or wrong, stop and report. E2E_PORT=9491.
