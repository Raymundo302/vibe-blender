## Solidify
registerModifier('solidify', 'Solidify', ...). Params: thickness (number,
default 0.05, step 0.01), offset (number -1..1, default 1 — 1 = extrude
outward along normals like the tutorial uses), rimCrease (number 0..1,
default 0 — writes crease weight onto the NEW rim edges in the OUTPUT mesh so
a following creased Subsurf keeps the rim tight; this mirrors Blender's
Solidify "Edge Data > Crease" the donut tutorial uses on the icing).
Algorithm (open or closed manifold): duplicate all verts, offset originals and
duplicates along area-weighted vertex normals per `offset`/`thickness`
(offset 1 → outer shell at +thickness, inner at 0), flip the duplicated
faces' winding, stitch boundary edges with quads (the rim). Deterministic
vert/face creation order. Preserve input creases on BOTH shells' corresponding
edges; faceTints copy to both shells.

## Creased Subsurf
Extend the existing Catmull-Clark to honor `mesh.creases` per edge, using the
standard crease rule: for crease weight w on an edge, the new edge point is
lerp(smooth-rule point, sharp midpoint, w); vertex points bordered by ≥2
creased edges lerp toward the sharp (corner/crease) rule by the average
weight. Output creases: child edges of a creased edge inherit
max(0, w * 2 - 1)? NO — keep Blender-simple: child edges inherit the parent's
weight unchanged (repeated subdivision keeps the crease; document this
simplification in Result). Subdividing must also carry faceTints (children of
a tinted face inherit the tint).

## Merge by Distance
`mergeByDistance(mesh, vertIds, threshold): number` — greedily cluster the
given verts (spatial hash or O(n²) at demo scale) within `threshold`
(default 0.0001), merge each cluster via the same remapping rules as
mergeVertsAtCenter (lowest id survives at the cluster centroid, faces remap,
degenerate faces drop). Returns removed-vert count. Menu item: edit mode,
runs on the selected verts (≥2), status "Removed N vertices" — Blender's
wording. Undoable via MeshEditCommand.capture like Merge at Center.

### Solidify
Two shells built from area-weighted vertex normals (accumulated UNnormalized
Newell vectors → area weighting for free). Outer shell = originals (unflipped),
inner shell = duplicates (reversed winding, normal ≈ −outer). Offset mapping:
`outerDisp = t·(offset+1)/2`, `innerDisp = t·(offset−1)/2`, so offset 1 → outer
at +t / inner at 0 (the tutorial "extrude outward" case). One rim quad per source
boundary edge. Input creases copied to BOTH shells' corresponding edges;
faceTints copied to both shells. Deterministic creation order: outer verts,
inner verts, outer faces, inner faces, rim quads.

**rimCrease decision:** writes the weight onto ALL FOUR edges of every rim quad
(both boundary loops + the two vertical connectors), so a following creased
Subsurf keeps the whole side wall / drip edge tight. The acceptance test checks
both a loop edge (`crease(0,1)`) and a connector edge (`crease(0,4)`).

### Creased Subsurf
- Edge point (interior edge, weight w): `lerp(smoothCatmullClarkPoint, sharpMidpoint, w)`.
  Boundary edges are already the sharp midpoint, so crease is a no-op there.
- Vertex point: a vert bordered by ≥ 2 creased edges lerps from its smooth
  position toward the sharp rule by the AVERAGE incident crease weight — 2 creased
  edges → crease rule `(m1+m2+6P)/8`; ≥ 3 → corner rule `P`. A vert with 0/1
  creased edges stays fully smooth (standard: a single crease doesn't sharpen its
  own endpoints).
- **Output-crease simplification (as the spec directs):** each child edge of a
  creased edge inherits the parent weight UNCHANGED (not `max(0, 2w−1)`), so
  repeated subdivision keeps the crease. Documented here per the spec.
- faceTints: every child quad of a tinted parent face inherits the tint.
- Honest sharpness assertions used: at level 1 the creased edge's edge-point
  equals the exact linear midpoint while the uncreased one does not; w=0.5 lands
  strictly between; at level 2 full-weight creases still propagate (4 of them) and
  the surface near the creased span stays closer to the original corner than the
  uncreased cube. (Note: a crease edge is sharp ACROSS but smooth ALONG its
  length, so an interior crease-edge midpoint legitimately drifts along the edge
  under further subdivision when its endpoints aren't corners.)

### Merge by Distance
`mergeByDistance(mesh, vertIds, threshold=0.0001)` greedily clusters the given
verts (O(n²), seeds visited in ascending id so the lowest id survives and the op
is deterministic), collapsing each ≥2 cluster through the existing
`mergeVertsAtCenter`. Returns removed-vert count. Menu item (edit mode, ≥2 verts)
runs it, status reads Blender's "Removed N vertices", undoable via
`MeshEditCommand.capture`.
