## Solidify
registerModifier('solidify', 'Solidify', ...). Params: thickness (number,
default 0.05, step 0.01), offset (number -1..1, default 1 — 1 = extrude
outward along normals like the tutorial uses), rimCrease (number 0..1,
default 0 — writes crease weight onto the NEW rim edges in the OUTPUT mesh so
a following creased Subsurf keeps the rim tight; this mirrors Blender's
Solidify "Edge Data > Crease" the donut tutorial uses on the icing).
Algorithm (open or closed manifold): duplicate all verts, offset originals and
duplicates along area-weighted vertex normals per `offset`/`thickness`
(offset 1 → outer shell at +thickness, inner at 0), flip the duplicated
faces' winding, stitch boundary edges with quads (the rim). Deterministic
vert/face creation order. Preserve input creases on BOTH shells' corresponding
edges; faceTints copy to both shells.

## Creased Subsurf
Extend the existing Catmull-Clark to honor `mesh.creases` per edge, using the
standard crease rule: for crease weight w on an edge, the new edge point is
lerp(smooth-rule point, sharp midpoint, w); vertex points bordered by ≥2
creased edges lerp toward the sharp (corner/crease) rule by the average
weight. Output creases: child edges of a creased edge inherit
max(0, w * 2 - 1)? NO — keep Blender-simple: child edges inherit the parent's
weight unchanged (repeated subdivision keeps the crease; document this
simplification in Result). Subdividing must also carry faceTints (children of
a tinted face inherit the tint).

## Merge by Distance
`mergeByDistance(mesh, vertIds, threshold): number` — greedily cluster the
given verts (spatial hash or O(n²) at demo scale) within `threshold`
(default 0.0001), merge each cluster via the same remapping rules as
mergeVertsAtCenter (lowest id survives at the cluster centroid, faces remap,
degenerate faces drop). Returns removed-vert count. Menu item: edit mode,
runs on the selected verts (≥2), status "Removed N vertices" — Blender's
wording. Undoable via MeshEditCommand.capture like Merge at Center.

## Acceptance criteria
1. Unit (solidify): a single quad → 8 verts, 2+4 faces (two shells + 4 rim
   quads), outer shell displaced +thickness along the normal; winding of the
   inner shell is flipped (its faceNormal ≈ -outer normal); rimCrease 1 →
   rim edges have crease weight 1 in the output; deterministic double-apply.
2. Unit (subsurf): cube with one edge creased 1.0 stays sharp along that edge
   after 1 and 2 levels (the two faces meeting there stay planar near it — or a
   cheaper honest assertion: the creased edge's midpoint equals the linear
   midpoint of its endpoints, unlike an uncreased edge's smoothed midpoint);
   crease 0.5 lands strictly between the w=0 and w=1 positions; tint carries
   to child faces.
3. Unit (merge): two coincident vert clusters merge, count reported, faces
   remap; verts farther than threshold survive.
4. e2e: add a plane, solidify via the Modifiers tab UI (thickness visible:
   evaluated mesh has more verts than base via __app), undo removes it;
   merge-by-distance menu path works on a mesh with doubled verts.
5. All e2e suites (each under flock) + npm test + npx tsc --noEmit pass.
