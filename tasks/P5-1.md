# P5-1 — Edit-mode delete UX + dissolve edges

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Deleting mesh data must be discoverable and complete: the Delete key works like
X (users kept pressing it and concluding delete didn't exist), and the delete
menu gains **Dissolve Edges** — remove edges while MERGING their faces, instead
of leaving holes.

## Files
- NEW `src/core/mesh/ops/dissolve.ts` + NEW `src/core/mesh/ops/dissolve.test.ts`
- `src/ui/deleteMenu.ts` — add the Dissolve Edges entry
- `src/input/InputManager.ts` — Delete key = X (both the object-mode delete
  and the edit-mode menu)
- `e2e/edit.mjs` — append checks

## Dissolve algorithm
`dissolveEdges(mesh, edgeKeys: Set<string>): void`
For each edge with exactly 2 distinct faces (skip boundary/wire edges):
merge the two faces into one n-gon — walk face A's vert loop from the edge's
far side, then face B's, skipping the shared edge verts' duplicate entries;
preserve A's winding. Handle chains: dissolving several edges between the same
face pair collapses iteratively (re-query faces after each merge). Drop
degenerate results (<3 distinct verts). Verts are kept (Blender keeps them;
they sit inside the merged n-gon rendered via fan triangulation).

## Acceptance criteria
1. Unit: cube, dissolve one edge → 8 verts, 5 faces (two quads became one
   6-gon... actually the merged face has 6 corners of which 2 are the shared
   verts — assert 5 faces and the merged face has 6 verts); all remaining
   edges manifold; dissolving a boundary-only selection is a no-op; undo via
   MeshEditCommand restores.
2. e2e: Delete key in edit mode opens the same menu as X; Delete key in object
   mode deletes the object (undoable); dissolve entry merges faces (6 → 5)
   and Ctrl+Z restores.
3. All four e2e suites + npm run build + npm test pass.

## Result
Built `dissolveEdges(mesh, edgeKeys)` in `src/core/mesh/ops/dissolve.ts`: for each
edge with exactly 2 distinct faces it merges the pair into one n-gon (A's full
loop from the edge's far vert back, then B's far-side corners spliced in,
preserving A's winding), re-querying faces each iteration so chains collapse;
boundary/wire/stale edges are skipped and degenerate (<3 distinct vert) results
dropped. Verts are always kept. Added a `Dissolve Edges` entry to the edit-mode
Delete menu (`dissolveEdgeSelection`, enabled in edge mode with edges selected,
undoable via `MeshEditCommand.capture`), and aliased the **Delete** key to X in
`InputManager` (one normalization line) so it drives both the object-mode object
delete and the edit-mode Delete menu.

Verification: `npm run build` clean; `npm test` 166 unit tests pass incl. 6 new
dissolve tests (cube edge → 8 verts/5 faces/one 6-gon, all-manifold, boundary
no-op, stale-key safe, chain 6→4, undo/redo). All four e2e suites pass
sequentially — `edit.mjs` now asserts Delete opens the same menu as X, the
Dissolve entry merges 6→5 with a 6-gon and Ctrl+Z restores, and Delete deletes
the object in object mode (undoable). No deviations from spec.
