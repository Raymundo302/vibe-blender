## The run (mirror the tutorial's beats, adapted to our substitutions)
Drive ONLY through public entry points (__app scene/io APIs, key events,
UI clicks) — the point is proving a user could do this. Screenshot after
every major stage to /tmp/donut/NN-<stage>.png.
1. Torus (Shift+A → Torus), adjust via the op panel: major 1, minor ~0.55,
   48×18 segments (staying in our unit scale; document the mapping to the
   tutorial's 0.1m). Shade smooth.
2. Proportional-edit a few verts for lumpiness (O, small random pulls) —
   our stand-in for the lattice.
3. Duplicate → icing: delete bottom half (x-ray box select), Solidify
   (thickness, offset 1, rimCrease 1), Subsurf level 2, Shrinkwrap → donut,
   modifier order shrinkwrap→solidify→subsurf, apply all three.
4. Icing dribbles: select rim verts, extrude down with proportional editing;
   sculpt-lite inflate on 2-3 droplet ends.
5. Plate: circle → extrude/scale rim per part 4 (simplified is fine), under
   the donut. Table plane below.
6. Materials via the Material tab: icing pink (roughness ~0.25, SSS 0.4),
   donut brown (SSS 0.6/0.1 radius), plate ceramic white (roughness 0.15),
   table grey. Colors from the tutorial.
7. Sprinkles: model one capsule (cylinder + subsurf + loop cuts), Scatter on
   the icing: count ~120, upOnly, minDistance, colorVariation 1, seed chosen
   for looks (try 3 seeds, keep the best, note it).
8. Camera: frame the donut (lock-to-view snap), focal ~50; Sun (warm,
   angled) + point "sky" fill (blue tint, radius for soft shadows) + white
   bounce point, per part 8's three-lamp setup.
9. F12 → path trace ≥64 samples (give it time), save the PNG to /tmp/donut/,
   plus a control render at 4 samples for the progress story. Set DoF focus
   on the icing with mild aperture.
10. Save the scene (__app.io.serialize) to research/donut.vibe.json (commit-
    able artifact); reload it and assert byte-identical re-serialization.

### Workarounds & deviations (all detailed in DONUT-RUN.md)
- **Sample cap (spec ≥64 → headless 16):** the tracer runs ~25 s/sample for this
  scene (960×540, subsurf-2 icing ≈16.7k faces, 3 soft-shadow lights) under
  headless swiftshader → 64 spp ≈30 min, infeasible for a suite. The progressive
  tracer yields a valid image at any count and criterion 2 is met far below 64,
  so the hero caps at 16 spp (clear step up from the 4-spp control) and DoF at 8.
  Full 64+ spp beauty render is an offline F12 pass; the saved scene is
  full-quality.
- **Geometry driven via mesh/scene API where headless gestures are imprecise:**
  bulk lumpiness (seeded wobble; the O+G proportional *tool* is exercised for
  real once), bottom-half deletion (deleteFaces by centroid vs x-ray box-select),
  icing drips, plate/table. Each of these gestures is separately proven by a
  sibling suite (p9-select, p9-sculpt, p9-oppanel). The pipeline itself —
  op panel, modifier stack + dropdown, Material tab, camera snap, F12 tracer,
  save/load — is driven for real.

### 🔴 Bugs found for the final fix round (ranked in DONUT-RUN.md)
1. **`ApplyModifierCommand` empties the mesh when baking a ctx-less object
   modifier** (shrinkwrap/scatter): it calls `obj.mesh.copyFrom(modifier.apply(
   obj.mesh))`; the pass-through returns the same instance, so `copyFrom(self)`
   wipes it (522→0 verts). Thread `ModifierContext` into Apply AND make
   `EditableMesh.copyFrom` self-assignment safe. (Live stacks are fine — the
   tracer/renderer thread ctx — only *baking* is broken.)
2. **Scatter sprinkles render pale/near-white** despite colorVariation 1 —
   per-face tints exist (12 hue buckets) but wash out in the path trace; verify
   the tracer honours `faceTints` through tone-mapping.
3. **Lock-to-view framing too tight** — inherits default orbit distance, clips
   the plate/table out of frame.
4. Proportional default radius (2) too coarse for donut-scale lumps.
