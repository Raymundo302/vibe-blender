## The run (mirror the tutorial's beats, adapted to our substitutions)
Drive ONLY through public entry points (__app scene/io APIs, key events,
UI clicks) — the point is proving a user could do this. Screenshot after
every major stage to /tmp/donut/NN-<stage>.png.
1. Torus (Shift+A → Torus), adjust via the op panel: major 1, minor ~0.55,
   48×18 segments (staying in our unit scale; document the mapping to the
   tutorial's 0.1m). Shade smooth.
2. Proportional-edit a few verts for lumpiness (O, small random pulls) —
   our stand-in for the lattice.
3. Duplicate → icing: delete bottom half (x-ray box select), Solidify
   (thickness, offset 1, rimCrease 1), Subsurf level 2, Shrinkwrap → donut,
   modifier order shrinkwrap→solidify→subsurf, apply all three.
4. Icing dribbles: select rim verts, extrude down with proportional editing;
   sculpt-lite inflate on 2-3 droplet ends.
5. Plate: circle → extrude/scale rim per part 4 (simplified is fine), under
   the donut. Table plane below.
6. Materials via the Material tab: icing pink (roughness ~0.25, SSS 0.4),
   donut brown (SSS 0.6/0.1 radius), plate ceramic white (roughness 0.15),
   table grey. Colors from the tutorial.
7. Sprinkles: model one capsule (cylinder + subsurf + loop cuts), Scatter on
   the icing: count ~120, upOnly, minDistance, colorVariation 1, seed chosen
   for looks (try 3 seeds, keep the best, note it).
8. Camera: frame the donut (lock-to-view snap), focal ~50; Sun (warm,
   angled) + point "sky" fill (blue tint, radius for soft shadows) + white
   bounce point, per part 8's three-lamp setup.
9. F12 → path trace ≥64 samples (give it time), save the PNG to /tmp/donut/,
   plus a control render at 4 samples for the progress story. Set DoF focus
   on the icing with mild aperture.
10. Save the scene (__app.io.serialize) to research/donut.vibe.json (commit-
    able artifact); reload it and assert byte-identical re-serialization.

## Report (research/DONUT-RUN.md)

is the input for the final fix round AND the video script.

2. The final render PNG exists, is non-trivial (luminance variance), and the

3. research/donut.vibe.json round-trips byte-identical.
4. All other e2e suites still pass (flock) + npm test + tsc clean.
