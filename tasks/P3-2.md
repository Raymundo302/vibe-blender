# P3-2 — Scene save/load (JSON)

Read `tasks/P3-CONVENTIONS.md` first. This task runs FIRST in the batch — it
creates `e2e/ship.mjs` and the `topbar-btn` style others reuse.

## Goal
Ctrl+S (and a topbar "Save" button) downloads the scene as `scene.vibe.json`.
Ctrl+O (and "Open") loads one via a file input, replacing the current scene.
Loading clears the undo history (`undo.clear()` exists).

## Files
- NEW `src/io/sceneJson.ts` — `serializeScene(scene, camera): string` and
  `applySceneJson(json: string, scene: Scene, camera: OrbitCamera): void`
  (throws `Error` with a readable message on malformed input)
- NEW `src/io/sceneJson.test.ts`
- `src/ui/topbar.ts`, `src/ui/theme.css` — Save/Open buttons (`data-action`:
  `save-scene`, `open-scene`), `topbar-btn` class
- `src/input/InputManager.ts` — Ctrl+S / Ctrl+O (object mode AND edit mode both
  fine; exit edit mode before applying a load; both need preventDefault)
- `src/main.ts` — wire buttons/callbacks; expose io helpers on `window.__app.io`
  for e2e
- NEW `e2e/ship.mjs`

## Format (version it)
```json
{ "format": "vibe-blender-scene", "version": 1,
  "camera": { "target": [x,y,z], "distance": n, "yaw": n, "pitch": n },
  "objects": [ { "name": "Cube", "visible": true,
    "transform": { "position": [..], "rotation": [x,y,z,w], "scale": [..] },
    "mesh": { "verts": [[id,x,y,z],..], "faces": [[id,[vertIds]],..] } } ] }
```
Preserve vert/face ids and insertion order exactly (element pick maps depend on
order). On load: rebuild each `EditableMesh` (you may add mesh entries directly
to `mesh.verts` / `mesh.faces` Maps — or extend `EditableMesh.fromData`; if you
need a new EditableMesh method, STOP and report per the escape hatch instead),
`scene.objects.length = 0` is NOT allowed — use existing `remove()`/`add()` or
rebuild via public API only. Restore camera. `undo.clear()`.

## Acceptance criteria
1. Unit: round-trip — build a 2-object scene (cube + extruded cube), serialize,
   apply into a fresh scene, serialize again → identical strings; ids and
   iteration order preserved; malformed JSON and wrong `format` throw readable
   errors and leave the scene untouched.
2. e2e (`e2e/ship.mjs`): via `__app.io`, serialize → mutate the scene (move the
   cube, add an object) → apply the saved string → object count/positions
   restored, undo stack empty (`__app.undo` has nothing to undo — check via
   Ctrl+Z status "Nothing to undo"), app still renders (no exceptions).
3. Ctrl+S in the browser does not open the browser save dialog (preventDefault)
   — assert via e2e that a keydown dispatch with ctrl doesn't throw and status
   or download path ran (intercept per conventions).
4. `npm run build`, `npm test`, all three e2e files pass.
