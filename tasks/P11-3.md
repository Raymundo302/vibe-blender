# P11-3 — Textures: checker + image maps through UVs

Read `tasks/P9-CONVENTIONS.md` (same rules) and `tasks/P11-0-DESIGN.md`.
Parallel with P11-1 (unwrap/menus/InputManager) and P11-2 (uvEditor/main.ts) —
touch only your files.

## Core provided (F11-1, committed — do NOT modify beyond listed files)
Material gains `texKind: 'none' | 'checker' | 'image'` and `texDataUrl:
string | null` (+ runtime `texImage` decoded-pixels cache, worldData-style);
serialized in sceneJson. meshToGpu emits per-corner UVs at attribute
location 3 (0,0 when a face has no UVs). Snapshot triangles carry per-corner
UVs (`triUV`). Your job is CONSUMING all of it.

## Files (yours alone)
- `src/render/passes/renderedPass.ts` — sample the texture: checker =
  procedural in-shader (u_texKind uniform; ~8×8 via floor(uv*8)), image =
  bound TEXTURE_2D (Renderer uploads — see next line); multiply into baseColor
  before lighting.
- `src/render/Renderer.ts` — lazy per-material texture upload cache
  (dataUrl → WebGLTexture, keyed by material id + a version/url check),
  mirroring the P10-4 HDRI upload pattern; bind per draw in rendered mode.
- `src/renderEngine/tracer.ts` (+ snapshot.ts if needed) — barycentric UV
  interpolation at hit points; checker procedural / image bilinear sample
  multiplied into baseColor. Deterministic, no RNG changes.
- `src/ui/materialTab.ts` — texture rows: kind select (None/Checker/Image),
  image file input + thumbnail when kind=image (an <img>, NOT a canvas),
  undoable via the existing command patterns (image load = one command with
  old/new url, worldTab HDRI style).
- NEW unit test file(s); NEW `e2e/p11-textures.mjs`

## Acceptance criteria
1. Unit: tracer checker — two rays hitting known UV regions of a unit quad
   (seeded, tiny render or direct hit-shade probe) return alternating colors;
   image sample — a 2×2 test image maps corners correctly (bilinear at
   centers); material serialization round-trips texKind/texDataUrl
   byte-identically.
2. e2e: cube with programmatic UVs (seed directly; don't depend on P11-1),
   material → Checker via the Material tab, rendered mode → viewport shows
   alternating light/dark squares on a face (probe 2 pixels expected to
   differ); F12 render sky-vs-cube unchanged but cube pixels show checker
   variance; switch to a tiny generated data-URL image → dominant color
   changes accordingly; undo returns to None; save→load round-trip keeps the
   texture (byte-equal serialize).
3. Materials with texKind 'none' render EXACTLY as before (regression: a
   pre-P11 scene's rendered pixels unchanged — spot-probe).
4. All e2e suites (flock, donut excluded) + npm test + tsc clean.
