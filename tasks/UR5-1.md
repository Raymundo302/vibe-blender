# UR5-1 — Unified per-mode Hidden Line toggle; cage-over-wires bug; select-through

## The reported bug (fix first, regression-test it)
In edit mode, a selected edge's ORANGE only shows in wireframe shading with Hidden
Line off. With Hidden Line on — and in shaded modes with the Wireframe overlay on —
the grey wires cover the orange cage edges. Root cause (architect diagnosis,
verify it): the edit cage (`editOverlayPass`, drawn ~line 786 of Renderer.ts)
depth-fights `wirePass` lines drawn earlier with their own bias; where wire
fragments land closer, grey wins. **Fix: while an object is in edit mode, ALL
wirePass edge draws SKIP that object** (wireframe solid depth-prime still includes
its triangles so it occludes others; only its EDGE lines are skipped) — the cage
IS the edit object's wireframe, exactly Blender's model. The cage must then render
its edges/verts with its existing fractional bias against the solid depth.

## The new setting: Hidden Line, per shading mode
Replace `wireHiddenLine` in `src/render/shadePrefs.ts` with
`hiddenLine: Record<ShadingMode, boolean>` — defaults
`{ matcap: true, studio: true, rendered: true, wireframe: false }`.
- Storage: bump STORAGE_KEY to `vibe-shading-v4`; migrate a stored v3 blob's
  `wireHiddenLine` into `.wireframe` (other modes take defaults).
- Shading dropdown (`src/ui/shadingMenu.ts`): the existing Hidden Line checkbox
  (currently wireframe-only) now shows/sets the CURRENT mode's entry and is
  available in every mode. Re-sync it when the mode changes.

## Semantics per mode
- **wireframe**: hiddenLine=true → current depth-primed hidden-line look;
  false → classic see-through (current default). Unchanged rendering, new flag.
- **matcap/studio/rendered**: affects the Wireframe OVERLAY and the EDIT CAGE:
  - hiddenLine=true (default): overlay wires + cage depth-tested (current), with
    the skip-edit-object fix above.
  - hiddenLine=false: overlay wires drawn with depth test OFF (full wireframe
    visible through geometry) AND the edit cage drawn with depth test off (full
    cage, orange always visible). "so I can see the full mesh".
- **Edit-mode element click-through**: `elementPickPass` already has
  `xrayState.enabled` (Alt+Z). Effective xray becomes
  `xrayState.enabled || !shadePrefs.hiddenLine[currentMode]` — plumb it where the
  pick pass reads xrayState; do not change Alt+Z itself.

## Select-through for OBJECT mode ("can't select the object behind")
When `hiddenLine[currentMode] === false` (any mode): `Renderer.pick()` gains a
wire-proximity phase BEFORE the surface pick:
- Render object IDs into the pick target as EDGE lines (each object's `edges`
  VAO through the picking shader), depth test OFF.
- Read an N×N window (N=11) around the cursor; the non-zero pixel nearest the
  center wins → clicking on/near ANY visible wire — including one behind other
  geometry — selects that object.
- If the window is empty, fall back to the existing triangle pick (clicking a
  face body still works in shaded modes).
When hiddenLine=true: pick() unchanged. Gizmo handle picking is untouched either
way (it runs before object picking — keep that ordering).
