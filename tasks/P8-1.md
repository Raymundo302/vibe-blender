# P8-1 — Light properties tab

Read `tasks/P8-CONVENTIONS.md` first.

## Goal
A "Light" properties tab (💡) that edits the ACTIVE object's LightData live.
Lights already exist end-to-end (Shift+A → Add Light, viewport icon,
click-select, rendered mode uses them) — this task is ONLY the data UI.

## Files (yours alone)
- `src/ui/lightTab.ts` — fill the stub (registerPropertiesTab; it is already
  imported by main.ts)
- NEW unit test file next to it for any pure helpers you extract
- NEW `e2e/p8-light.mjs` — your own suite (harness pattern from ship.mjs)
- NEW `src/ui/lightTab.css` — imported from lightTab.ts, `light-tab-` prefix

## Behavior
- Tab shows an empty state ("No light selected") unless the active object's
  kind === 'light' (follow modifierTab's empty-state pattern).
- Fields: type select (Point/Sun/Spot), color (`<input type="color">`,
  hex ↔ linear 0..1 conversion — just treat hex as linear like obj.color
  elsewhere), power (number, min 0), and for spot only: angle (degrees in the
  UI, RADIANS in LightData) + blend (0..1). Spot fields hidden for other types.
- Changing type keeps color/power (defaultLight is only for NEW lights).
- Every commit (change event) pushes ONE undoable command that restores the
  full previous LightData on undo. Follow the modifierTab capture pattern.
- Live: the rendered viewport must reflect edits immediately (it already reads
  scene state per frame — you just mutate obj.light via your command).

## Acceptance criteria
1. Unit: hex↔rgb round trip; command undo restores the exact prior LightData.
2. e2e (p8-light.mjs): via __app add a light (`scene.addLight('Point','point')`,
   selectOnly), switch Properties to the Light tab, assert fields present and
   power edit + Ctrl+Z round-trips; switch type to spot → angle/blend appear.
3. All e2e suites + npm test + npm run build pass.
