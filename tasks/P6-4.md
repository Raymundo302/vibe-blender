# P6-4 ‚Äî Autosave + restore

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Every 30s (and on visibilitychange‚Üíhidden), serialize the scene to
localStorage `vibe-blender-autosave` (with a timestamp field wrapper). On boot,
if an autosave exists and differs from the pristine default scene, show a
small toast: "Restore previous session? [Restore] [Discard]" ‚Äî Restore applies
it (undo cleared), Discard deletes it. Saving/loading a file clears the
autosave (the file is now the source of truth).

## Files
- NEW `src/io/autosave.ts` (start/stop, save-now, load/clear; interval owned
  here); NEW `src/ui/restoreToast.ts`; `src/main.ts` wiring; `src/ui/theme.css`;
  `e2e/ship.mjs` appended. Expose `__app.autosave = { saveNow, clear }` for e2e.

## Acceptance criteria
1. e2e: mutate scene ‚Üí `__app.autosave.saveNow()` ‚Üí reload ‚Üí toast appears ‚Üí
   Restore brings the mutation back; second flow: Discard ‚Üí default cube scene
   and autosave key gone. Timer NOT tested (interval injected/config, unit-test
   the wrapper shape instead).
2. Boot with no autosave shows no toast (existing suites keep passing ‚Äî they
   reload often; ensure the toast only appears when an autosave EXISTS and
   differs from the default scene, and that dismissing splash doesn't race it).
3. All four e2e suites + npm run build + npm test pass.
