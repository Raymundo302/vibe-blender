## Closest point
`closestPointOnMesh(mesh, p: Vec3): { point: Vec3; faceId: number; normal: Vec3 }`
— exact closest point over triangulated faces (fan-triangulate polygons the
same way meshToGpu does). Closest-point-on-triangle via the standard
region-based method (Ericson). Brute force over faces is fine at demo scale;
keep it a pure function.

## Modifier
registerModifier('shrinkwrap', 'Shrinkwrap', ...). Params: target ('object'
kind, default -1), offset (number, default 0 — displace result along the
target's surface normal, the tutorial uses a small positive offset via
Solidify instead so 0 is fine). apply(): no ctx or unresolvable target → return
input unchanged. Otherwise for every vert: transform host-local position into
target-local space (inv(targetMatrix) * hostMatrix * p), find closest surface
point, add offset * normal, transform back to host-local. Preserve
creases/tints/topology (only vert positions change). depVersion(ctx) must
include the target's version AND both matrices (host or target moving must
re-evaluate — serialize matrix floats to fixed precision).

## Acceptance criteria
1. Unit (closestPoint): point above a triangle → projection; beyond an edge →
   clamps to edge; beyond a corner → the corner; picks the nearer of two faces.
2. Unit (modifier): host plane above a unit cube target snaps onto the cube's
   top face (all vert y ≈ cube top + offset); no ctx → identity; moving the
   TARGET transform changes depVersion; cycle (A wraps B, B wraps A) does not
   hang and both still render (context guard gives null — identity).
3. e2e: two objects via __app, add Shrinkwrap through the Modifiers tab UI,
   pick the target in the new object dropdown, assert evaluated verts moved;
   save→load round-trips the target reference (serialize twice, byte-equal,
   and the reloaded modifier's target resolves to the same-named object).
4. All e2e suites (flock) + npm test + npx tsc --noEmit pass.
