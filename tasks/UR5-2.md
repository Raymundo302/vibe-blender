# UR5-2 — Face-mode center dots + proximity face picking (builds on UR5-1)

UR5-1 (already in your tree) added the per-mode `hiddenLine` shading pref and
see-through cage/pick behavior. This task adds Blender's face-center dots and
dot-proximity face selection.

## Face dots
- `src/core/mesh/editOverlayData.ts`: in face element mode, additionally emit a
  DOT per face at its centroid (average of the face's vert positions): position +
  color arrays, orange (`SELECTED`) when that face is selected, the vert-default
  grey otherwise.
- `src/render/passes/editOverlayPass.ts`: render the dots the same way vert
  points are rendered (smaller point size than verts — pick something that reads
  clearly; verts aren't drawn in face mode so there's no clash). Depth behavior
  follows UR5-1's cage rule: depth-tested when hiddenLine=true for the current
  mode, depth-off when false.
- Dots appear ONLY in face element mode (any shading mode).

## Proximity face picking ("whatever center dot is close gets autoselected")
- When `hiddenLine[currentMode] === false` and elementMode==='face': face
  click-selection picks the face whose projected CENTROID DOT is nearest the
  click in screen pixels, THROUGH geometry (no occlusion), within a 40 CSS px
  radius (miss = deselect-all, matching current miss behavior). CPU
  implementation is fine: project all face centroids of the edit mesh (world
  matrix × centroid → screen); nearest 2D distance wins. Find where edit-mode
  face picks currently go through `renderer.pickElement(...., 'face')`
  (InputManager ~line 1164 area) and branch there — keep box/loop select
  untouched.
- When hiddenLine=true: current surface-based face picking, unchanged.
- Shift-click toggle semantics must keep working in the proximity path.

## Files
editOverlayData.ts (+ its tests), editOverlayPass.ts, the pick branch (smallest
possible touch in InputManager or a helper it calls — coordinate: UR5-4 also
edits InputManager, you run BEFORE it), e2e below.

## Acceptance criteria
- tsc clean; npm test green (editOverlayData unit tests: dot count = face count
  in face mode, zero otherwise; selected face's dot is orange; centroid math).
- e2e (extend `e2e/edit.mjs`): (1) face mode on a cube → dot pixels at visible
  face centers (region probe, camera-known coords); (2) two planes stacked along
  the view axis, front one closer; hiddenLine OFF: click nearest the BACK
  plane's center dot → back face selected, front not; hiddenLine ON: same click
  selects the front face; (3) shift-click adds a second face in proximity mode.
- Green on SwiftShader and `E2E_GPU=1`. Screenshot the dots for eyes-on.
- If ambiguous or wrong, stop and report.
