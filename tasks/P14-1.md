## The pane (editor type 'shader', title 'Shader Editor')
Header row: material name of the ACTIVE object (materialOf), a **Use Nodes**
checkbox (on first enable: create `emptyGraph()` if nodeGraph is null), and a
hint when the active object is not a mesh / has no material ('assign a
material in Properties first').

Canvas below (own 2D canvas, uvEditor.ts is the pane pattern):
- Draw nodes as rounded boxes (title bar with label, input sockets down the
  left, outputs down the right, socket dots color-coded by type: float grey,
  color yellow, vector blue). Links as bezier curves output→input.
- Param widgets: draw-in-canvas is NOT required — clicking a node selects it
  and a small DOM side-strip in the pane (right edge) shows its ParamDefs:
  float → number input+range, color → color input, select → dropdown,
  image → file input (FileReader → data URL → param + decodeNodeImage),
  ramp → SKIP rendering a custom widget; render as disabled '(ramp)' text
  (P14-3's colorRamp node may keep a JSON param editable via 2 stop color
  inputs if trivial — their problem, not yours).
- Interactions: drag node body moves it (params x/y; command on drag END);
  drag from an output socket to an input socket calls addLink (rejected link
  → brief red flash of the attempted wire); click a LINKED input socket
  unlinks it; wheel zooms (0.2–2.5×), MMB/space-drag pans; Shift+A opens an
  add-node menu (list every allNodeDefs() label; the KEYDOWN MUST be scoped
  to the pane — only when the pointer is over the shader editor canvas;
  NEVER touch InputManager); X or Delete with a selected node removes it
  (the principled output node is NOT deletable); Esc closes the add menu.
- Redraw on every change; also cheap-poll in update() for external changes
  (material switch, undo) via a signature (nodeGraphVersion + node count +
  active material id).

## Undo (graphCommands.ts)
`GraphEditCommand`: snapshots the ENTIRE graph as JSON before/after one
discrete mutation (add/delete node, link/unlink, param commit, move end,
use-nodes toggle) — undo/redo re-parse via sanitizeGraph and write back to
the material + bump nodeGraphVersion. One command per user gesture. Unit-test
round-trips.
