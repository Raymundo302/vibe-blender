# P6-3 — Per-object viewport color

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Objects get a display color (default #b0b0b0-ish neutral): a color field on
SceneObject, a color input in the properties Object tab, tinting in BOTH mesh
shaders (matcap: `texture * 2.0 * color`, studio: base color replaced), scene
persistence (accept older files → default), OBJ export unaffected.

## Files
- `src/core/scene/Scene.ts` — `color: [number, number, number]` on SceneObject
  (0..1 floats; this is the ONE allowed core edit, keep it minimal)
- `src/render/passes/meshPass.ts`, `src/render/passes/studioPass.ts` — u_color
  uniform; `src/render/Renderer.ts` — pass it per object (wireframe/picking
  unaffected)
- `src/ui/propertiesEditor.ts` — `<input type=color>` row (no undo, like
  visibility); convert hex↔float
- `src/io/sceneJson.ts` + test — persist `color`, default on absence
- `e2e/ship.mjs` appended

## Acceptance criteria
1. Unit: sceneJson round-trips color; old files without color load with the
   default.
2. e2e: set color red via the picker → readback pixel at canvas center is
   red-dominant (use canvas 2D readback via `renderer` pick FBO? No — simplest:
   screenshot before/after differ AND `__app.scene.objects[0].color[0] > 0.9`);
   matcap AND studio modes both tint (cycle Z between checks).
3. All four e2e suites + npm run build + npm test pass.
