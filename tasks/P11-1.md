## Ctrl+E edge menu (edit mode, edge selection)
Popup (AddMenu pattern): Mark Seam / Clear Seam. Sets mesh.setSeam over the
selected edges; undoable (MeshEditCommand.capture — seams live on the mesh so
the snapshot covers them). Seam edges render tinted in the edit cage.

## U unwrap menu (edit mode; operates on ALL faces if none selected, else the
selection — Blender operates on selection; document your choice)
- **Unwrap**: split faces into islands by walking face adjacency WITHOUT
  crossing seam edges or boundary; per island: map boundary loop to a circle
  (arc-length parameterized), interior verts via iterative uniform-Laplacian
  relaxation (Tutte-style, ~100 iters or convergence; corner UVs — a vert on a
  seam gets DIFFERENT uv per island side, which per-corner storage handles
  naturally). Islands scaled to equalize texel density (island uv area ∝ 3D
  area) and shelf-packed into [0,1]² with a small margin. Deterministic
  (iteration counts fixed; no RNG).
- **Smart UV Project**: cluster faces by normal (greedy: 6 axis buckets or
  angle threshold ~66°), planar-project each cluster along its dominant axis,
  pack. The robust fallback for ugly topology.
- **Project From View**: project selected faces through the current viewport
  (view matrix from ctx camera), normalize into [0,1]², replacing their UVs.
- All three undoable (mesh snapshot).

## Acceptance criteria
1. Unit: seam-split island count (cube with one seam loop → expected islands);
   Tutte boundary is a circle + interior inside it (no NaNs, all in [0,1]²
   after pack); a torus with two seam loops unwraps to a rectangle-ish island
   (aspect sanity); smart project on a cube → 6 islands; determinism (two runs
   byte-equal serialize); project-from-view maps a front-facing quad to its
   screen-space aspect.
2. e2e: cube → edit → select an edge loop → Ctrl+E Mark Seam (cage shows seam
   tint — assert via __app mesh.seams size), U → Unwrap → mesh.uvs populated
   for every face, all values in [0,1]; Ctrl+Z clears them; U → Smart UV
   Project also populates.
3. All e2e suites (flock, donut excluded) + npm test + tsc clean.
