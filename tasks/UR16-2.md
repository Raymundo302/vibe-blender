# UR16-2 — Material tab v2: shader-first UI, socket rows, fixed layout

Builds on UR16-1 (in tree). Ray's critique screenshot:
`research/ray-material-tab-critique.png` — labels and controls separated by an
ocean, swatch heights inconsistent, maps split from their sliders, preset
buttons floating mid-panel, a confusing detached "Texture" row.

## The new layout (Blender-informed, Ray-specified)
1. **Row grid**: a shared two-column property-row layout — label column
   ~110px, control column immediately adjacent, NO full-width gap. Ship it as
   a reusable CSS module (`src/ui/propRow.css` + tiny helper) — other tabs
   adopt it later (UR14-3). Uniform control heights: ALL color swatches the
   SAME height as sliders/inputs (Ray: "make the color box the same height").
2. **Shader row at the very top**, label **"Shader"** (not "Surface"):
   value = the shader name; the LEFT SOCKET CIRCLE (see 3) opens the chooser:
   Diffuse (default) / Super Shader / Metal / Glass / Emit. Switching shaders
   rebuilds the row set below (UR16-1 channel lists). The old Glass/Metal
   preset buttons are REMOVED (the shader chooser replaces them).
3. **Socket circles**: every channel row gets a small circle at its far left
   (the Blender node-socket affordance). Click → popover with the input kinds
   valid for that channel: **Value/Color · Image · Gradient**. Current kind
   shown by the circle's fill (empty ring = value, filled = image/gradient
   tint). Choosing Image opens the file picker inline; choosing Gradient
   expands an indented sub-row: two color swatches + axis X|Y|Z segmented +
   offset/scale numerics. The channel's slider/swatch stays ON THE SAME ROW —
   maps are no longer a separate section (Super's normal/bump join their
   channels the same way).
4. **Color picker clamping (Ray's bug)**: whatever popover/picker we open must
   clamp into the viewport — reusable positioning helper (the add-menu flyout
   already solves edge-flipping; reuse its approach). Native <input
   type=color> can't overflow, but our popovers can — audit every popover this
   tab opens.
5. Super shader keeps: subsurface rows, Nodes toggle + Bake Res, Always
   Textured checkbox (it moves next to the color channel row).
6. Every edit undoable exactly as today (same command path).

## Acceptance criteria
- tsc clean; units green.
- e2e (rewrite the material sections of the affected suites + a new
  `e2e/material-tab-v2.mjs`): (1) shader chooser lists 5, default Diffuse on
  a new material; switching to Glass shows exactly color/roughness/IOR/alpha
  rows; (2) socket → Image on Diffuse color loads a dataURL and the viewport
  shows it (probe); (3) socket → Gradient with axis Z on a tall object renders
  a top/bottom color split in Rendered (two-point probe) AND raytraced GPU
  viewport (parity with UR16-1's engine support); (4) swatch heights: computed
  heights of every color input in the tab are EQUAL (DOM probe); (5) label→
  control horizontal gap ≤ 16px (DOM probe on a representative row — pins the
  layout fix); (6) popover opened near the right screen edge stays fully
  on-screen (rect probe); (7) an emit image plane's Properties now shows the
  image in the EMIT color socket row (Ray's exact scenario); (8) undo works
  per edit. Existing image/html/material regressions green.
- Screenshots: the new tab for Diffuse + Super + an emit image plane, LOOKED
