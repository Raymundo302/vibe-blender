# P5-3 — Bevel edges (Ctrl+B)

Read `tasks/P4-CONVENTIONS.md` first.

⚠ This is the hardest topology task so far. Read the algorithm twice; if any
step contradicts the mesh API or you hit an unhandled case, use the escape
hatch rather than shipping a mangled mesh. Scope is intentionally v1-narrow.

## Goal
Edit mode, edge select: Ctrl+B starts a modal bevel of the selected edges —
each beveled edge slides apart into two edges with a new flat face between
them, width driven by the pointer (numeric input supported), LMB/Enter
confirm, RMB/Esc cancel.

## v1 scope (enforce, don't extrapolate)
- Only edge selections where every selected edge is INTERIOR (2 faces) and the
  selection forms disjoint simple chains/rings: every vert touches at most 2
  selected edges. Anything else → status "Bevel: unsupported selection (v1)".
- Uniform width, 1 segment, no clamping against face size (document).

## Algorithm `bevelEdges(mesh, edgeKeys, width)`
Per chain/ring of selected edges:
1. For each vert v on the chain, for each of its (1 or 2) selected edges e,
   and for each of e's two adjacent faces f: the "wing edge" is the edge of f
   at v that is NOT selected. Slide target: a NEW vert at
   `v + normalize(wingDir) * width` where wingDir points from v along the wing
   edge. A chain-internal vert (2 selected edges) gets ONE new vert per side
   (shared by both edges via the face they share); a chain-END vert gets one
   new vert per wing (2 total).
2. Each beveled edge (a,b) is replaced by a quad `[a_left, b_left, b_right,
   a_right]` (the new slid verts), wound to face outward like the removed
   edge's average face normal.
3. Adjacent faces: replace their references to a/b with the appropriate slid
   vert on THEIR side; end verts additionally keep the original vert with the
   two new verts inserted around it (triangle fan corner at chain ends).
4. Original verts with no remaining face references are deleted.
Determinism as always.

## Acceptance criteria
1. Unit: cube, bevel ONE edge, width 0.25 → 10 verts, 7 faces (6 originals
   reshaped + 1 new bevel quad), all edges manifold, Euler holds; bevel a full
   4-edge ring around the cube → 4 new quads, still manifold; width 0 →
   geometry ~unchanged positions (new verts at the originals); unsupported
   selection (a vert with 3 selected edges — cube corner) → error, untouched.
2. e2e: cube, edge mode, select one pickable edge, Ctrl+B, drag, confirm →
   face count 7; Ctrl+Z → 6; Esc cancel path untouched.
3. All four e2e suites + npm run build + npm test pass.
