# P11-5 — Modifiers preserve UVs (the dry run's gap #1)

Read `tasks/P9-CONVENTIONS.md`, `tasks/P11-0-DESIGN.md`, and
`research/UV-RUN.md` (gaps #1/#2) first. Sole worker in the tree.

## Problem
`e2e/p11-uv-dryrun.mjs` had to CLEAR the icing's modifier stack (and then hide
the torus against z-fighting) because no modifier carries `mesh.uvs` to its
output — a modified mesh renders untextured. Fix the modifiers, then remove
the dry run's workaround.

## Files (yours alone)
- `src/core/modifiers/subsurf.ts` (+ its test) — the real work: corner UVs
  subdivide EXACTLY like positions, but in per-face UV space (which handles
  seams naturally): each child quad's corners take (a) the original corner's
  uv, (b) the midpoint of the two adjacent corner uvs along that face edge,
  (c) the face-average uv. NO cross-face averaging for UVs (unlike positions)
  — that's what keeps islands intact; document this standard simplification.
  Faces without UVs stay without UVs.
- `src/core/modifiers/solidify.ts` (+ test) — copy face UVs to both shells;
  rim quads get no UVs (document).
- `src/core/modifiers/mirror.ts`, `src/core/modifiers/array.ts` (+ tests) —
  mirrored/instanced faces copy the source face's UVs verbatim.
- `src/core/modifiers/scatter.ts` (+ test) — instances copy the SOURCE
  object's face UVs.
- `src/core/modifiers/shrinkwrap.ts` — verify (and test) it already preserves
  UVs (it should if it clones; add the regression test either way).
- `e2e/p11-uv-dryrun.mjs` + `research/UV-RUN.md` — remove the clear-stack and
  hide-torus workarounds: the checker must now show on the FULL modified icing
  (shrinkwrap+solidify+subsurf live). Update the report's gap section to
  "FIXED (P11-5)" with the new numbers. Regenerated research/donut-uv.vibe.json
  is expected to change — fine.

## Acceptance criteria
1. Unit per modifier: a single UV'd quad through subsurf level 1 → 4 child
   quads whose corner UVs are the exact bilinear subdivision (hand-check
   values); level 2 stays in the face's original UV span; solidify → both
   shells carry the quad's UVs; mirror/array/scatter copies verbatim;
   shrinkwrap preserves. Determinism: apply twice → byte-equal serialize.
2. e2e: the updated dry run runs green WITHOUT clearing the icing stack — the
   checker shows on the evaluated (subsurf'd) icing in rendered mode AND F12,
   with the same non-vacuity controls as before.
3. All e2e suites green under flock (donut.mjs INCLUDED) + npm test + tsc.

## Result

**Modifiers now carry `mesh.uvs` to their output:**
- `subsurf.ts` — corner UVs subdivide EXACTLY like positions but in PER-FACE UV

  midpoint of the two adjacent corner uvs along that face edge, (c) the
