# UR11-1 — Curve objects: Bezier + NURBS, curve edit mode

## Goal
New object kind `'curve'`: Bezier and NURBS splines as first-class objects —
drawn as smooth polylines in the viewport, editable control points via Tab,
the geometric substrate for the Pipe modifier (UR11-2) and future
follow-path/text-on-curve.

## Model (`src/core/curve/` new module + payload)
- Payload:
```ts
curve?: {
  kind: 'bezier' | 'nurbs';
  cyclic: boolean;                        // closed loop
  resolution: number;                     // eval segments per span (default 12, 2–64)
  // bezier: points with handles; nurbs: weighted control points
  points: Array<{ co: [x,y,z]; hl?: [x,y,z]; hr?: [x,y,z]; w?: number }>;
  order?: number;                         // nurbs order k (default 4, clamped ≤ #points)
}
```
- `src/core/curve/eval.ts` (pure, unit-tested): `evaluateCurve(data): Vec3[]`
  — bezier: cubic per span from (co,hr) → (hl,co) of the next (auto-mirrored
  handles when absent); nurbs: clamped-uniform knot vector, Cox-de Boor basis
  with weights (rational). Cyclic closes the loop (bezier wraps spans; nurbs
  periodic-lite: wrap control points, document the simplification). Also
  `curveLength(pts)` + `frames(pts)` — parallel-transport frames (position,
  tangent, normal, binormal) for UR11-2's sweep; unit-test frame continuity
  (no flips) on a helix.
- Shift+A → **Curve ▸ Bezier / Circle / NURBS** (new category above Empty):
  Bezier = 2-point gentle S (Blender-like default), Circle = 4-point cyclic
  bezier circle radius 1, NURBS = 5-point straight-ish strip. Spawn at cursor.
- Serialization (version bump, tolerant), kind-aware duplicate (deep copy),
  join guard skips curves, outliner glyph (∿ or similar).

## Display
- Viewport: the evaluated polyline drawn through the wire ribbon path (UR6) in
  all modes — object color, selection tint when selected; the curve has NO
  faces/mesh (empty EditableMesh; pick via the polyline: reuse the wire-
  proximity pick so clicking the line selects it in every mode — check what
  UR5-1's wireProximityPick needs to also run for curves when hiddenLine is
  ON; simplest: curves' polylines always go into the edge-pick pass).

## Curve edit mode (Tab)
- Tab on a curve → CURVE EDIT (not mesh edit): control points drawn as vert-
  style points (orange when selected); bezier also draws handle lines +
  smaller handle points. Click select (point or handle), Shift extends;
  **G** moves selected (modal, same operator infra — axis locks + numeric
  work via the existing Move machinery if feasible; else a curve-move modal
  following the edgeSlide undo pattern); moving a bezier CO drags its handles
  along; moving one handle mirrors the other when they were mirrored.
- **Ctrl+click**: append a new point after the last (co at the picked
  ray∩grid-plane or surface point — reuse the 3D-cursor placement logic).
  **X**: delete selected points. Cyclic toggle in the N-panel/properties
  section ("Curve" section: kind [readonly], cyclic checkbox, resolution,
  order for nurbs).
- Undo: every commit one entry (move = modal capture; add/delete = commands).
- Esc/Tab exits. Mesh objects unaffected (regression).

## Acceptance criteria
- tsc clean; npm test green (eval: bezier endpoint/midpoint exactness vs
  closed-form, nurbs circle-ish weight test + partition of unity, cyclic
  continuity, parallel-transport no-flip on helix; serializer round-trip).
- e2e (new `e2e/curves.mjs`): add each preset; polyline pixels present;
  click the line selects the object; Tab → points visible, click+G moves a
  point and the polyline follows (pixel probe), one undo reverts; Ctrl+click
  appends (point count +1); X deletes; cyclic toggle closes the loop (probe:
  pixels near the gap position); save→load round-trip; Tab on a cube still =
  mesh edit.
- Screenshot: edited bezier with handles visible, for eyes-on. Free-text
  report (STATUS:/VERDICT: first line). If ambiguous, STOP and report.
