## World type
```ts
interface World {
  mode: 'flat' | 'gradient' | 'hdri';
  color: [number, number, number];          // flat
  horizon: [number, number, number];        // gradient
  zenith: [number, number, number];
  strength: number;                          // default 1
  hdri: string | null;                       // data URL, mode 'hdri'
}
```
defaultWorld(): gradient matching the tracer's CURRENT hardcoded sky (so old
scenes render identically ‚Äî that's the regression bar), strength 1.

## Acceptance criteria
1. Unit: serialization round-trip (all three modes, byte-identical; hdri data
   URL survives); default world reproduces the old sky (tracer miss-ray color
   equals the pre-change gradient at 3 sample directions ‚Äî import the tracer
   in Node); equirect lookup math (known pixel ‚Üí known direction).
2. e2e: World tab present without any selection; set flat red ‚Üí rendered-mode
   viewport background pixel reads red AND an F12 render's sky pixels read
   red; gradient mode round-trips through save/load; undo restores the
   previous mode. (HDRI file-input e2e may drive the internal load function
   with a small generated data URL instead of a real file dialog.)
3. Old scenes (no world key ‚Äî e.g. research/donut.vibe.json) load with the
   default world and the donut still renders with its familiar sky (assert
   loading it doesn't throw and world.mode === 'gradient').
4. All e2e suites pass under flock (donut.mjs excluded ‚Äî architect's final
   sweep) + npm test + npx tsc --noEmit.
