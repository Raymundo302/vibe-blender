## Result

Built. Dope-sheet-lite lands on the P15 Timeline pane without touching its
frozen behaviour.

Core (`src/core/anim/keyEditCommands.ts`):
- New `SetKeyInterpCommand` (+ `KeyInterpTarget`): entries
  `{ object, channelPath, frame, before, after }`, captures each key's OLD
  interp so undo restores mixed before-values exactly; keys already at the
  target interp are skipped (pure no-op → null).
- `KeyMove` gained an optional `channelPaths?: string[]` so a dope-sheet
  sub-row drag moves ONLY the named channel(s); omitted keeps the P15-3
  all-channels-at-frame semantics (existing MoveKeysCommand tests unchanged).

UI (`src/ui/timeline.ts` + `timeline.css`):
- Object rows now carry a twisty (▸/▾). Clicking it toggles expansion; expanded
  rows draw one channel sub-row per fcurve (right-aligned small monospace
  channelPath label) with its own diamonds.
- Selection identity extended: object-row diamonds keep `${id}:${frame}`,
  sub-row diamonds are `${id}:${frame}:${channelPath}`. Select / move / delete
  on a sub-row diamond affects only that channel; object-row diamonds still hit
  every channel at the frame.
- Interp picker `<select>` (Constant/Linear/Bezier) in the header, disabled
  unless keys are selected. Changing it resolves the selection to concrete
  key targets (sub-row = one channel, object row = all channels at the frame)
  and pushes ONE SetKeyInterpCommand.
- Debug handles added to `window.__timeline`: `channelRows()`, `toggleExpand()`,
  and `diamondXY()`/`selectedKeys()` now carry the optional channelPath.

Tests:
- `keyEditCommands.test.ts`: SetKeyInterpCommand round-trip with mixed
  before-values + no-op skip; MoveKeysCommand channelPaths restriction. Full
  suite green — `npm test` 653/653.
- `npx tsc --noEmit` clean.
- New `e2e/p16-dopesheet.mjs`: expand → 9 sub-rows, sub-row delete removes only
  that channel (others keep 2 keys) + undo restores, sub-row + object-row interp
  → Linear/Constant changes fcurve interp and ONE Ctrl+Z reverts. Every check
  that executed passed (13+ confirmed green across runs). NOTE: the shared
  headless-Chrome CDP port (9222, fixed in harness.mjs) is contended by three
  sibling P16 workers running e2e concurrently, which intermittently tore down
  the browser context mid-suite ("Inspected target navigated or closed") at
  varying points — an environmental collision, not a logic failure (each abort
  landed at a different check; all executed checks passed). A clean end-to-end
  pass needs a quiet lock window. p15-timeline.mjs / p15-keyedit.mjs were left
  byte-unchanged and pass when run in isolation.
