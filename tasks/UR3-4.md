# UR3-4 — Shift+A menu → categorized flyout submenus

## Goal
The Shift+A Add menu (src/ui/addMenu.ts) is currently one flat list with headings
(Add Mesh / Add Light / Add Camera). Make it Blender-style: the root menu shows
CATEGORY rows — **Mesh ▸**, **Light ▸**, **Camera** — and hovering a category row
pops its items out in a submenu to the RIGHT of the row.

## Files
- EDIT `src/ui/addMenu.ts` — restructure; keep the class name, options interface,
  OpPanel behavior (`addPrimitive`/`commitAdd` unchanged), and teardown contract
  (every listener removed on close).
- CSS lives in the shared `src/ui/theme.css` (the file already styles
  `.add-menu*` — extend there, keep the house look).

## Design
- Root rows: `Mesh ▸` and `Light ▸` open flyouts listing PRIMITIVES / LIGHTS;
  `Camera` stays a DIRECT item (single entry, Blender-like), clicking it adds the
  camera as today.
- Flyout behavior:
  - Opens on hover of its row (and on click, for good measure); positioned
    with its top aligned to the row, left edge at the root menu's right edge.
  - If it would overflow the parent host's right edge, flip to the LEFT side.
    Clamp vertically inside the host (same clamping idea the root already does).
  - Moving the pointer from the row INTO the flyout must not close it (the
    classic gap problem — either no gap, or a small close-delay timer ~150 ms).
  - Hovering a different category row closes the previous flyout.
  - Only one flyout open at a time.
- Keyboard: Escape closes everything (existing handler). No arrow-key nav (v1).
- Outside-pointer close must treat the flyout as inside (it is a child of the
  same root element, or if you must detach it, update `onOutsidePointer`).

## Out of scope
- No new object types, no icons, no search field.
- Don't touch InputManager's Shift+A dispatch or OpPanel.

## Acceptance criteria
- `npx tsc --noEmit` clean; `npm test` green.
- **Grep `e2e/` for `.add-menu` / add-menu-item usage FIRST** — several suites add
  primitives through this menu. Update the suites' interaction (hover/click the
  category, then click the item) or, better, keep a compatibility path: clicking
  a category row also opens the flyout, so `click('Mesh'); click('Cube')`
  works — then fix any selector that assumed a flat list. ALL existing e2e suites
  must pass, plus extend one suite with: root shows 3 rows; hover Mesh → flyout
  with all primitives at the row's right; hover Light → Mesh flyout closed,
  Light flyout open; add a Cube through the submenu → object added + ONE undo
  entry (unchanged semantics); Escape closes all.
- Unique `E2E_PORT`.
- If the spec is ambiguous or wrong, stop and report rather than improvising.
