# UR4-4 — Shift+A → Image → HTML… (live or snapshot HTML planes)

Builds ON TOP of UR4-3 (already in your tree: Image submenu, createImagePlane,
shadeless materials). Adds an **HTML…** item: pick an .html file, it renders onto
a plane. A small choice popup (or two items: "HTML Snapshot…" / "HTML Live…" —
pick the two-items design, it matches the menu grammar and needs no new widget)
controls whether it stays a one-shot snapshot or auto-refreshes from disk.

## Files
- NEW `src/tools/htmlPlane.ts` (+ tests for the pure parts).
- EDIT `src/ui/addMenu.ts` — two items under Image ▸, after Diffuse/Emit.

## Rasterization (core mechanism)
- Read the file text; rasterize in-browser via SVG foreignObject:
  build `<svg xmlns=… width=W height=H><foreignObject width="100%" height="100%">
  <html xmlns="http://www.w3.org/1999/xhtml">…user content…</html>
  </foreignObject></svg>`, load as an Image via a data/blob URL, draw to a canvas
  → dataURL → `createImagePlane(..., {mode: 'emit'})` (blueprints want exact
  pixels; shadeless). Default raster size 1024×768 unless the html's root
  declares explicit width/height (keep simple: fixed 1024×768, document it).
- **Document the hard limits in the module header AND the status line on add:**
  scripts do NOT execute and external resources (http images/fonts/css) do NOT
  load inside SVG-image rasterization — self-contained HTML only. Malformed HTML
  that kills XML parsing → wrap body in a try; on failure rasterize an error
  card (plane with "HTML failed to parse" text) rather than throwing.

## Live vs snapshot
- **Snapshot…**: plain `<input type=file>` read once. Works in every browser.
- **Live…**: use `window.showOpenFilePicker()` (Chrome File System Access API —
  the app ships as a Chrome `--app` window, see CLAUDE.md "Launching the App") to
  get a re-readable FileSystemFileHandle. Poll every 2s: `getFile()`, compare
  `lastModified`; changed → re-rasterize → update the SAME material's texDataUrl
  in place + invalidate whatever texture cache Rendered mode keeps for image
  materials (grep the RAW decode cache in the material tab / renderer and reuse
  its invalidation path). Edit-the-file-in-your-editor → texture updates in-app.
  - The poller: one module-level interval while any live plane exists; store the
    handle on the SceneObject via a NON-serialized runtime map (WeakMap by object
    — file handles can't go in sceneJson). After save/load the plane loads as a
    static image plane (document: liveness is session-only).
  - Deleting the object (or undo of the add) must stop its polling.
  - If `showOpenFilePicker` is missing (Firefox/headless), fall back to snapshot
    behavior with a status note "Live HTML unavailable — added snapshot".
- Undo: same single-entry contract as UR4-3.

## Out of scope
- No interactive/clickable HTML on the plane, no JS execution, no iframes/URLs.
- No serialization of live handles.
