# UR10-1 — Area light

## Goal
New light type `'area'` — a rectangle emitter with soft shadows in the path
tracer and a sane approximation in the Rendered viewport.

## Model
- `LightData.type` union += 'area'; new fields `width`, `height` (world units,
  defaults 1×1; clamp > 0.01). Emits from its face along LOCAL **−Z** (Blender
  convention; our lights already aim via object rotation — grep how spot aims).
  sceneJson bump, tolerant load, kind-aware duplicate already generic (verify).
- Light tab (src/ui/lightTab.ts): Type select gains Area; Width/Height numeric
  rows shown only for area (existing per-type row pattern — spot has cone
  fields). Undo per commit as the tab already does.
- Viewport gizmo: rectangle outline (4 lines in the light's local XY, w×h) +
  the existing aim arrow along −Z — extend lightDirPass or the frustum-pass
  line drawing, riding the icons overlay toggle + selection tint.

## Tracer (the real implementation)
- Direct-light sampling: sample a uniform random point on the rect per light
  sample (the P9-4 soft-shadow machinery samples point/sun/spot radii — follow
  its structure), one-sided emission (contribution 0 when the shading point is
  behind the rect's −Z face), standard solid-angle/pdf weighting
  (dist², cosθ_light). Multiple samples come free from progressive spp.
- Energy: match Blender's feel loosely — radiance scales with energy /(w·h)
  so a bigger light at the same energy is dimmer per-area but casts softer
  shadows. Document the exact formula.

## Rendered viewport (approximation — document it)
- Shade as a point light at the rect CENTER with the area's energy and a
  clamped-distance falloff, PLUS wrap the N·L term slightly by light size
  (softens terminators). NO shadow map for area v1 (skip it in renderShadows;
  document). This is a stopgap until LTC; looks reasonable, never crashes.

## Acceptance criteria
- tsc clean; npm test green (serializer round-trip incl. gaps; energy formula
  unit test).
- e2e (extend the suite covering lights — grep p8/lights): (1) add area light
  via Shift+A → Light ▸ Area; fields appear; (2) tracer F12 low-spp: floor lit
  by an area light overhead shows a PENUMBRA — probe: shadow edge luminance
  gradient across ≥3 px (vs a point light's hard edge ≤1-2 px on the same
  scene); (3) one-sided: flip the light 180° → floor goes dark; (4) rect gizmo
  lines present (pixel probe, icons overlay on); (5) Rendered viewport shows
  the floor lit (approximation active, no crash), old scenes load.
- Screenshot: tracer render of cube on floor under a tilted area light
  (visible soft shadow) for eyes-on. E2E_GPU=1 spot check. Free-text report
  (STATUS:/VERDICT: first line). If ambiguous, STOP and report.
