# P7-1 — Join objects (Ctrl+J)

Read `tasks/P4-CONVENTIONS.md` first (chains to earlier conventions).

## Goal
Object mode, ≥2 selected: **Ctrl+J** merges every selected mesh into the ACTIVE
object (Blender semantics). Geometry is baked through each source's world
transform into the active object's LOCAL space (`inv(activeModel) * srcModel`).
Source objects are removed. One undoable command restores everything (objects,
list order, selection, the active object's original mesh).

## Files
- NEW `src/core/mesh/ops/join.ts` (pure: `appendBaked(target, source, matrix)`)
  + NEW test file
- NEW `src/core/undo/joinCommand.ts` (perform-style like DeleteObjectsCommand —
  study `objectCommands.ts` first)
- `src/input/InputManager.ts` — Ctrl+J (object mode only)
- `e2e/ship.mjs` — append checks

## Notes
- Modifiers: keep ONLY the active object's stack (Blender does the same);
  sources' modifier stacks are dropped with the objects (the undo snapshot
  restores them on Ctrl+Z — snapshot whole SceneObjects like DeleteObjectsCommand
  does, plus the active mesh via clone()).
- <2 selected → status message, no-op. Active must be among the selection.

## Acceptance criteria
1. Unit: cube at origin + cube moved (3,0,0) & scaled 2 → join → active has
   16 verts / 12 faces; a baked vert lands exactly where its world position was
   (hand-compute one); ids don't collide; edges remain manifold per-shell.
2. e2e: add second cube (Shift+A menu path or __app), move it, select both,
   Ctrl+J → 1 object, evaluated verts 16; Ctrl+Z → 2 objects with the moved
   one back at its position AND active object's mesh back to 8 verts.
3. All four e2e suites + npm run build + npm test pass.
