# P3-3 — Shading modes: matcap / wireframe / studio

Read `tasks/P3-CONVENTIONS.md` first. Runs after P3-1.

## Goal
Z cycles the viewport shading: **matcap** (current) → **wireframe** (edges only,
no solid fill) → **studio** (flat lambert with a fixed key light) → matcap.
A topbar chip shows the current mode.

## Files
- NEW `src/render/passes/studioPass.ts` — lambert shader pass, same shape as
  `meshPass.ts` (begin/setObject, normal matrix, `layout(location = 0/1)`)
- NEW `src/render/passes/wirePass.ts` — draws every visible object's unique
  edges (`meshToRenderData(...).edgePositions`) as dark lines; cache per
  (object id, mesh.version) like `Renderer.gpuMesh` does — you will need a
  second cached VertexArray; add it to the renderer's cache entry type
- `src/render/Renderer.ts` — `shadingMode: 'matcap' | 'wireframe' | 'studio'`
  (default 'matcap'); branch the solid pass on it. In wireframe mode the
  outline/edit passes still work (mask uses triangles as today).
- `src/input/InputManager.ts` — Z key (both object and edit mode, no modifiers)
- `src/ui/topbar.ts` — mode chip (`data-action="shading-mode"`, clicking it also
  cycles)
- `e2e/ship.mjs` — append checks

## Studio shader
Two hardcoded view-space directional lights (key `normalize(0.4, 0.6, 0.7)`
intensity 0.9 warm-white, fill `normalize(-0.5, -0.2, -0.4)` intensity 0.35
cool) + 0.12 ambient; base color `vec3(0.62)`; no specular. Half-lambert is NOT
wanted — plain `max(dot(n,l), 0)`.

## Acceptance criteria
1. e2e: `__app.renderer.shadingMode` cycles matcap→wireframe→studio→matcap on
   Z presses; topbar chip text follows; clicking the chip cycles too; screenshot
   each mode to /tmp (matcap vs studio vs wireframe must differ — compare PNG
   byte lengths differ as a cheap proxy and save all three for human review).
2. Edit mode still fully works in wireframe shading (Tab + A + G smoke check
   in e2e).
3. `npm run build`, `npm test`, all three e2e files pass.
