## Behavior
🎞 (and Ctrl+F12 keydown registered by YOUR module on window — check it's
free) opens a small modal: mode (WebM video / PNG sequence), fps (default
scene.fps), Start/End (default scene range), Render + Cancel. Rendering:
loop start→end; per frame applyAnimation → force a Rendered-mode
renderer.render pass → capture the canvas. WebM: canvas.captureStream(0) +
MediaRecorder, requestFrame per rendered frame, timestamps at fps; PNG mode:
toBlob per frame into a minimal STORE-only (no compression) ZIP writer you
implement locally (~100 lines, CRC32 table) → one .zip download. Progress bar
+ frame counter + working Cancel (abort cleanly). Restore frameCurrent,
playing and prior shading mode after. Refuse when start ≥ end (status msg).

## Result
BUILT (2026-07-08).

### What shipped
- `src/renderEngine/animRender.ts` — pure helpers `frameCount` (inclusive),
  `crc32` (table, canonical), `buildStoreZip` (STORE-only ZIP: local headers +
  central directory + EOCD, deterministic) + the `AnimRender` controller.
  - Modal: mode (WebM video / PNG sequence), fps (default scene.fps), Start/End
    (default scene range), Render + Cancel, progress bar + frame counter.
  - Ctrl+F12 registered on window in the CAPTURE phase with stopPropagation, so
    the render engine's plain-F12 bubble handler (init.ts, no modifier check)
    does NOT also fire. Plain F12 still reaches the render engine untouched.
  - Render loop: force `renderer.shadingMode='rendered'` + look through the
    active camera (`renderer.cameraViewId = scene.activeCameraId`); per frame
    `scene.frameCurrent = f` → `applyAnimation` → `renderer.render`.
  - PNG: `gl.readPixels` into a Y-flipped 2D scratch canvas (synchronous —
    robust against the context's `preserveDrawingBuffer:false`) → `toBlob` →
    packed into one store-only .zip.
  - WebM: `canvas.captureStream(0)` + `MediaRecorder`, one `requestFrame` per
    rendered frame spaced at 1/fps → one .webm.
  - Cancel aborts cleanly (loop breaks, promise resolves `null`). Restores
    frameCurrent / playing / shading mode / cameraViewId in a `finally`, then
    re-poses the saved frame. Refuses start >= end with a status message.
- `src/renderEngine/animRender.css` — modal styling (theme vars).
- `src/renderEngine/animRender.test.ts` — 7 unit tests (frame math inclusive,
  crc32 canonical "123456789"=0xCBF43926, zip local-header + CRC parse, EOCD
  entry count, determinism).
- `src/ui/topbar.ts` — 🎞 Render Animation button (title "Render Animation
  (Ctrl+F12)") + `toggleRenderAnimation` action.
- `src/main.ts` — constructs `AnimRender`, wires the topbar action, exposes
  `__app.animRender` { render(opts)→Promise<Blob|null>, cancel, open, close,
  isRunning } for headless use.
- `e2e/p16-animrender.mjs` — loads research/donut-flythrough.vibe.json, drives
  `__app.animRender.render`. 19 checks, all pass under flock.
