## Alt+click loop select (edit mode)
- Edge mode: Alt+click an edge → select the full edge loop (walk: at each
  4-valence vert continue with the edge "across" — not sharing a face with the
  incoming edge; stop at boundaries/poles or on closing the loop). Shift+Alt+
  click adds a second loop to the selection.
- Face mode: Alt+click a face → face loop through quads (enter via the edge
  under the click: use the picked face + nearest edge; document your choice).
- Vertex mode: like edge mode, selecting the loop's verts.
- Pure walk logic in loopSelect.ts with unit tests on grid/torus meshes
  (makeTorus is parametric now).

## X-ray select-through (Alt+Z + topbar chip)
When ON: element picking and box select must hit BACK-side elements too.
Implementation guidance (verify feasibility, adjust honestly): elementPickPass
renders element ids with depth — for x-ray, render WITHOUT the mesh-occlusion
prepass / with depth test disabled so back elements win where front ones are
absent; box select in x-ray mode selects every element whose projection falls
in the rect regardless of depth (the CPU projection path in boxSelect can just
skip its occlusion check if one exists — read the real code first). Blender
also draws the mesh translucent in x-ray; a cheap alpha hint in the solid pass
is optional polish, not required.

## Shift+N recalculate normals (edit mode)
recalcNormals(mesh, faceIds): flood-fill winding consistency across shared
edges (BFS from the largest-area face of each island; neighbors sharing an
edge traversed in the SAME direction get flipped), then orient each island
outward (positive signed volume vs island centroid; flip all if negative).
Undoable (MeshEditCommand.capture). Status "Recalculated normals".

## Shift+E crease (edit mode, edge selection)
Modal operator like the existing transforms: horizontal pointer motion (or
typed numeric 0..1, or -1 to clear) sets the crease weight on the selected
edges via mesh.setCrease; live preview (creased subsurf updates each frame);
LMB/Enter confirm, RMB/Esc cancel+restore, undoable. Show weight in status.
Creased edges draw highlighted in the edit cage IF cheap (editOverlayPass is
NOT yours — skip visual flair rather than touching it; say so in Result).

## Ctrl+Alt+Numpad0 — snap active camera to current view
Sets the active camera's transform from the current OrbitCamera pose
(position = eye, rotation = look direction via quaternion from the view
basis; write a pure helper + unit test round-tripping through cameraViewMatrix
from P8-2). Undoable (transform command pattern exists in nPanel/properties).
Status 'Camera set to view'. If no camera exists: create one via
scene.addCamera + AddObjectsCommand, then snap it (Blender-ish, document).
