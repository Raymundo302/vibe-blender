## PieMenu component (generic — P13+ will reuse it)
`new PieMenu({ parent, x, y, title, items, onClose })` where items are
`{ label, disabled?, action }`. Render up to 8 wedge buttons arranged in a
circle around (x, y) (Blender order: W, E, S, N, NW, NE, SW, SE for items
0..7), a small title chip in the center, dark theme via CSS variables from
theme.css (--bg-panel etc. — see addMenu/collectionMenu styling for the
pattern). Interactions: click a wedge → run action + close; Escape or click
outside → close; disabled wedges render greyed and inert. Clamp within the
parent. Own all listeners; remove them on close (AddMenu is the pattern).

## snapOps.ts (pure logic, unit-tested)
Export functions taking (scene) or (scene, undo):
- `cursorToOrigin(scene)` — cursor = (0,0,0).
- `cursorToSelected(scene)` — object mode: cursor = median of selected
  objects' WORLD positions (`scene.worldTransformOf(o).position`). Edit mode:
  median of selected verts mapped through `scene.worldMatrix(editObject)`
  (use `scene.editMode.selectedVertIds(mesh)`).
- `cursorToGrid(scene)` — cursor = snapVec(cursor, SNAP_STEP).
- `selectionToCursor(scene, undo)` — OBJECT MODE ONLY: every selected object's
  world position := cursor (each object individually, Blender default). Use
  captureWorldTargets + writeWorldPosition, push ONE TransformCommand
  ('Selection to Cursor') covering all moved objects.
- `selectionToGrid(scene, undo)` — object mode only: each selected object's
  world position := snapVec(worldPos, SNAP_STEP). Same command pattern
  ('Selection to Grid').
Cursor-only moves are NOT undoable (Blender semantics). Return a short status
string from each (e.g. `'Cursor to Selected'` / `'Snapped 3 object(s)'`) so
the binding can setStatus it.

## Shift+S binding (InputManager)

Shift+C): open the pie at this.pointer with items in this order:
Cursor to World Origin / Cursor to Selected / Cursor to Grid /
Selection to Cursor / Selection to Grid. The two Selection items are
`disabled` in edit mode and when the selection is empty. Track the open pie in
a field like the collectionMenu pattern (re-press closes). Modal-operator
active → the activeOp branch already returned; nothing to do.
