## The pane
Editor type 'image', title 'Image Viewer'. Layout: a toolbar row (source
dropdown + zoom controls + 'Fit' button + pixel readout) over a canvas that
fills the rest.

- **Source dropdown**: 'Render Result' (default) + one entry per material
  image slot in the scene, labeled like `Icing — Base Color` /
  `Icing — Normal` / `Icing — Roughness` / `Icing — Metallic` (only non-null
  slots). Rebuild the list on update() when the material set changes (compare
  a cheap signature string; don't rebuild every frame).
- **Render Result**: draws the last completed F12 render, or a centered
  'No render yet — press F12' hint. Material images: decode the data URL via
  an Image element (cache per URL).
- **Zoom/pan**: wheel zooms around the pointer (0.1×–32×), MMB-or-LMB drag
  pans, 'Fit' re-letterboxes. Nearest-neighbor upscale ≥8× (crisp pixels:
  ctx.imageSmoothingEnabled = false above 1×).
- **Pixel readout**: hovering shows `x, y  R G B` of the image pixel under the
  pointer (read from an offscreen canvas of the source; linear 0–255 display
  values are fine).

## Result
Built. New `src/ui/imageViewer.ts` + `imageViewer.css` register an `'image'`
editor (title "Image Viewer") in `src/main.ts`'s `editorFactories`.

- **Layout:** toolbar (source `<select>` + −/+ zoom buttons + Fit + right-aligned
  pixel readout) over a canvas that fills the pane body.
- **Sources:** "Render Result" (default) plus one entry per non-null material
  image slot, labeled `Name — Base Color / Normal / Roughness / Metallic`
  (SLOT_ORDER tex/normal/rough/metal). The dropdown is rebuilt only when a cheap
  material-slot signature changes, not every frame.
- **Render Result** reads the last completed F12 frame via the ONE additive edit
  outside my files: `getLastRender(): HTMLCanvasElement | null` exported from
  `src/renderEngine/init.ts` (returns the render window's canvas when
  `sample > 0`; the canvas retains its pixels after the window closes). Shows a
  centered "No render yet — press F12" hint when there is none.
- **Material images** are decoded through a per-URL cached `Image`, blitted to an
  offscreen natural-size source canvas (the read-back surface for the readout).
- **Zoom/pan:** wheel zooms around the pointer (0.1×–32×), LMB-or-MMB drag pans,
  Fit re-letterboxes. `imageSmoothingEnabled=false` above 1× for crisp pixels.
  Source changes / first-appearance auto-Fit once (signature-gated so a
  progressive render doesn't re-fit every frame).
- **Pixel readout:** hovering shows `x, y  R G B` of the image pixel under the
  pointer (read from the offscreen source canvas, stored 0–255 values).
- Pure view math (`fitView`, `zoomAroundPoint`, `screenToImage`,
  `imageToScreen`, `clampZoom`) is exported and unit-tested
  (`imageViewer.test.ts`, 11 tests).
