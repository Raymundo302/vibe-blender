# UR8-2 — Text object: kind, typing mode, properties, fonts, keyable thickness, convert to mesh

Builds on UR8-1 (in your tree: `src/core/text/*` engine). Integrates Text as a
first-class object.

## Object model
- Kind `'text'` (objectData precedent: light/camera/empty). Payload:
```ts
text?: {
  content: string;           // the string, \n allowed
  font: string;              // family name (probed availability, fallback monospace)
  size: number;              // world units per em (default 0.5)
  wrap: boolean;             // OFF by default
  wrapWidth: number;         // em units (default 12) — used when wrap on
  align: 'left'|'center'|'right'|'justify';
  style: 'face'|'outline'|'both';
  faceColor: [number,number,number];
  outlineColor: [number,number,number];
  thickness: number;         // world units, KEYABLE channel 'text.thickness'
}
```
- The object's `mesh` is REGENERATED from the payload via buildTextMesh on any
  payload change (and per sampled frame when thickness is animated — UR8-1's
  glyph cache makes this cheap; verify the ~2ms claim in playback).
- sceneJson: next version bump, old files tolerate absence. Kind-aware
  duplicate; join skips text? NO — join converts: treat text like mesh for
  join by converting first (simplest: join GUARD skips text v1, document).
- Shift+A root item **Text** (below Empty) — spawns "Text" with content
  "Text", face style, thickness 0.05.
- Keyable channel `text.thickness` via channelPaths (UR7-1 adds html.playing —
  same pattern; coordinate: that code is in your tree already).

## Typing mode (Tab)
- Tab with a text object active → **Text Edit mode** (not mesh edit): a hidden
  input/textarea captures keystrokes; characters append/edit `content` at a
  caret; draw a caret marker in the viewport? v1: NO in-viewport caret — show
  the live-updating mesh plus a status chip "Editing text — Tab/Esc to finish";
  typing edits the END of the string, Backspace deletes, Enter = newline,
  arrow keys move a caret INSIDE content (track caret index; edits apply
  there). All other viewport keys suppressed while typing (F1 etc. must not
  fire — swallow like the help overlay does).
- ONE undo entry per typing session (capture content at entry, push on exit
  if changed — the modal-geometry capture pattern applied to the payload).
- Esc/Tab exits. Plain meshes keep mesh edit (regression).

## Properties → "Text" section (active object has text payload)
- **Textarea** bound to content (commit on blur/Enter-with-Ctrl, one undo per
  commit) — Ray: "a little text box in the property panel".
- **Font dropdown rendered in the actual fonts**: a CUSTOM dropdown (not
  <select> — option styling is unreliable): each entry styled
  `font-family: <that font>` showing the family name. Curated candidate list
  (probe with fontAvailable, show only available): Arial/Helvetica, Georgia,
  Times New Roman, Courier New, monospace, Verdana, Trebuchet MS, Impact,
  Comic Sans MS, DejaVu Sans, DejaVu Serif, Liberation Sans, Liberation Mono,
  Press Start 2P (if installed). Selecting rebuilds the mesh.
- Size, Wrap toggle + Wrap Width (enabled when wrap on), Align (4-way
  segmented control), Style (face/outline/both), Face Color + Outline Color
  (color inputs — grep the per-object color field pattern), Thickness numeric
  + **● key button** on text.thickness (lightTab keyBtn pattern).
- **Convert to Mesh button** (Ray follow-up): replaces the object in place —
  same id/name/transform/parent/material, mesh = the current generated mesh
  (deep copy), `text` payload REMOVED → it's a plain mesh now (editable in
  mesh edit mode, modifiers, everything). ONE undo entry restores the text
  object exactly. Status: "Converted to mesh".

## Acceptance criteria
- tsc clean; npm test green (payload serializer round-trip; convert-to-mesh
  undo restores payload; regenerate-on-change dirty logic).
- e2e (new `e2e/text-object.mjs`, unique port): (1) Shift+A → Text appears,
  mesh non-empty; (2) Tab → type "Hi" → content updates + mesh vert count
  changes, Esc exits, ONE undo reverts the session; (3) properties textarea
  commit changes content (undoable); (4) align right vs left shifts projected
  glyph positions the correct direction; wrap on + narrow width increases line
  count (probe mesh bbox height grows); (5) style face vs outline vs both
  face-count relations (both > face > outline in tri count on "oo"); (6) key
  thickness at two frames → scrub → mesh bbox depth differs between frames;
  (7) Convert to Mesh → Tab now enters MESH edit on it, undo → text payload
  back; (8) font dropdown lists ≥3 entries, each option's computed
  font-family equals its label.
- Screenshot: "Bouncy Ball" in a serif at thickness 0.2, style both, two
  colors, for eyes-on. If ambiguous or wrong, stop and report.
