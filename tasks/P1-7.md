## Topbar content (left → right)
- App title: `Vibe Blender` (small, bold).
- Mode indicator chip: `Object Mode` (hardcoded for now — Phase 2 adds Edit
  Mode; render it as a distinct pill/chip so the video can point at it).
- Right side: active object name + object count (`Cube — 1 object`), updated
  in `update()` with the same signature-diff pattern the panels use.

## Theme (Blender 4.x dark, close is good enough)
- Palette: window chrome `#1d1d1d`, panels `#2b2b2b`, inset fields `#1f1f1f`,
  borders `#3d3d3d`, text `#d0d0d0`, muted text `#8a8a8a`, accent (selection,
  focus rings, active row) `#4772b3`, header text 12px, body 12px
  `"Segoe UI", system-ui, sans-serif`.
- Style: topbar (28px tall, bottom border), sidebar panels (`.panel`,
  `.panel-title` — collapsible NOT required), outliner rows (hover, selected =
  accent bg at ~35% alpha, active = brighter text), number inputs (dark,
  no spin buttons in Firefox/Chrome, accent focus ring), buttons, the add-menu
  popup (dark, hover accent), scrollbar (thin, dark, WebKit + Firefox).
- The viewport/status bar look must not regress.

## Acceptance criteria (headless CDP + screenshot)
1. Topbar renders title, `Object Mode` chip, and live right-side status (add an object via console → count updates).
2. All UI (topbar, both panels, add-menu) uses the shared palette — capture `Page.captureScreenshot` with the add-menu open and eyeball-check the PNG yourself by decoding it (verifier will too).
3. No stray `<style>` blocks remain from the panel files IF you consolidated them (grep).
4. `npm run build`, `npm test`, `node e2e/smoke.mjs` all pass — layout changes must not break canvas sizing (`syncSize` math is display-size driven; the sidebar/topbar simply reduce the canvas box).
5. Number inputs still editable, rename input still works (focus styling visible).
