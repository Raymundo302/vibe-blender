# P5-2 — Bridge edge loops

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Edit mode, edge select: pick two separate edge loops with the same edge count
and press **Ctrl+E** → the loops get connected by a ring of quads (Blender's
Bridge Edge Loops). This is how you connect two cylinders, close tubes, etc.

## Files
- NEW `src/core/mesh/ops/bridge.ts` + NEW `src/core/mesh/ops/bridge.test.ts`
- `src/input/InputManager.ts` — Ctrl+E in edit mode (edge mode only; status
  message otherwise)
- `e2e/edit.mjs` — append checks

## Algorithm
`bridgeLoops(mesh, edgeKeys: Set<string>): { newFaceIds: number[] } | { error: string }`
1. Partition the selected edges into connected chains by shared verts. Exactly
   2 chains required, equal edge counts, both closed or both open — otherwise
   return `{ error }` (shown in the status bar, no mutation).
2. Order each chain into a vert path (open: end-to-end; closed: cyclic).
3. Pairing: for open chains, try both directions of loop B and pick the one
   with the smaller total vert-to-vert distance. For closed chains, try every
   rotation of B in both directions (≤ 2n candidates) and pick the minimal
   total distance — this avoids twisted bridges.
4. Quads: `[a_i, a_{i+1}, b_{i+1}, b_i]` for each i (n-1 quads open, n closed).
5. Winding: after building, if the average dot of each new quad's Newell normal
   with the vector from the bridge centroid to that quad's centroid is negative,
   rebuild all quads reversed (heuristic outward orientation for tube-like
   bridges; document that interior bridges may need a manual flip later).
6. Select the new faces? No — keep the edge selection; return newFaceIds.
Wrap in MeshEditCommand.capture('Bridge Edge Loops') at the call site.

## Acceptance criteria
1. Unit: two 4-edge square loops (two detached quads' boundaries, one offset
   +2 on Y, built via EditableMesh.fromData with the two quads) → bridge →
   4 new quads, previously-boundary edges of both loops become manifold
   (2 faces each); mismatched counts (4 vs 3) → error, mesh untouched;
   >2 loops → error; open-chain case: two parallel 2-edge paths bridge into
   2 quads; twisted closed pairing picks the non-twisted rotation (assert no
   new edge longer than 1.5× the loop offset distance).
2. e2e: delete cube; import (via __app.io.importObj) a two-quad OBJ fixture,
   enter edit mode, edge mode, select-all, Ctrl+E → faces grow by 4, undo
   restores; Ctrl+E in vert mode shows a status message and mutates nothing.
3. All four e2e suites + npm run build + npm test pass.
