# P4-5 — Mirror + Array modifiers

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
The first two real modifiers: **Mirror** (reflect geometry across an object-
space axis plane) and **Array** (N copies along an offset vector).

## Files
- NEW `src/core/modifiers/mirror.ts`, `src/core/modifiers/array.ts`
- NEW `src/core/modifiers/builtins.ts` — imports both for their side-effect
  registration; exports nothing
- `src/main.ts` — one `import './core/modifiers/builtins';` line
- NEW `src/core/modifiers/mirrorArray.test.ts`
- `e2e/ship.mjs` — append checks

## Mirror
- Fields: axis ('x'|'y'|'z', default 'x').
- Output = input clone + mirrored copy: every vert duplicated with the axis
  coordinate negated; every mirrored face's vert order REVERSED (winding flips
  under reflection — backface culling is on, this is visible if wrong).
- Deterministic id/order: first all original verts/faces (same ids via clone),
  then mirrored verts in original iteration order, then mirrored faces.
- No merge-at-seam in v1 (verts on the mirror plane simply duplicate).

## Array
- Fields: count (int 1..10, default 2), offsetX/offsetY/offsetZ (number,
  default 2/0/0). count includes the original (count 1 = unchanged clone).
- Copies at i * offset for i in 0..count-1, same dedup-free duplication
  approach as Mirror, deterministic ordering.

## Acceptance criteria
1. Unit: cube+Mirror → 16 verts / 12 faces, mirrored face normals still point
   OUTWARD (use faceNormal · centroid-direction like mesh.test.ts does);
   verts on the plane duplicate; axis param respected. Cube+Array count 3 →
   24 verts / 18 faces at the right offsets; count 1 → same counts as input;
   both modifiers pure (input mesh untouched — assert verts.size unchanged).
2. e2e: add Mirror to the cube via the modifier tab UI, offset the cube's base
   mesh so the mirror is visible (G move in edit mode or set transform), assert
   `evaluatedMesh().verts.size === 16` while `mesh.verts.size === 8`; toggle
   enabled off → evaluated === base. Add Array on top → evaluated counts
   multiply. Apply (first modifier) → base mesh grows, stack shrinks, Ctrl+Z
   restores both.
3. Save/load: scene with modifiers round-trips through __app.io (v2 format from
   P4-4).
4. All four e2e suites + npm run build + npm test pass.

## Result
Built **Mirror** (`src/core/modifiers/mirror.ts`) and **Array**
(`src/core/modifiers/array.ts`) modifiers, registered via new
`src/core/modifiers/builtins.ts` (imported once by `main.ts`). Both are pure:
`apply` clones the input, then appends duplicated geometry with deterministic
ids/order (originals via `clone()`, then new verts in iteration order, then new
faces). Mirror reflects across an object-space axis plane and reverses each
mirrored face's winding so normals stay outward; no seam merge in v1. Array
emits `count` (1..10) copies at `i * (offsetX,Y,Z)`, count 1 = unchanged clone.

Verification: `npm run build` clean; `npm test` 148 passed (11 new in
