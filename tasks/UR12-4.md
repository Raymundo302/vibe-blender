# UR12-4 — GPU path tracer, stage 4: parity harness + performance ledger

Builds on UR12-3. The proof stage — no new features.

## Parity harness
1. `e2e/gpu-parity.mjs`: for a curated scene list — default cube, donut
   fixture (frozen), area-light penumbra scene, emissive-panel room, glass+
   gold hero, DoF two-depth, HDRI world — render CPU and GPU at 128 spp,
   256×144, compute **SSIM on luminance** (implement SSIM as a pure exported
   helper + unit test on known pairs; no dependency). Thresholds: ≥ 0.90 per
   scene, ≥ 0.95 for cube/donut (looser where fireflies dominate — justify
   any per-scene threshold in comments). Any scene whose feature was
   documented-out in UR12-2 is SKIPPED with a visible log line, not silently
   green.
2. Difference images (amplified |CPU−GPU|) saved to research/gpu-parity-*.png
   for every scene — the AO-saga diagnostic habit.
3. **Performance ledger**: a small table appended to research/GPU-RENDERER.md
   — per scene: CPU ms/spp, GPU ms/spp, speedup, at 512² and 960×540 on the
   real Vega (E2E_GPU=1). Plus SwiftShader sanity timings (slow is fine;
   crash is not).
4. Firefly/NaN sweep: 1024-spp GPU render of the glass hero → assert zero
   non-finite pixels and < 0.01% pixels above 10× median luminance.
5. Wire the parity suite into the repo's e2e conventions (unique port, GPU
   flag respected) and document how to run it in the file header.

## Acceptance criteria
- All parity scenes pass their thresholds on the real GPU; diff images exist
  and were LOOKED at (call out the worst scene + where the residual lives).
- The ledger table exists with real numbers.
- tsc clean, units green, no regressions in gpu-tracer-1/2/3 suites.
- Free-text report: first line STATUS/VERDICT, then the headline speedup and
  the parity table inline.
