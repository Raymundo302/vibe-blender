# P4-3 — Properties editor: tabbed shell + Object tab

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Rebuild the Properties panel as Blender's tabbed properties editor: a slim
VERTICAL tab strip on the left edge (icon buttons), content on the right.
Ships with one tab — **Object** (the existing transform fields + rename +
visibility). The Modifier tab arrives in P4-4 through the same registration
API you build here.

## Files
- NEW `src/ui/propertiesEditor.ts` — the tabbed shell + registration API +
  the Object tab implementation (move/adapt code from `src/ui/properties.ts`)
- `src/ui/properties.ts` — DELETE (git rm) once nothing imports it
- `src/main.ts` — the 'properties' editor factory creates the new editor
- `src/ui/theme.css` — tab strip styles (Blender-dark, ~26px wide buttons,
  active tab highlighted with the accent color)
- `e2e/ship.mjs` — append checks

## Design
- `export interface PropertiesTab { id: string; icon: string; title: string;
  build(container: HTMLElement, ctx: PropertiesTabContext): { update(): void } }`
  where `PropertiesTabContext = { scene: Scene; undo: UndoStack }`.
- `export function registerPropertiesTab(tab: PropertiesTab): void` — module-
  level registry, same pattern as modifiers. The editor renders one icon button
  per registered tab (`data-tab="<id>"`, `title` tooltip); clicking switches;
  active tab persists per session (plain field, not localStorage).
- Object tab: object name input (rename pushes the existing RenameObjectCommand),
  visibility checkbox, and the transform Location/Rotation/Scale fields exactly
  as they behave today (reuse the logic/undo semantics from properties.ts —
  move the code, keep the behavior). Multiple editor instances may exist at
  once (two areas can both show Properties) — no module-level DOM state.
- Icons are text/emoji for now: Object '⬛', tooltip 'Object'.

## Acceptance criteria
1. e2e: properties area shows the tab strip with the Object tab active; rename
   via the name input updates the outliner + topbar; visibility checkbox hides
   the object in the viewport (pick at canvas center returns null); transform
   edit still works and undoes (existing ship.mjs properties checks keep
   passing, adjusted only if selectors changed).
2. All four e2e suites + npm run build + npm test pass.

## Result
Built `src/ui/propertiesEditor.ts`: a tabbed shell (`PropertiesEditor`) with a
slim vertical icon strip (26px buttons) + content pane, a module-level tab
registry (`PropertiesTab`, `PropertiesTabContext`, `registerPropertiesTab`,
`propertiesTabs`) mirroring the modifier pattern, and the built-in **Object** tab
(icon '⬛') carrying an editable name (rename → `RenameObjectCommand`), a Visible
checkbox, and the Location/Rotation/Scale fields moved verbatim from
`properties.ts` (same undo semantics, same skip-while-focused refresh). No
module-level DOM state — each editor instance owns its tabs, so two Properties
areas coexist. `main.ts` creates it from the 'properties' factory; `properties.ts`
git-rm'd (nothing imported it); tab-strip + name/visibility styles added to
`theme.css`. Verified: `npm run build` (tsc + vite) clean, `npm test` 133/133,
and all four e2e suites green (smoke, edit, ship, workspace) — ship.mjs gained
P4-3 checks covering tab strip + active tab, rename→outliner+topbar+undo,
visibility checkbox hiding the object (center pick → null), and transform
edit+undo. No deviations from the spec.
