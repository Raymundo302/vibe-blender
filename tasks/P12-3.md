## Hierarchy display
Within each collection group (and the root group), show parent objects with
their children indented beneath (16px per depth), depth-first, cycles
impossible (core guards). A parent row gets a ▸/▾ twisty (expanded state kept
per panel instance like the collection twisties); collapsed parents hide their
subtree rows. A child whose parent is in ANOTHER collection still renders
under its own collection, un-nested, with a subtle `↖ parent-name` hint suffix
— do not invent cross-collection nesting.

## Drag to parent
Dragging an object row and dropping it ONTO another object row parents the
dragged object (and every OTHER selected object, if the dragged row was part
of the selection) to the drop target via SetParentCommand ('Parent'). Dropping
onto a collection header or empty panel space clears the parent
(SetParentCommand with null, 'Clear Parent'). Use HTML5 drag events or
pointer-based dragging — match whatever the outliner already uses for rows (if
nothing, pointer-based: down + move threshold 4px + highlight drop row + up).
A refused drop (cycle — SetParentCommand returns null) sets a status message
'Parent: would create a cycle' via the panel's existing status/toast access;
if the outliner has no setStatus access, silently no-op (do NOT add new
plumbing for it).

## Result
Built. Outliner now nests children under parents within each group (collection
or scene root), 16px per depth, depth-first: `renderGroup`/`renderSubtree` walk
group-roots (members whose parent isn't in the same group) and their in-group
descendants. Parent rows carry a ▸/▾ twisty (per-panel `collapsedObjects` Set,
default expanded) that hides the subtree; leaves get an aligned spacer. A child
whose parent lives in another collection renders un-nested with a subtle
`↖ parent-name` hint. Pointer-based drag-to-parent (4px threshold, drop-target
highlight): dropping a row onto another object row parents the dragged object
(and the whole selection if it was part of it) via `SetParentCommand('Parent')`;
onto a collection header or empty space clears via `SetParentCommand('Clear
Parent', …, null)`. Cycle-refused drops (null command) silently no-op — the
panel has no setStatus access (constructed `new OutlinerPanel(scene, undo)`), so
per spec no new plumbing was added. Rename/eye/delete/collection controls all
preserved; deleting a parent keeps its children (core reparents, we re-render).
Touched only `src/ui/outliner.ts`, `src/ui/outliner.css`, and new
`e2e/p12-outliner.mjs` (17 checks, all pass). tsc clean, 490 unit tests pass,
smoke/edit/workspace e2e pass under flock.

Reviewer note: a drag that re-renders detaches the press target so the browser's
synthesized click never fires; `suppressNextClick` is therefore cleared on the
next tick (`setTimeout 0`) to avoid it leaking onto a later unrelated click
(this was the one e2e failure caught and fixed during the build).
