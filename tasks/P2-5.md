# P2-5 ‚Äî Inset (I), individual faces

Read `tasks/P2-CONVENTIONS.md` first.

## Goal
In edit mode with faces selected, I insets each selected face individually:
a smaller copy of the face appears inside it, connected by a ring of quads, and
the pointer drives the inset amount until confirm/cancel. Vert/edge mode: status
message "Inset: face mode only".

## Files
- NEW `src/core/mesh/ops/inset.ts` ‚Äî pure topology function + unit tests
  (NEW `src/core/mesh/ops/inset.test.ts`)
- NEW `src/tools/inset.ts` ‚Äî the modal operator
- `src/input/InputManager.ts` ‚Äî wire I in `onEditModeKey`
- `e2e/edit.mjs` ‚Äî append checks

## Topology algorithm (per selected face ‚Äî follow exactly)
`insetFaces(mesh, faceIds: Set<number>): { innerFaceIds: number[], innerVertsByFace: Map<number, number[]> }`
For each face F (verts v0..vn-1, centroid c):
1. New vert per corner at the SAME position as the corner (`addVert(co)`) ‚Äî the
   modal phase moves them toward the centroid; topology and geometry stay separate.
2. New inner face with the new verts, same winding.
3. Ring quads: for each edge (vi, vi+1): `[vi, vi+1, newVi+1, newVi]`.
4. F itself is DELETED (`deleteFaces([F.id])`) ‚Äî the inner face replaces it.
Store each inner face's `(cornerCo, centroid)` pairs so the modal phase can set
`co = lerp(cornerCo, centroid, t)` for t in [0, 0.95].

## Modal operator
- "Modal TOPOLOGY tools" undo pattern; command name 'Inset'.
- t driven by horizontal pointer distance from where I was pressed (clamped
  0..0.95); numeric input sets t directly (0.3 = 30% toward the centroid).
- After topology: selection = inner faces, `sel.touch()`.
- Status: `Inset  t: <t>  LMB/Enter: confirm  RMB/Esc: cancel`.

## Acceptance criteria
1. Unit: inset one cube face ‚Üí 12 verts, 10 faces (5 original + 1 inner + 4 ring);
   inset with t=0.5 puts inner verts at the midpoint corner‚Üîcentroid; cancel
   restores 8/6; all edges manifold after inset.
2. e2e: edit mode, face mode, select one face (via `__app` evaluate or click),
   I, move pointer, confirm ‚Üí face count 10; Ctrl+Z ‚Üí 6.
3. `npm run build`, `npm test`, `node e2e/edit.mjs` all pass.

## Result
Built individual-face inset.
- NEW `src/core/mesh/ops/inset.ts` ‚Äî `insetFaces(mesh, faceIds)` creates per-face corner copies (at the same position), an inner face (same winding), a ring of quads `[vi, vi+1, newVi+1, newVi]`, and deletes the original face. Also exports `faceCentroid(mesh, faceId)` for the modal phase. NEW `inset.test.ts` (5 tests).
- NEW `src/tools/inset.ts` ‚Äî `InsetOperator` following the modal-TOPOLOGY undo pattern (snapshot before ‚Üí mutate as preview ‚Üí `MeshEditCommand.fromSnapshots` on confirm, `copyFrom(before)` on cancel). `t` driven by horizontal pointer distance from the press point (300 px ‚Üí t=0.95, clamped 0..0.95); numeric input sets `t` directly. Slides each inner vert `lerp(cornerCo, centroid, t)`. Selection becomes the inner faces. Status: `Inset  t: <t>  LMB/Enter: confirm  RMB/Esc: cancel`.
- `src/input/InputManager.ts` ‚Äî wired `I` in `onEditModeKey`; vert/edge mode reports "Inset: face mode only".
- `e2e/edit.mjs` ‚Äî appended P2-5 checks.

Verification: `npm run build` (tsc + vite) passes; `npm test` 93/93 (unit criteria: 12 verts / 10 faces, t=0.5 midpoint corner‚Üîcentroid, cancel restores 8/6, all edges manifold); `node e2e/edit.mjs` all checks pass (I in face mode ‚Üí 10 faces / 12 verts / inner face selected, Ctrl+Z ‚Üí 6 faces, vert-mode I shows the face-only status).
