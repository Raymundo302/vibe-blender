## Soft shadows
Lights gain radius: sample a point on the light's sphere (uniform, seeded RNG)
per shadow ray instead of the center. Sun: `radius` reinterpreted as angular
radius in radians (default 0 = hard) — jitter the sun direction within the
cone. Rendered-viewport shading ignores radius (raster stays hard) — fine.

## Subsurface scattering (tracer)
Honest cheap approximation, document it: when a diffuse bounce hits a material
with subsurfaceWeight > 0, with probability = weight treat it as SSS:
(a) direct light: wrap the NdotL term (lerp toward (NdotL+1)/2) and tint by
baseColor, (b) continue by offsetting the hit point along -normal by a random
distance ~ subsurfaceRadius and re-emerging with a cosine-weighted direction
(single-scatter dipole-ish). Energy must not increase (clamp). The donut/icing
must visibly glow at grazing light vs subsurfaceWeight 0 — prove via pixel
comparison in e2e, not adjectives.

## Depth of field
Camera gains OPTIONAL thin-lens params in the SNAPSHOT only (no CameraData
change): `fstop`-style aperture radius + focusDistance, exposed as two fields
on the render window UI (aperture default 0 = pinhole = current behavior,
focus distance default = distance to scene bounding-box center). Primary rays:
jitter origin on the lens disk, aim at the focal plane point (seeded RNG).

## Acceptance criteria
1. Unit: light with radius 0 → identical image to pre-change (seeded, compare

   vs 0 changes grazing-light pixels; energy conservation (average luminance
   does not increase by more than ~5% with SSS on); aperture 0 → pinhole-

2. e2e (p9-render2.mjs): donut-ish scene (torus + point light with radius,

   radius 0) in the expected direction (shadow-edge pixel brighter, grazing
   pixel brighter). Set aperture in the render window → off-plane blur
   detectable. Esc still terminates cleanly.
3. All e2e suites (flock) + npm test + npx tsc --noEmit pass.

### e2e recalibration (the actual fix here)
The old `p9-render2.mjs` metric ("darkest quartile of LIT pixels over a center
crop") did not isolate either effect: the crop is dominated by bright neutral
floor, so a soft-shadow lift was diluted and the SSS signal was swamped. Two
checks failed statistically even though the physics is right.

I probed the real rendered frames (headless, via a scratch diagnostic) to find
where each effect actually lands, then rewrote the checks as PAIRED per-pixel
measurements — bands defined on the CONTROL frame, the SAME pixels re-read in
the feature frame (seed is a constant 0x1234567, so control vs feature differ
only by the feature). Donut vs floor is separated by per-pixel warmth (R−B):
the donut material is warm #e8b48c, the floor neutral grey, so `rb > 15` =
donut, `rb < 8` = floor — robust to Monte-Carlo noise, no hard-coded crop.

Physically-honest findings that drive the new checks (16-sample reads):
- **Soft shadow** lifts the FLOOR cast-shadow penumbra: floor pixels with
  control lum 40..120 go 81.8 → 93.6 (+14.5%); fully-lit floor (>170) is
  unchanged (213.2 → 212.9). The light was moved to a grazing position
  (point, pos (3.5,1.2,2.5), power 4000) so the shadow has a wide penumbra and
  the donut is grazing-lit.
- **SSS** makes the DONUT glow: the spec's wrapped-diffuse model lifts
  grazing-LIT donut pixels (control lum 110..220) 180.1 → 194.6 (+8.0%), warm-
  tinted, while the neutral fully-lit floor is unchanged (energy is
  redistributed on the donut, not added globally). NOTE: this is a deviation
  from the task's assumed "terminator brightens / contrast shrinks" narrative —
  the spec's SSS (wrapped NdotL + dipped, tinted, lossy continuation) brightens
  the grazing-LIT surface (the glow) and slightly DARKENS the deep, indirect-lit
  terminator crescent; it does not lift the terminator. I verified this against
  the frames and the wrapped-diffuse unit test, so I asserted the true effect
  (glow) rather than force a contra-physical "contrast shrinks". Matches the
  spec's acceptance wording "grazing pixel brighter".
- **DoF** unchanged in spirit: pinhole vs aperture 1.2 / focus 60 drops crop
  variance 3040.8 → 1855.5.

Non-vacuity proven: a scratch variant that forces the comparison renders to
radius 0 / weight 0 flips exactly the two feature detectors to FAIL (penumbra
81.78 → 81.78, glow 180.15 → 180.15 — identical), while the "unchanged" guards
correctly stay green.
