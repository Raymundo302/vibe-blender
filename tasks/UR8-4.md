# UR8-4 — Extract Element tool (html planes → layered depth scenes)

Builds on UR7 (page mode, html payload) + UR8-3 (transparent auto-cropped
planes, alphaBlend). Ray's goal: click elements of a page and pull them out as
their own alpha planes to build parallax/depth from one HTML file.

## The tool
- Toolbar entry **Extract Element** (icon ⧉ or similar), visible ONLY while in
  page mode on an html plane (UR7-2's mode; toolbar is mode-aware already).
  Also a key? No key v1.
- Activating enters an ELEMENT-PICK overlay inside page mode:
  - Maintain a hidden live DOM mirror of the page (a detached iframe or a
    display:none div rendering the sanitized page at pageW×pageH — pick the
    approach where getBoundingClientRect + elementFromPoint work; a
    same-origin sandboxed iframe srcdoc is the robust one).
  - As the pointer moves over the plane in the viewport, map pointer → plane
    UV → page px (the plane's world quad ↔ raster mapping; invert what the
    portal matrix / raster pipeline already knows) → `elementFromPoint` in the
    mirror → draw a highlight rectangle overlay on the plane texture (cheap:
    a DOM outline div positioned over the viewport like the knife preview —
    do NOT re-rasterize per hover).
  - Click = extract that element. Shift+click = extract several, finishing
    with Enter; Esc cancels.
- **Extraction semantics** (one undo entry for the whole activation):
  1. The element's `outerHTML`, with COMPUTED styles inlined (walk the element
     + descendants, `getComputedStyle`, inline the non-default longhands —
     pragmatic subset: layout/box/color/font/background/border/shadow/
     transform/animation properties; document the subset) becomes a new
     FRAGMENT source; positioning context normalized (strip absolute page
     offsets; the fragment renders standalone at its own size).
  2. New plane via the UR8-3 fragment path (transparent + autoCrop), sized by
     the element's rendered bbox, PLACED co-planar with the source plane at
     the element's position on it (bbox center → plane-local offset), nudged
     +0.01 world units along the plane normal (in front) — "add depth": Ray
     then moves it further.
  3. The ORIGINAL plane's source gets the element hidden persistently:
     inject a unique data attribute during mirroring and a
     `[data-vibe-extract="N"]{visibility:hidden !important}` rule appended to
     the source's head (visibility not display — keeps the layout hole, which
     is what you want for parallax). Original re-rasters.
  4. Animations: the fragment carries its own animation CSS; its `html`
     payload starts with playing=false and its own page clock (UR7-1) —
     document that extracted elements animate independently.
- Undo restores: source text, the removed planes, selection.

## Acceptance criteria
- tsc clean; npm test green (style-inline subset function on a fixture DOM —
  jsdom ok for the pure walker if it runs; else e2e-only with the function
  exported).
- e2e (new `e2e/extract-element.mjs`, unique port): load a two-element page
  (bouncy ball div + shadow div inside a wrapper, distinct colors, on a plain
  background); page mode → Extract Element → synthetic pointer over the ball
  wrapper → highlight rect appears (DOM probe) → click → (1) a NEW plane
  exists, transparent-cropped, its raster contains the ball color and NOT the
  page background at corners; (2) the ORIGINAL plane's raster no longer shows
  the ball (pixel probe at its old position) but still shows the rest;
  (3) the new plane sits in front (world position probe along the normal);
  (4) ONE undo → original pixels back, extracted plane gone; (5) save→load
  round-trips both planes (source text with the visibility rule serializes).
- Screenshot: extracted ball hovering in front of its page, viewed at an
  angle, matcap + Always Textured. If the pointer→page mapping fights the
  workspace layout or portal math, STOP and report rather than approximating.
