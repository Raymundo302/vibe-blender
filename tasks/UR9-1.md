# UR9-1 — Shading dropdown: collapsible AO / Wireframe / Intersections sections with controls

## Goal
The viewport shading dropdown (src/ui/shadingMenu.ts) is a flat list. Three of
its features become **collapsible sub-sections** (disclosure rows: ▸/▾ + the
feature's existing enable checkbox on the header row; expanding reveals the
controls; collapsed state persisted per section in the prefs blob):

1. **Ambient Occlusion** — the EXISTING controls (Mode, Method, Radius,
   Strength, Samples) move under the disclosure; the AO checkbox stays on the
   header row. No behavior change, pure reorganization.
2. **Wireframe** (the overlay checkbox row — and these controls also apply to
   wireframe SHADING MODE wires):
   - **Color** — color input driving the wire color (today hardcoded dark
     0.05,0.05,0.06 in wirePass; the ribbon path already has a u_color
     uniform — plumb it). One color pref for both overlay + wireframe mode.
     The edit cage keeps its own colors (NOT affected).
   - **Thick to Thin toggle** — the UR6 proximity scaling on/off. Off = a
     constant width (use the Max value below).
   - **Thin / Thick** numeric sliders — MIN_PX (0.3–2, default 0.6) and
     MAX_PX (1–8, default 3.5) feeding the ribbon clamp (constants in
     src/render/passes/ribbon.ts become defaults for the prefs).
3. **Intersections** — **Color** input for the intersection line
   (intersectPass core grey 0.72 / rim behavior stays; the COLOR becomes the
   pref, rim derives from it darkened as today's ratio).

## shadePrefs
New fields: `wireColor: [r,g,b]`, `wireProximity: boolean` (default true),
`wireMinPx`, `wireMaxPx`, `intersectColor: [r,g,b]`, plus
`sections: { ao: boolean; wire: boolean; intersect: boolean }` (expanded
state, default collapsed). Storage key bump v4→v5 with migration (v4 blob →
same values + new defaults; the UR5-1 v3→v4 migration chain still works from
v3). Clamp/sanitize on load per house pattern.

## Plumbing
- wirePass / ribbon: width params + color from prefs per draw (Renderer
  reads shadePrefs where it begins the passes — grep the aoPass pattern);
  wireProximity=false → both clamp bounds = wireMaxPx.
- intersectPass: color uniform from prefs.
- Live update: changing a control re-renders next frame (prefs are read per
  frame already — verify).

## Files
shadingMenu.ts (+css), shadePrefs.ts (+test), ribbon.ts/wirePass.ts,
intersectPass.ts, Renderer.ts call sites. Nothing else.
