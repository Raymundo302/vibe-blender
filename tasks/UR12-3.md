# UR12-3 — GPU path tracer, stage 3: engine picker + render window + Ctrl+F12

Builds on UR12-2. Ship the GPU engine to the user.

## Integration
1. **Render window (F12)**: an Engine select `CPU | GPU` (persisted viewport
   pref via viewPrefs.ts pattern; default GPU when the context+extensions
   probe succeeds, CPU otherwise, with the reason shown as a tooltip).
   GPU progressive display: accumulate in gpuTracer, blit/readback to the
   render window canvas at a steady cadence (~4 Hz readback, not per sample);
   sample counter + Save PNG + the existing f/I fields keep working; Cancel
   aborts cleanly (no context leak; reuse one gl context across renders).
2. **Glare + post**: the UR10-2 glare post must apply identically to GPU
   renders (it operates on the output buffer — route the GPU result through
   the same seam; if glare is CPU-side on Float32Array, readback then post —
   fine at end-of-render cadence).
3. **Ctrl+F12 animation renders**: Engine select gains GPU; per-frame:
   re-pack ONLY what changed (transforms/lights/materials cheap path vs full
   re-pack on topology change — key on the same caches the viewport uses;
   document granularity), render N spp, feed the existing sinks. The UR7
   html-plane prepareFrame await still happens BEFORE packing each frame.
4. **Determinism**: same scene+seed+spp ⇒ identical PNG on the same machine
   (both engines, each internally deterministic; engines need NOT match each
   other bit-wise).
5. Failure honesty: context loss mid-render → surface "GPU context lost —
   falling back to CPU" status and continue the job on CPU from sample 0.

## Acceptance criteria
- tsc clean; units green. New `e2e/gpu-tracer-3.mjs`: (1) F12 with GPU engine
  → progressive counter advances, Save PNG non-black; (2) cancel mid-render →
  window closes, second render works (no leak); (3) glare-on GPU render shows
  the halo (probe); (4) Ctrl+F12 GPU 3-frame PNG-seq: animated cube moves
  across frames, per-frame determinism (re-render frame 2 → identical CRC);
  (5) engine pref persists; (6) CPU path regression: the existing render
  suites (p8-render, ur10-2-render, ur10-3-glass, donut S9 if box allows)
  still green with engine=CPU.
- Report GPU vs CPU wall-clock for the donut hero at 64 spp 960×540 on the
  Vega (the video's headline number).
- Screenshots: render window mid-GPU-render + final. Free-text report.
