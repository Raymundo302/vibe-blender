# UR8-3 — Element planes: transparent auto-cropped fragments; alpha materials; Always Textured

Builds on UR7 (in your tree: html payload, rasterizeHtmlAt, page raster
pipeline). Three coupled features.

## A. Fragment adds crop to content + carry alpha
Ray's case: an HTML fragment of just the bouncy ball + its shadow should
become a plane sized to that content's bounding box (ball's apex to shadow),
with real transparency around it.
- Rasterizer options: `{ transparent?: boolean; autoCrop?: boolean }`.
  transparent → NO white ground fill (svgToCanvas skips the fillRect; the
  wrapXhtml reset's `background:#ffffff` becomes background:transparent).
  autoCrop → after drawing, scan the alpha channel for the content bbox
  (padding 2px), re-draw into a canvas of exactly that size; return the crop
  size so the plane geometry uses the CONTENT aspect, not 1024×768.
- Heuristic on add (both file planes and UR7 dialog): a source WITHOUT a
  `<body`/`<html` tag (a bare fragment) defaults to transparent+autoCrop; full
  documents keep the white full-page raster. Expose both as checkboxes in the
  add dialog so the default is overridable.
- pageW/pageH semantics for cropped planes: store the crop box; scrolling
  (page mode) is disabled for cropped fragment planes (nothing to scroll —
  document).

## B. Alpha in the materials/renderers
Textures currently render opaque. Add material flag `alphaBlend: boolean`
(default false; set true automatically for transparent rasters):
- Rendered viewport: draw alphaBlend materials in a second pass after opaque
  meshes, blended (SRC_ALPHA/ONE_MINUS_SRC_ALPHA), depth-test ON, depth-WRITE
  OFF, sorted back-to-front by view depth (object granularity is fine v1).
  Shadeless path multiplies alpha through.
- Matcap/studio: only relevant via part C's always-textured draw — same
  blended pass there.
- Tracer: alpha < 0.5 → the hit is SKIPPED (cutout — continue the ray);
  alpha ≥ 0.5 → opaque as today. Document (no partial transparency in the
  tracer v1).
- Shadow maps + AO: alphaBlend objects DON'T cast (skip in shadow pass and
  SDF/GTAO occluder sets — a floating cutout casting a full quad shadow looks
  broken; document).
- sceneJson: version bump, tolerant load.

## C. "Always Textured" (Ray's default-on toggle; naming decided: **Always Textured**)
Image planes and HTML planes should look like themselves in EVERY shading
mode, not just Rendered.
- Material flag `alwaysTextured: boolean` — set TRUE by default when
  createImagePlane / html planes create their material; false for everything
  else. Properties: a checkbox "Always Textured" in the material tab's
  image-texture section (undoable, same commit pattern).
- Renderer: in matcap / studio / wireframe modes, objects whose material has
  alwaysTextured draw with the SHADELESS textured path (the UR4-3 shadeless
  branch) instead of matcap/studio shading — combined with part B's blending
  when alphaBlend. In wireframe mode they draw their textured fill FIRST,
  then wires as usual on top. Rendered mode: unchanged (they already show).
- The toggle off → plane shades like any mesh (matcap grey etc.).
