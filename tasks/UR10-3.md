# UR10-3 — Glass + Metal shaders (stretch batch)

Builds after UR10-2 (in your tree). Materials get transmission; the tab gets
one-click presets.

## Model
- Material fields: `transmission: number (0..1, default 0)` and
  `ior: number (default 1.45, clamp 1.0–2.5)`. Serialized (this batch's
  version), tolerant load. `metallic` already exists.
- Material tab: Transmission + IOR rows (IOR enabled when transmission > 0),
  plus two preset buttons **Glass** (transmission 1, ior 1.45, roughness 0,
  metallic 0, baseColor near-white) and **Metal** (metallic 1, roughness 0.15,
  transmission 0) — presets are ONE undo entry each ("Apply Glass Preset").

## Tracer (the real thing)
- transmission > 0 → dielectric BSDF: at each hit choose reflect/refract by
  Fresnel (Schlick with real ior; total internal reflection handled), refract
  via Snell across the interface (track inside/outside by normal flip), tint
  transmitted rays by baseColor (Beer-lite: multiply per surface pass-through,
  no distance attenuation v1 — document). Roughness > 0 on glass: perturb the
  refracted/reflected direction GGX-ish (reuse the existing spec-lobe sampling
  if present) — cheap rough glass, document the approximation.
- Metal: verify the existing Cook-Torrance metallic path gives TINTED mirror
  reflections at low roughness in the tracer (F0 = baseColor); fix if the
  metallic lobe isn't actually tracing reflection rays at roughness→0. The
  preset must look like chrome/gold, not grey plastic.
- Emissive/mesh-light interplay (UR10-2): glass must pass NEE shadow rays as
  TRANSPARENT-ish? NO — v1: glass blocks shadow rays like opaque (documented,
  standard cheap choice); revisit with caustics later. Note it in the header.

## Rendered viewport (approximation — document)
- transmission > 0 draws via the UR8-3 alphaBlend path with
  alpha ≈ 1 − 0.85·transmission and a Fresnel-boosted rim highlight (view-angle
  term in the shader) — reads as glass, no refraction. Metal: existing GGX is
  fine (env-less; matcap mode unaffected).

## Acceptance criteria
- tsc clean; npm test green (fresnel/snell unit tests incl. TIR; serializer;
  preset undo).
- e2e: (1) tracer: glass sphere in front of a red wall on a checker floor —
  probe THROUGH the sphere shows wall/floor colors (not the sphere's own),
  and a probe at the rim shows reflection (brighter than transmitted center);
  TIR case doesn't NaN (probe for black/NaN pixels = 0); (2) metal preset
  sphere under a colored light reflects with baseColor tint (gold-ish probe
  vs white plastic baseline); (3) viewport: transmission sphere lets the
  grid/cube behind show through (UR8-3 blend path, pixel probe); (4) presets:
  one click sets the field cluster, one undo reverts; (5) donut + shading
  suites stay green.
- Screenshots: glass sphere + gold sphere tracer render for eyes-on (this is
  a video-money shot — make it look good: checker floor, area light).
- Free-text report (STATUS:/VERDICT: first line). If ambiguous, STOP and
  report. E2E_GPU=1 spot check for the viewport blend.
