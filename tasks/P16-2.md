### Changes
- `src/core/nodes/nodeGraph.ts` — extended `EvalContext` with optional
  `gen?: [number, number, number]` (the only change to this file).
- `src/core/nodes/nodesC.ts` (NEW) + import line in `builtins.ts`:
  - `texCoord` — no inputs; outputs `generated` = `ctx.gen ?? [ctx.u, ctx.v, 0]`
    and `uv` = `[ctx.u, ctx.v, 0]`.
  - `mapRange` — input `value` (0.5), params `fromMin/fromMax/toMin/toMax`
    (0/1/0/1) + `clamp` select (yes/no, default yes); linear remap, clamp honors
    inverted output ranges, degenerate `fromMin==fromMax` → `toMin`.
- `src/core/nodes/nodesA.ts` — noise gains a `contrast` float param (0..1,
  default 0). Blends the raw fbm toward its own smoothstep by `contrast`;
  `contrast 0` early-exits returning the value UNCHANGED (bit-identical to the
  pre-P16 output — nodesA.test.ts still 12/12).
- `src/renderEngine/snapshot.ts` — new `triGen` Float32Array (3 floats × 3
  corners per tri): each corner vert's LOCAL position normalized to the base
  evaluated-mesh AABB (0..1/axis), computed once per object; a flat axis maps to
  0.5. Parallel to `triUV`.
- `src/renderEngine/tracer.ts` — `TraceScene.triGen`; barycentric-interpolated
  per hit (same weights as UV) and passed as `ctx.gen` ONLY when the material
  uses nodes, so the non-node path stays byte-identical (tracer.test.ts 19/19,
  empty diff).

### Honest limitation (also documented in nodesC.ts)
The Rendered/bake path (`bake.ts`) has no surface positions — it bakes the graph
over a flat UV grid — so `ctx.gen` stays undefined there and `texCoord.generated`
falls back to `(u, v, 0)`, i.e. reads as plain UV. Generated coords are correct
only in F12. `bake.ts` was intentionally left untouched.
