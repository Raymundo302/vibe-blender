## Result
Built.

New `src/core/anim/keyEditCommands.ts` — `MoveKeysCommand` (+ `.test.ts`, 4
tests). A diamond = a frame on an object, so a move shifts EVERY fcurve keyed
at that frame together. Two-pass: lift all sources off `fromFrame` first, then
drop them at `toFrame` capturing any collided key (insertKey's replace return)
so undo restores both the moved key at its origin AND the clobbered key.
Cross-object moves are just entries on different objects. `perform` returns null
for no-ops.

`src/ui/timeline.ts` (extends P15-1, does NOT rewrite it — P15-1's debug handle
+ pane behaviour untouched):
- Exported module-level `autoKeyState = { enabled }` (NOT on Scene, not saved).
- Exported `keyOf(objectId, frame)` selection identity.
- Select: pointerdown hit-tests diamonds (before the P15-1 scrub path — ruler /
  empty track still scrub). Click selects (shift extends/toggles); selected
  diamonds render enlarged + white. Empty click clears + scrubs.
- Move: drag selected diamonds → snap to integer frames, live preview (draw uses
  `frame + dragDelta`); release pushes ONE `MoveKeysCommand`, re-poses via
  `applyAnimation`, and the selection follows the keys to their new frames.
- Delete: X / Delete (window capture-phase listener + `stopPropagation`, gated on
  hover + non-empty selection + no focused form field, so it wins over
  InputManager's object-delete and only fires over the pane) → `DeleteKeysCommand`
  over every channel keyed at each selected frame. Also a `🔑 −` header button.
- Auto-key: `pollAutoKey()` in `update()` watches `UndoStack.pushCount`; on a
  NEW push whose top is a `TransformCommand` (confirmed G/R/S, Object Mode) while
  auto-key is on, inserts LocRotScale keys for the selected objects at
  frameCurrent as its own `InsertKeysCommand`. pushCount (not peek identity)
  avoids re-firing when undo reveals an older TransformCommand. No hooks in the
  operators or InputManager. The undo stack is read via `window.__app.undo` (the
  pane is constructed as `{ scene }` and main.ts is not in my file set).
- Debug handle extended: `selectedKeys()`, `diamondXY(id, frame)`, `autoKey`.

`src/core/undo/UndoStack.ts` — the one authorized core edit: read-only
`get pushCount()` (increments only on push) + `peek()`.

`src/ui/topbar.ts` — ONLY added the ⏺ auto-key toggle button (data-action
`autokey`, glows red via inline color when on), imported `autoKeyState`.
