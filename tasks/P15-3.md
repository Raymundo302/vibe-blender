## Features
- **Select keys**: click a diamond selects it (shift extends); selected
  diamonds render highlighted. Selection is per-pane runtime state.
- **Move keys**: drag selected diamonds horizontally → snap to integer
  frames, live preview; on release push ONE MoveKeysCommand (per object+
  channel: old frame → new frame, colliding same-frame keys at the target are
  REPLACED and restored on undo — build on insertKey's replace return).
  Moving re-poses the scene (applyAnimation at frameCurrent) after commit.
- **Delete keys**: X or Delete with selected diamonds → DeleteKeysCommand
  (all channels keyed at that object+frame — a diamond represents the frame,
  not one channel).
- **Auto-key** (topbar ⏺, runtime flag — module-level state file of yours or
  a field you add to the timeline module, NOT Scene): when on, confirming a
  G/R/S transform in OBJECT mode inserts LocRotScale keys at frameCurrent for
  the moved objects. Wire it WITHOUT touching the operators or InputManager:
  poll in the timeline pane's update() — keep a WeakMap of last-seen
  transforms? NO — simpler + robust: subscribe to the undo stack: expose is

  detect that the top of the undo stack is a NEW TransformCommand since last
