like designed objects. Fields: levels (int 1..3, default 1).

## Algorithm (one Catmull-Clark iteration; apply `levels` times)
For the input mesh:
1. **Face points**: centroid of each face.
2. **Edge points**: for an INTERIOR edge (2 faces): average of the edge's two
   endpoints and the two adjacent face points. For a BOUNDARY edge (1 face):
   midpoint of the endpoints.
3. **Vert points** (new position of each original vert v):
   - Interior (all adjacent edges have 2 faces): `(F + 2R + (n-3)P) / n` where
     F = average of adjacent face points, R = average of adjacent edge
     MIDPOINTS (not edge points), P = original position, n = number of
     adjacent faces.
   - Boundary: `(sum of midpoints of adjacent boundary edges + P·... )` — use
     the standard rule: `(m1 + m2 + 6P) / 8` with m1, m2 the midpoints of the
     two adjacent boundary edges. A vert on >2 boundary edges (non-manifold
     corner) keeps P unchanged.
4. **Faces**: each n-gon face with verts v0..vn-1 becomes n quads:
   `[vertPoint(vi), edgePoint(vi, vi+1), facePoint(f), edgePoint(vi-1, vi)]`
   — this winding preserves the original face orientation.
Build the output as a fresh EditableMesh; deterministic ordering (iterate
faces/edges/verts in Map insertion order).

## Acceptance criteria
1. Unit (cube, level 1): exactly 26 verts (8 vert-points + 12 edge-points +
   6 face-points), 24 faces, 48 edges; Euler V−E+F = 2; every edge manifold;
   all faces quads; result shrinks toward a sphere: every vert's |co| strictly
   less than √3 and greater than 0.9; face normals outward. Level 2 = 98 verts /
   96 faces; pure (input untouched); levels clamped to 1..3.
2. Unit (boundary): a single standalone quad (plane), level 1 → 9 verts /
   4 faces, corner verts stay INSIDE the original square (boundary rule pulls
   them in), edge-point verts at edge midpoints.
3. e2e: add Subdivision to the cube via the UI, evaluated verts 26 with base 8,
   levels field set to 2 → evaluated 98; performance guard: level 3 on the cube
   evaluates in < 200ms (time it via __app evaluate).
4. All four e2e suites + npm run build + npm test pass.

## Result
Built `src/core/modifiers/subsurf.ts` — a real Catmull-Clark subdivision modifier
(`SubsurfModifier`, registry key `subsurf`, "Subdivision Surface"; single int
field `levels` 1..3, default 1). `subdivideOnce()` computes face points, edge
points (face-aware interior / midpoint boundary), and vert points (interior
`(F+2R+(n-3)P)/n`, boundary crease `(m1+m2+6P)/8`, non-manifold corner keeps P),
then emits n quads per n-gon in deterministic insertion order (vert-points →
edge-points → face-points). Pure: derives fresh meshes, never mutates input.
Registered in `builtins.ts`. Added `subsurf.test.ts` (12 tests: cube L1 26v/24f/48e,
Euler=2, manifold, all-quads, 0.9<|co|<√3, outward normals, L2 98v/96f, purity,
clamp, determinism; plane boundary 9v/4f, corners pulled inside, edge midpoints,
boundary ring). Appended P4-6 e2e checks to `e2e/ship.mjs` (add via UI → 26 eval /
8 base, levels→2 → 98, level 3 <200ms; measured 1.8ms). Verification: `npm run build`
clean, `npm test` 160/160, all four e2e suites green. No deviations.
