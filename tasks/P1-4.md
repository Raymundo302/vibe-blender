# P1-4 — Outliner panel (list, select, rename, delete, visibility)

Read `tasks/P1-CONVENTIONS.md` first. The panel system already exists:
`src/ui/shell.ts` (`UiShell.addPanel`, per-frame `update()`), mounted in
`src/main.ts`, sidebar `<aside id="sidebar">` in index.html.

## Files
- `src/ui/outliner.ts` — new: `OutlinerPanel implements Panel` (id `outliner`, title `Outliner`)
- `src/main.ts` — wire it: `shell.addPanel(new OutlinerPanel(scene, undo))`
- CSS: inject a `<style>` block once from outliner.ts with an `outliner-` class prefix (P1-7 will do the full theme; keep yours functional and dark-friendly)

## Behavior
- One row per `scene.objects` entry, in list order: a mesh glyph (▲ or similar),
  the name, an eye visibility toggle, and a small ✕ delete button.
- Row click → `scene.selectOnly(id)`; Shift-click → `scene.toggleSelect(id)`.
- Selected rows highlighted; the active object's row visually distinct
  (Blender: brighter text).
- Eye toggle → flip `obj.visible` (no undo — matches our scope).
- ✕ → `undo.push(DeleteObjectsCommand.perform('Delete', scene, [id]))`.
- Double-click the name → inline `<input>`; Enter/blur commits: set
  `obj.name`, push `new RenameObjectCommand(obj, before, after)` (skip push if
  unchanged); Escape reverts. See `src/core/undo/objectCommands.ts`.
- `update()` (called every frame): recompute a cheap signature
  (id/name/visible per object + selection ids + activeId joined into a string);
  rebuild the row DOM only when it changed, and NEVER while the rename input
  is focused.

## Acceptance criteria (headless CDP)
1. Panel appears in the sidebar listing `Cube`; adding objects via `__app.scene.add(...)` from the console makes rows appear within a frame or two.
2. Clicking a row selects the object in the 3D view (selection Set updates, outline appears); Shift-click multi-selects.
3. Rename via double-click updates the row and `obj.name`; Ctrl+Z reverts the name.
4. ✕ removes the object from scene and list; Ctrl+Z restores it at the same index.
5. Eye toggle hides the object in the viewport (`visible=false` objects skip rendering — already implemented in Renderer).
6. `npm run build`, `npm test`, and `node e2e/smoke.mjs` (full 12 checks) pass.
