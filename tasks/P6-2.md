# P6-2 — N-panel (viewport overlay sidebar)

Read `tasks/P4-CONVENTIONS.md` first (chains to P1-P3).

## Goal
Blender's N-panel: pressing **N** toggles a slim overlay panel on the viewport's
right edge (inside #viewport-wrap, NOT a workspace area) showing the active
object's Item info: name (read-only), Location/Rotation/Scale (editable, same
undo semantics as the properties Object tab — share code by importing, don't
duplicate), and Dimensions (read-only: evaluated-mesh world-space bbox size).
In edit mode: median of selected element verts (read-only) + counts line
"Verts 8 · Edges 12 · Faces 6" of the edit mesh.

## Files
- NEW `src/ui/nPanel.ts`; `src/input/InputManager.ts` (N key, both modes, not
  while a modal op runs); `src/main.ts` wiring; `src/ui/theme.css`;
  `e2e/ship.mjs` appended. If sharing the transform-fields code requires
  exporting a helper from `src/ui/propertiesEditor.ts`, that ONE export is
  allowed — no other changes there.

## Acceptance criteria
1. e2e: N toggles the panel (`.n-panel`); location edit via the panel moves the
   object and undoes; dimensions update after a scale (cube scaled 2x → dims 4);
   edit-mode shows counts; N while G is modal does NOT toggle (key routes to op).
2. All four e2e suites + npm run build + npm test pass.
