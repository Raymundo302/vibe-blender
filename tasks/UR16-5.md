# UR16-5 — Smooth shading in the path tracers (CPU + GPU)

## Ray's report (2026-07-12, both systems)
Shade Smooth works in matcap/studio/Rendered-live but the CPU path tracer and
the GPU kernel render FLAT. Root cause (verify, then fix): the tracers shade
with the triangle's geometric normal; per-corner smooth normals never enter
the snapshot/pack path.

## Fix
- Snapshot (`snapshot.ts`): for objects whose shadeSmooth flag is set (grep
  the raster path for where smooth normals are computed — meshToGpu already
  builds them; REUSE that computation or its helper, don't fork the
  area-weighting rules), carry per-corner shading normals per triangle.
  Flat objects carry nothing extra (size guard).
- CPU tracer: at a hit, barycentric-interpolate the corner normals →
  normalize → use as the SHADING normal (BRDF, NEE cosines, bounce
  hemisphere). Keep the GEOMETRIC normal for: ray offset epsilon, backface
  determination for glass enter/exit, and clamp the shading normal to the
  geometric hemisphere (dot(Ns, Ng) flip guard — the classic shadow-terminator
  black-splotch source; document the choice).
- GPU pack/kernel: add a corner-normals payload (or extend triangle texels);
  same interpolation + same geometric/shading split, highp.
- Rendered-live and all raster modes: unchanged (already correct).
