# UR5-6 — N-panel tabs: Item + View (lens, passepartout, camera fields)

## Goal
The N panel (`src/ui/nPanel.ts`) is a single "Item" panel. Give it Blender-style
vertical tabs on its right edge — **Item** (existing content, unchanged) and
**View** (new).

## View tab contents
1. **Focal Length** (viewport lens): numeric field in mm editing the orbit
   camera's `fovY` (`src/camera/OrbitCamera.ts`, default 50° today). Use the SAME
   mm↔fov convention the through-camera projection uses for `CameraData.
   focalLength` (36×24 sensor — find that conversion in the renderer and share
   one exported helper; do not invent a second formula). Live update, no undo
   entry (viewport state, not scene data — matches Blender).
2. **Passepartout** checkbox: new `viewPrefs` module (pattern-match
   `shadePrefs.ts`: localStorage, defaults, clamped load) with
   `passepartout: boolean` default true; `src/ui/passepartout.ts` update() shows
   the overlay only when (in camera view && pref on).
3. **In-camera-view section**: when the viewport is looking through a camera
   (`renderer.cameraViewId` set), show that camera's:
   - Focal Length (mm) — edits `CameraData.focalLength`, SAME undo command the
     Camera tab uses (grep cameraTab.ts; share its commit path, no duplicate).
   - Location X/Y/Z — the camera object's position via the existing
     `TransformFields` component (nPanel already instantiates one for Item; a
     second instance bound to the camera object is fine).
   Hidden when not in camera view.

## Files
nPanel.ts (+ its css/theme entries), NEW src/render/viewPrefs.ts (+ test),
passepartout.ts (one-line gate), OrbitCamera.ts ONLY if fovY needs a setter.
Do not touch cameraTab.ts beyond importing/reusing its command, and do not
restructure the Item content.

- tsc clean; npm test green (viewPrefs load/clamp; mm↔fov round trip through the
  shared helper).

  camera view, on → present (DOM probe); pref survives reload; (4) camera-view
  section appears only through a camera; editing its focal length changes
  CameraData and is undoable; (5) Item tab regression: transform fields still
