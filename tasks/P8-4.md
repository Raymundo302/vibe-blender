# P8-4 — Render engine: progressive path tracer (F12)

Read `tasks/P8-CONVENTIONS.md` first.

## Goal
"Cycles-lite": F12 opens a render-result window and progressively path-traces
the scene from the ACTIVE CAMERA (scene.activeCamera; if none, render from the
current viewport OrbitCamera view). Pure TypeScript, runs in a Web Worker so
the app stays interactive. This is the video's flagship feature — quality of
the core math matters more than feature count.

## Files (yours alone — you own src/renderEngine/)
- `src/renderEngine/init.ts` — fill the stub (main.ts already calls
  initRenderEngine with scene/camera/setStatus/host). Install a window keydown
  listener: F12 toggles the window open+render; while open, Esc closes it
  (listener must not fight InputManager: stopPropagation only while open).
- NEW `src/renderEngine/snapshot.ts` — plain-data scene snapshot: world-space
  triangles (apply evaluatedMesh() + transform; skip non-mesh/hidden objects)
  with per-triangle material index, materials array, lights array (reuse the
  LightData semantics; power scaling like collectLights), camera (position,
  orientation basis from objectForward/transform, fovY via cameraFovY, else
  OrbitCamera fovY/eye/target). Everything structured-cloneable.
- NEW `src/renderEngine/tracer.ts` — the pure tracer: ray-triangle (Möller–
  Trumbore), a simple BVH (median split on centroid, max-leaf ~4 tris) —
  brute force fallback is NOT acceptable above 5k tris —, cosine-weighted
  diffuse bounce + GGX-ish glossy reflection lerped by metallic/roughness
  (keep it simple: reflect + roughness-scaled jitter is fine, document it),
  direct light sampling each bounce (point/sun/spot with the same falloff
  as renderedPass, shadow rays), emissive materials glow, max depth 4,
  Russian roulette after depth 2, sky = simple vertical gradient
  (~0.05 dark grey ground → 0.15 blue-grey up) so unlit scenes aren't void.
  MUST be importable in Node (no DOM/GL types) for unit tests.
- NEW `src/renderEngine/worker.ts` — Web Worker: receives snapshot + size,
  loops sample passes, posts Float32Array accumulation tiles or full frames
  (your choice) + sample count. Use `new Worker(new URL('./worker.ts',
  import.meta.url), { type: 'module' })` — Vite handles bundling.
- NEW `src/renderEngine/renderWindow.ts` — DOM overlay window (theme.css
  style, class prefix `render-win-`): canvas (default 960×540), sample/time
  readout, Save PNG button (toDataURL), close ×. Tonemap+gamma when blitting
  (Reinhard + 1/2.2, same as renderedPass).
- `src/ui/theme.css` — additive only, `render-win-` prefix.
- NEW unit test file(s) for tracer math.
- NEW `e2e/render.mjs` — copy the harness pattern from ship.mjs.

## Determinism
Seeded RNG (e.g. mulberry32) — same snapshot + seed → identical first-pass
image. Tests rely on this; no Math.random in the tracer.

## Acceptance criteria
1. Unit: Möller–Trumbore hit/miss/backface; BVH returns identical nearest-hit
   results to brute force on a ~100-tri random soup (seeded); direct light:
   a point light 2m above a floor quad lights the point under it with
   intensity ∝ power/4π/d² and NdotL; shadow ray blocked by an occluder tri
   → black; sun direction honored.
2. e2e (render.mjs): scene = default cube + a point light above-right (via
   __app), F12 → window appears, sample counter increments, and after ≥4
   samples the canvas center row has: lit-face pixels notably brighter than
   the background gradient, AND a darker shadowed region under… (keep it
   simple: assert max pixel luminance in the cube area > 3× the sky-gradient
   luminance at the same rows). Esc closes the window and terminates the
   worker (assert no further sample-count changes).
3. F12 with an active camera renders from that camera: put the camera at a
   known pose looking away from the cube → cube pixels absent (flat gradient
   only); looking at the cube → present.
4. All e2e suites + npm test + npm run build pass.
