# UR7-2 — HTML planes: browse mode (Tab = scroll the page), page-extent property, Properties section

Builds on UR7-1 (in your tree: `html` payload, rasterizeHtmlAt, page clock).

## A. Browse mode — "treat it like a website"
- Pressing **Tab with an html plane active** enters **Page mode** instead of
  mesh edit mode (InputManager's Tab branch: if activeObject?.html, toggle
  page mode; plain meshes keep edit mode exactly as-is). Status chip
  "Page Mode — scroll to browse, Tab/Esc to exit" (grep how Edit Mode sets its
  topbar/status indicator and do the closest equivalent).
- In page mode, **wheel over the viewport scrolls the page**: adjust
  `html.scrollY` (± wheel delta, clamp ≥ 0; no content-height clamp v1 —
  document), re-raster (throttled, latest-wins, reuse UR7-1's in-flight guard).
  Camera zoom is suppressed while in page mode (wheel belongs to the page).
  Everything else (orbit MMB, pan) stays live.
- `scrollY` consumption in the rasterizer: wrap the sanitized body in a fixed-
  size clipping viewport div scrolled by scrollY (transform: translateY(-scrollY)
  inside an overflow:hidden 100%-height wrapper — pick the approach that
  renders correctly with foreignObject and document it).
- Esc or Tab exits page mode. Entering page mode does NOT create an undo entry;
  scrollY changes are NOT undoable (viewport-ish state, but it serializes with
  the scene — Blender-precedent: like a pane's view, saved but not undo-tracked).

## B. Page extent — "how much of the page you want"
- `html.pageH` (and pageW) become editable; the PLANE GEOMETRY follows:
  width stays as-built, height = width · pageH/pageW, anchored so the TOP edge
  stays put and the BOTTOM edge extends downward (Ray: "making the plane's
  bottom edge go further down to show more of the page"). Implement as a
  regenerate-the-plane-mesh helper (verts + UVs; UVs stay 0..1 full-raster).
  One undo entry per committed field edit (grep the properties-tab numeric
  commit pattern).

## C. Properties → "Web Page" section
In the Properties editor's Object tab (propertiesEditor.ts), a section shown
only when the active object has `html`:
- Fields: Page Width / Page Height (px, drive B), Scroll Y (readonly display
  is fine), FPS (1–15), and the source (file: name/first chars readonly;
  url: the address readonly — UR7-3 makes it editable).
- **Play/Pause toggle button** (▶/⏸) driving `html.playing` + a **● key
  button** inserting a key on the `html.playing` channel at the current frame
  (exact pattern: lightTab.ts ~line 225 keyBtn — reuse, don't copy).
- A "Re-rasterize" button (forces a fresh raster — useful after font loads).
