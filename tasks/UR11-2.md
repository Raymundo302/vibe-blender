# UR11-2 — Pipe modifier (sweep a tube along a curve)

Builds on UR11-1 (in your tree). Ray's name for it: **Pipe**.

## The modifier
- New modifier type `'pipe'` (src/core/modifiers/pipe.ts, registered like
  displace/solidify): only meaningful on CURVE objects — it consumes the
  object's evaluated curve polyline and OUTPUTS a tube mesh (the modifier
  system already turns payload → evaluated mesh; check how curve objects with
  an empty base mesh flow through ModifierContext and make the pipe the thing
  that materializes geometry).
- Params (schema fields, all animatable if the modifier system supports
  keying — match whatever displace supports):
  - `radius` (default 0.1, > 0)
  - `sides` (profile segments, default 12, 3–64)
  - `capEnds` (bool, default true; ignored when the curve is cyclic)
  - `radiusEnd?` (optional taper: lerp radius→radiusEnd along the length,
    default = radius)
- Geometry: sweep a circle of `sides` verts along the UR11-1
  parallel-transport frames; ring per evaluated point; quads between rings;
  cyclic curve → welded seam ring (no caps); caps = triangle fans. UVs:
  u around the profile, v = arclength/length (tube-conventional).
- Modifier tab: appears in the modifier list for curve objects (guard: adding
  Pipe to a mesh object shows a disabled row note "Curve objects only" or is
  filtered from the add menu — pick the cleaner UX and document).
- Editing the curve (UR11-1 edit mode) live-updates the pipe (the modifier
  cache keys on the curve payload version — grep how mesh modifiers key on
  mesh.version and add a curve version counter bumped by every curve edit).
- Serialization inside the existing modifier stack format. Apply-modifier
  (existing command) bakes the tube to a real mesh object — verify it works
  on a curve object (the result should become kind 'mesh'; follow
  convert-to-mesh precedent from UR8-2 if the generic apply path can't
  change kinds; document what you did).

## Acceptance criteria
- tsc clean; npm test green (ring/quads counts for open+cyclic, taper radii,
  cap fans, UV ranges, weld on cyclic).
- e2e (extend e2e/curves.mjs): (1) bezier + Pipe → tube visible in matcap
  (pixel probe where the bare polyline was thin); (2) move a control point in
  edit mode → tube follows live; (3) radius field change → silhouette widens
  (probe); (4) cyclic circle + pipe = donut-like ring, no cap artifacts;
  (5) Apply → mesh object editable in mesh edit mode, one undo restores the
  curve+modifier; (6) tracer F12 renders the tube (probe); (7) donut +
  modifier suites stay green.
- Screenshot: tapered pipe along an S-curve, matcap, for eyes-on. Free-text
  report. If the modifier system fights curve objects structurally, STOP and
  report blocked with the architectural issue — do not hack the core.
