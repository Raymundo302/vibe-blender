# UR12-2 — GPU path tracer, stage 2: full feature port

Builds on UR12-1 (in your tree: packing, kernel v0, accumulation). Port the
CPU tracer's feature set into the GLSL kernel. THE CPU TRACER IS THE SPEC —
read src/renderEngine/tracer.ts and mirror behavior; where exact match is
impractical (RNG sequences), match DISTRIBUTIONS and document.

## Features to port (each with its own commit-sized kernel addition)
1. Multi-bounce (russian roulette after N bounces per CPU policy) + sky/world
   env (flat/gradient/HDRI — the CPU path samples the world; HDRI needs an
   env texture upload in pack.ts).
2. Direct lighting / NEE with shadow rays: point/sun/spot/**area** (uniform
   rect sampling, one-sided, same pdf math as the CPU port of UR10-1) and
   **emissive mesh lights** (UR10-2's area-weighted emitter CDF — pack the
   CDF as a texture; same double-count bookkeeping).
3. Materials: Cook-Torrance GGX (rough/metal), per-face tints, image/checker/
   noise textures via UV (pack normals+uvs now; texture atlas: one
   TEXTURE_2D_ARRAY layer per material image, capped + documented),
   normal/rough/metal maps if the CPU does them (it does — P13), SSS if
   feasible or documented-out, **shadeless**, **alphaBlend cutout** (alpha <
   0.5 pass-through with bounded continuation like the CPU), **glass**
   (Fresnel/Snell/TIR, tinted transmission, GGX-perturbed rough glass —
   UR10-3 math).
4. Thin-lens DoF (fStop/focusDistance incl. focus-object distance — the
   camera struct already resolves it CPU-side; reuse the resolved values).
5. Soft shadows from light radius (P9-4 parity).

## Constraints
- Deterministic per (pixel, spp, seed) — re-render must be bit-identical on
  the same GPU (accumulation order fixed).
- Kernel compiles under highp with NO dynamically-indexed arrays beyond
  texelFetch (GLSL ES 3.0 rules); loop bounds constant with uniform-driven
  early exit.
- Precision discipline: highp everywhere (the AO lowp lesson); watch fp32
  self-intersection — epsilon-offset hit points like the CPU does.
- Any feature you must cut: document in the module header AND the report —
  the stage-4 parity harness will hold you to the documented list.

## Acceptance criteria
- tsc clean; units green (pack additions). Extend e2e/gpu-tracer-1.mjs →
  `gpu-tracer-2.mjs`: for EACH ported feature a scene + probe pair vs the CPU
  engine at 64 spp / 128²: mean-abs-diff of downsampled 32² luminance < 0.06
  (loose parity — SSIM comes in stage 4): cube+point, cube+area (penumbra),
  emissive panel (indirect light), glass sphere (transmission probe), DoF
  two-depth blur ordering, textured donut fixture (UV correctness).
- Report per-feature diff numbers + 64-spp timing at 512² on the Vega
  (E2E_GPU=1) vs the CPU tracer (the headline speedup number).
- Screenshots: GPU vs CPU side-by-side for glass + area scenes; LOOK at them.
- Free-text report, first line STATUS: done/blocked.
