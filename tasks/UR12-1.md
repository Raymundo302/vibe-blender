# UR12-1 — GPU path tracer, stage 1: scene→texture packing + minimal GLSL kernel

## Context
The F12 path tracer is CPU (Web Worker, src/renderEngine/tracer.ts + worker.ts,
BVH, seeded RNG). UR12 builds a **WebGL2 fragment-shader** path tracer — one
universal backend for any GPU (Vega 7 today, anything tomorrow). Stage 1 =
the data plumbing + a walking skeleton that provably traces.

## Deliverables
1. `src/renderEngine/gpu/pack.ts` (+tests): serialize the EXISTING scene
   snapshot (grep snapshot.ts — reuse its output; do NOT build a second scene
   walker) into Float32 texture payloads:
   - BVH nodes (reuse the CPU BVH build — pack nodes as texels; if the CPU
     BVH layout is pointer-based, flatten to indexed),
   - triangles (positions; normals/uvs in stage 2),
   - materials (baseColor/rough/metal/emissive/transmission/ior — pack now,
     consume later),
   - lights (all 4 types incl. area axes).
   Each as { data: Float32Array, width, height, texelsPerItem } with
   documented layouts. Pure + unit-tested (layout round-trip readers).
2. `src/renderEngine/gpu/gpuTracer.ts`: WebGL2 context on an offscreen canvas,
   RGBA32F float targets (require EXT_color_buffer_float; report cleanly if
   absent), fullscreen-quad pass, textures uploaded from pack.ts.
3. GLSL kernel v0 (`src/renderEngine/gpu/kernel.ts` as template strings):
   camera ray gen (same thin-lens params as CPU, aperture 0 for now) → BVH
   traversal (stack-based, fixed max depth 64) → closest-hit → shade =
   `albedo · (N·L clamp)` for ONE hardcoded direct light sample, no shadows,
   no bounces. This stage's goal is TRAVERSAL CORRECTNESS, not beauty.
4. Accumulation: ping-pong float buffers averaging progressive samples with a
   per-pixel PCG hash seeded by (pixel, sampleIndex, frameSeed) — same seed
   philosophy as the CPU tracer (grep its RNG; document divergence).
5. Debug harness `e2e/gpu-tracer-1.mjs`: renders the default cube scene via
   the GPU kernel headless (E2E_GPU=1 required for meaningful perf, but
   SwiftShader must WORK) and asserts: non-black, cube silhouette pixels
   where the CPU tracer has them (loose mask agreement ≥ 90% on a 64×64
   binary hit-mask — an exported `renderHitMask()` on both engines; add the
   CPU one as a tiny debug export), no NaNs.

## Out of scope (stages 2–4)
Bounces, shadows, NEE, glass, DoF, textures/UVs, render-window UI, parity
SSIM. NOTHING outside src/renderEngine/gpu/ + the debug e2e + minimal
exports from tracer.ts/snapshot.ts (mark them clearly).

## Acceptance criteria
- tsc clean; npm test green (pack layout tests). e2e/gpu-tracer-1.mjs passes
  on SwiftShader AND E2E_GPU=1; report the hit-mask agreement % and a timing
  (ms for 16 spp at 256²) on both backends.
- Screenshot the GPU v0 render next to the CPU render for eyes-on.
- Free-text report, first line STATUS: done/blocked.
