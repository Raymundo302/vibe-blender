# P7-5 — Duplicate in edit mode (Shift+D)

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Edit mode, face select, ≥1 face selected: **Shift+D** duplicates the selected
faces INSIDE the same mesh (new verts + faces, seam-free copy — all verts
duplicated, even shared ones), selects the copies, and immediately rides a
Move (reuse EditTranslateOperator behavior — start it programmatically like
object-mode Shift+D does). Cancel (Esc/RMB) removes the duplicated geometry
entirely (Blender leaves it in place on cancel, but our v1 removes it — cleaner
without a "click again" convention; document this deviation in the overlay
tooltip text if any).

## Files
- NEW `src/core/mesh/ops/duplicateFaces.ts` (pure; returns new face/vert ids)
  + test
- `src/input/InputManager.ts` — Shift+D in edit mode (face mode; else status)
- `e2e/edit.mjs` — append checks

## Undo shape
The WHOLE gesture (topology + the move) is ONE command: modal-TOPOLOGY pattern
— clone before Shift+D mutates, `MeshEditCommand.fromSnapshots` on confirm;
cancel = copyFrom(before), push nothing.

## Acceptance criteria
1. Unit: duplicate 1 cube face → 12 verts / 7 faces, the new face disjoint from
   the original shell (no shared verts); 2 adjacent faces → copies share their
   seam verts WITH EACH OTHER (6 new verts, not 8) but not with the originals.
2. e2e: cube, face mode, click face, Shift+D, drag, confirm → 7 faces, ONE
   undo step restores 6; Esc path → still 6 faces, no stray verts.
3. All four e2e suites + npm run build + npm test pass.
