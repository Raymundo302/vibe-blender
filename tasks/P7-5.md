# P7-5 — Duplicate in edit mode (Shift+D)

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Edit mode, face select, ≥1 face selected: **Shift+D** duplicates the selected
faces INSIDE the same mesh (new verts + faces, seam-free copy — all verts
duplicated, even shared ones), selects the copies, and immediately rides a
Move (reuse EditTranslateOperator behavior — start it programmatically like
object-mode Shift+D does). Cancel (Esc/RMB) removes the duplicated geometry
entirely (Blender leaves it in place on cancel, but our v1 removes it — cleaner
without a "click again" convention; document this deviation in the overlay
tooltip text if any).

## Files
- NEW `src/core/mesh/ops/duplicateFaces.ts` (pure; returns new face/vert ids)
  + test
- `src/input/InputManager.ts` — Shift+D in edit mode (face mode; else status)
- `e2e/edit.mjs` — append checks

## Undo shape
The WHOLE gesture (topology + the move) is ONE command: modal-TOPOLOGY pattern
— clone before Shift+D mutates, `MeshEditCommand.fromSnapshots` on confirm;
cancel = copyFrom(before), push nothing.

## Acceptance criteria
1. Unit: duplicate 1 cube face → 12 verts / 7 faces, the new face disjoint from
   the original shell (no shared verts); 2 adjacent faces → copies share their
   seam verts WITH EACH OTHER (6 new verts, not 8) but not with the originals.
2. e2e: cube, face mode, click face, Shift+D, drag, confirm → 7 faces, ONE
   undo step restores 6; Esc path → still 6 faces, no stray verts.
3. All four e2e suites + npm run build + npm test pass.

## Result

Built edit-mode Shift+D (duplicate selected faces + ride a Move):

- **`src/core/mesh/ops/duplicateFaces.ts`** (new, pure) — `duplicateFaces(mesh, faceIds)`
  copies every vert used by the selected faces exactly ONCE (verts shared between
  two selected faces get a single shared copy, so the duplicated region is a
  seam-free island that shares no vert with the original shell), then rebuilds the
  faces in parent winding order. Returns `{ newVertIds, newFaceIds, vertMap }`.
  Deterministic (first-encounter vert order, input face order).
- **`src/core/mesh/ops/duplicateFaces.test.ts`** (new) — 3 tests: 1 cube face →
  12 verts/7 faces disjoint from the shell + winding preserved; 2 adjacent faces →
  6 new verts (not 8), the two copies share exactly their 2 seam-vert copies with
  each other but not the originals; non-face/empty ids are a no-op.
- **`src/input/InputManager.ts`** — added a `DuplicateFacesOperator` (subclass of
  `EditTranslateOperator`, so it reuses view-plane drag + X/Y/Z axis lock + grid
  snapping). Its `start()` snapshots the mesh, runs `duplicateFaces`, selects the
  copies, then calls `super.start()` to begin the move on the new verts. The whole
  gesture is ONE undo command (modal-TOPOLOGY pattern): `confirm()` pushes
  `MeshEditCommand.fromSnapshots('Duplicate', …)`; `cancel()` does
  `copyFrom(snapshot)` and pushes nothing (removing the copy entirely — v1
  deviation from Blender, documented in the status hint). A second G is swallowed
  so Shift+D→G stays a plain Move (never Edge Slide). Wired Shift+D in
  `onEditModeKey` (face mode only; else a "face mode only (v1)" status).
- **`e2e/edit.mjs`** — appended P7-5 checks: face select → Shift+D → drag → confirm
  gives 7 faces / 12 verts with the copy selected; ONE Ctrl+Z restores 6/8; the
  Esc path leaves 6 faces / 8 verts (no stray geometry); vert-mode Shift+D reports
  face-only and mutates nothing.

**Verification:** `npm test` 234 passing (30 files), `npm run build` clean, all four
e2e suites (smoke / edit / ship / workspace) green.

**Deviation from Files list:** the operator class lives in `InputManager.ts` (an
allowed file) rather than a new `src/tools/` file — the spec's Files list named only
the pure op + InputManager + e2e, and "reuse EditTranslateOperator behavior … start
it programmatically" is implemented by subclassing it there. No other files touched.
