# P2-4 — Extrude (E), face mode

Read `tasks/P2-CONVENTIONS.md` first.

## Goal
In edit mode with faces selected, E extrudes the selected face REGION and rides
the pointer along the average face normal until confirm/cancel (Blender's E).
In vert/edge mode, E just sets a status message "Extrude: face mode only (v1)".

## Files
- NEW `src/core/mesh/ops/extrude.ts` — pure topology function + unit tests
  (NEW `src/core/mesh/ops/extrude.test.ts`)
- NEW `src/tools/extrude.ts` — the modal operator
- `src/input/InputManager.ts` — wire E in `onEditModeKey`
- `e2e/edit.mjs` — append checks

## Topology algorithm (region extrude — follow exactly)
`extrudeFaces(mesh, faceIds: Set<number>): { newVertByOld: Map<number, number>, capFaceIds: number[] }`
1. Verts used by selected faces → duplicate each (`addVert` at same co); build
   `newVertByOld`.
2. Boundary edges = edges of selected faces where `edge.faces` contains exactly
   one SELECTED face. For each, with (a, b) ordered as traversed by that selected
   face, add side quad `[a, b, newVertByOld(b), newVertByOld(a)]` (this winding
   keeps the quad outward when the cap moves along the outward normal).
3. Remap each selected face's verts in place to the new ids (these become the caps).
4. Delete original verts that no longer appear in ANY face (`facesOfVert` empty).
5. Return the cap face ids (same ids as the input faces — they were remapped, not
   recreated).

## Modal operator
- Uses the "modal TOPOLOGY tools" undo pattern from the conventions doc.
- On E: run extrudeFaces, set selection to the cap faces (`sel.faces` = caps,
  `sel.touch()`), then translate the cap verts along the average of the cap face
  normals by a distance driven by pointer motion (project pointer delta onto the
  normal's screen direction — same plane-hit approach as `translate.ts`).
  Numeric input sets the distance exactly.
- Status: `Extrude  D: <dist>  LMB/Enter: confirm  RMB/Esc: cancel`.

## Acceptance criteria
1. Unit: extruding one cube face → 12 verts, 10 faces, 20 edges, every edge
   manifold (2 faces); extruding two ADJACENT faces as a region → their shared
   edge produces NO side quad (region, not per-face); cancel path restores the
   original counts via `copyFrom`.
2. e2e: edit mode → face mode → click top face (or select-all then face mode with
   a single face via click if P2-2 landed; otherwise drive selection through
   `window.__app` evaluate) → E, move pointer, click-confirm → vert count is 12;
   Ctrl+Z → 8.
3. `npm run build`, `npm test`, `node e2e/edit.mjs` all pass.

Returns `{ newVertByOld, capFaceIds }` (cap ids == input ids).
