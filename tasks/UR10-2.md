# UR10-2 — Emissive mesh lights (strength > 1), camera Glare, F-Stop DoF

Builds after UR10-1 (in your tree). Three camera/render features.

## A. Emission acts as a light source in renders
- Material `emissiveStrength` UI currently caps at/near 1 — allow up to 100
  (numeric field; slider optional). Grep materialTab for the field.
- **Tracer:** emissive triangles become sampled MESH LIGHTS (next-event
  estimation), not just bounce-found glow: build an emitter list (triangles of
  materials with emissiveStrength > 0, area-weighted CDF) at scene-snapshot
  time; at each shading point sample it alongside the analytic lights with
  proper pdf weighting (and avoid double counting when a bounce ray hits the
  emitter directly — MIS or the classic "skip emitter radiance on NEE-sampled
  bounces" bookkeeping; document which). A glowing plane must light the room
  and cast soft shadows.
- **Rendered viewport:** no GI — emissive surfaces just output
  `emissive·strength` HDR-ish (values >1 clamp in the LDR target; the glare in
  part B is what sells the glow). Document.

## B. Camera Glare (bloom)
- `CameraData.glare?: { enabled: boolean; threshold: number (1.0);
  strength: number (0.5); radius: number (0.05 of image height) }` —
  serialized (same version bump as this batch), Camera tab "Glare" section
  with the 4 controls (checkbox + 3 numerics), undoable commits.
- Applied wherever that camera renders:
  - **F12/tracer + Ctrl+F12**: post-process on the tracer output before
    display/capture — bright-pass (≥ threshold) → separable Gaussian at
    `radius` → add ×strength. Implement once, reusable (the render window and
    animRender share the seam; CPU on the float buffer in the worker OR a GL
    pass on the render-window canvas — implementer's choice, justify).
  - **Rendered viewport WHEN looking through that camera** (cameraViewId set):
    same post as a GL pass on the viewport (only through-camera view, not free
    navigation — it's a camera property; document).
- Determinism: pure function of the frame — no temporal accumulation.

## C. F-Stop drives DoF
- `CameraData.fStop` (default 2.8, clamp 0.5–22) replaces the raw aperture UI:
  lens radius = (focalLength/1000) / (2·fStop) in scene units (document the
  unit assumption; 50mm f/2.8 ≈ 0.0089 — verify it produces visible blur at
  scene scale and note the constant if a scale factor is needed to feel like
  Blender). Tracer thin-lens consumes the derived radius (it already has
  aperture/focusDistance — grep cam.aperture in tracer.ts).
- Camera tab: replace the Aperture numeric with **F-Stop** (smaller = blurrier
  hint in the tooltip); keep the DoF enable semantics (aperture 0 ⇔ disabled —
  a "DoF" checkbox + fStop field is cleanest). Focus Object/Distance (UR5-7)
  unchanged. MIGRATION: scenes storing raw `aperture` load as
  fStop = focal/(2000·aperture) clamped (document; aperture stays the derived
  runtime value so old saves render the same).

## Acceptance criteria
- tsc clean; npm test green (emitter CDF sampling unit test vs analytic
  half-space irradiance on a simple config; glare pure-function tests
  (bright-pass threshold, energy roughly conserved ±strength); fStop→radius
  mapping + migration).
- e2e (extend the render-engine suite): (1) dark room + emissive plane
  (strength 10): tracer F12 → cube face NOT facing the plane is lit indirectly
  vs the no-emitter baseline (probe), soft shadow behind the cube; (2) glare
  on: a small emissive dot bleeds ≥5px halo (probe ring pixels) vs glare off;
  Ctrl+F12 PNG frame carries the same halo; viewport through-camera shows it,
  free navigation does NOT; (3) fStop 0.5 vs 16 on the two-depth scene:
  low-fStop frame has measurably blurrier far object (reuse the UR5-7
  sharpness-metric prior art), fStop 16 ≈ near-pinhole; (4) old scene with raw
  aperture loads + renders (migration).
- Screenshots: glowing panel scene with glare, for eyes-on. Free-text report.
  If ambiguous, STOP and report.
