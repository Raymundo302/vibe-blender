## The editor (workspace type 'uv', title 'UV Editor', NOT singleton)
A 2D canvas panel (own <canvas>, DPR-aware, resizes with its area):
- Draws: [0,1]² frame, a subtle grid, a procedural checker background
  (canvas 2D is fine), then the ACTIVE object's evaluated-base mesh UVs —
  edges as lines, face fills faint, island hover/selection highlighted with
  the theme accent (read getComputedStyle --vb-accent; canvas can't use CSS
  vars directly).
- Which mesh: the active object's BASE mesh (same rule as edit mode). Empty
  state text when no active mesh object / no UVs ("Select a mesh with UVs —
  U to unwrap in the viewport").
- Island = connected faces sharing uv-space corners (within epsilon) — compute
  islands from mesh.uvs (pure helper + test).
- Interactions (listeners on the editor element, NOT InputManager):
  click = select island under cursor (point-in-uv-face test, pure + tested);
  shift-click adds; A selects all / Alt+A clears (when the editor has focus/
  hover); G/R/S start a modal transform of the selected islands (pointer moves,
  LMB/Enter confirm, Esc/RMB cancel) mirroring the viewport operators' feel;
  wheel zooms, MMB pans the UV view.
- Transforms mutate mesh.uvs (setFaceUVs), ONE MeshEditCommand per confirmed
  gesture; live redraw each frame via update() (the workspace already calls
  panel update()).
- 3D sync: after any island selection change, select the same faces in the 3D
  edit selection IF edit mode is active on that object (one-way sync, document).

4. All e2e suites (flock, donut excluded) + npm test + tsc clean.
