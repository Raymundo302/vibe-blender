# P2-2 — Element picking pass (click-select verts/edges/faces)

Read `tasks/P2-CONVENTIONS.md` first.

## Goal
Clicking in the viewport while in edit mode selects the vert/edge/face under the
cursor, respecting the current `elementMode`. Shift-click toggles. Clicking empty
space clears the element selection (without Shift).

## Files
- NEW `src/render/passes/elementPickPass.ts`
- `src/render/gl/Framebuffer.ts` — ADD one method only (signature below)
- `src/render/Renderer.ts` — add `pickElement(...)` + own an `ElementPickPass`
- `src/input/InputManager.ts` — replace the edit-mode LMB early-return with element picking
- `e2e/edit.mjs` — append checks

## Design (follow exactly)
- `ElementPickPass` renders the edit object's elements as ID colors into its own
  `Framebuffer` (RGBA8 + depth), sized to the canvas, resized like the other passes.
- Use `elementIndexMaps(mesh)` for index↔element mapping and `encodeId` from
  `src/render/passes/pickingPass.ts`. Bases per conventions doc; export
  `VERT_PICK_BASE`, `EDGE_PICK_BASE`, `FACE_PICK_BASE` from the new file.
- Element ID geometry is built per (mesh.version) and cached; colors are vertex
  attributes (reuse the wire-shader pattern from `editOverlayPass.ts` — position +
  color attribute, u_mvp, depth bias uniform).
- What renders depends on elementMode (Blender semantics — only the current kind
  is clickable, front elements win):
  - **face mode:** face triangles with face IDs.
  - **edge mode:** face triangles as occluders with id 0 (opaque black), then edge
    lines with edge IDs and depth bias 0.001.
  - **vert mode:** occluder faces (id 0), then vert points (gl_PointSize 10) with
    vert IDs and depth bias 0.0015.
- Click tolerance: add to `Framebuffer`:
  `readRegion(x: number, y: number, w: number, h: number): Uint8Array` (RGBA rows,
  GL bottom-up like readPixel). `pickElement` reads a 9×9 region centered on the
  cursor and takes the non-zero id closest to the center (Chebyshev distance).
- `Renderer.pickElement(scene, camera, cssX, cssY)` returns
  `{ kind: 'vert', id: number } | { kind: 'edge', key: string } | { kind: 'face', id: number } | null`
  (ids/keys are ELEMENT ids from the maps, not indices).
- InputManager edit-mode LMB: plain click → clear the current-mode Set, add the hit,
  `touch()`. Shift-click → toggle membership, `touch()`. Miss without Shift →
  `clearSelection()`. Do not touch object selection while in edit mode.

## Out of scope
Box select (P2-8), hover highlights, gizmo interaction in edit mode.

## Acceptance criteria (self-check before committing)
1. Unit test (new `src/render/passes/elementPick.test.ts` or similar, pure logic
   only): base/index encode-decode round-trips for all three kinds; region-scan
   picks the closest non-zero id.
2. e2e: enter edit mode; in vert mode click on a cube corner → `editMode.verts.size === 1`;
   shift-click a second corner → 2; in face mode click the cube center → 1 face
   selected; click empty space → selection cleared.
3. `npm run build`, `npm test`, `node e2e/edit.mjs` all pass.
