# P7-3 — Edge slide (GG)

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Edit mode: pressing **G twice quickly** (second G within the modal, i.e. G then
G as a modal key) switches the move into EDGE SLIDE: each selected vert moves
along its "rail" — factor t in [-1, 1] driven by horizontal pointer motion —
toward the neighbouring vert on either side. LMB/Enter confirm, RMB/Esc cancel,
numeric input sets t.

## Rails
For each affected vert v (from selectedVertIds): rail candidates are v's
adjacent edges NOT in the selection and not between two selected verts. Pick
the two rails most anti-parallel to each other (dot < 0 preferred); t > 0
slides toward rail A's far vert by t × edgeLength, t < 0 toward rail B's.
A vert with one rail slides only for the matching sign; zero rails → stays.
This is v1: no clamping beyond |t| ≤ 1, no even-mode.

## Acceptance criteria
1. Unit: on a 3x1 quad strip (build via fromData), the middle edge's verts
   slide toward the +X neighbours at t=0.5 by half the rail length (hand-check
   one coordinate); t=-0.5 goes the other way; cancel restores.
2. e2e: cube, edge mode, select one vertical edge, G then G → status contains
   'Slide', horizontal drag + confirm moves both verts along their top/bottom
   rails (assert both moved and stayed on the cube's top/bottom planes);
   Ctrl+Z restores.
3. All four e2e suites + npm run build + npm test pass.

## Result
Built edge slide (GG) for edit mode.

- NEW `src/tools/edgeSlide.ts` — pure rail math + modal operator:
  - `pickRails(mesh, vertId, selected)`: rail candidates = adjacent edges whose
    far vert is unselected (excludes selected edges / edges between two selected
    verts); picks the most anti-parallel pair (min dir dot). A/B assigned
    deterministically by far-vert id (larger id = rail A / +t direction); a
    single candidate becomes the +t rail; zero → stays.
  - `slidePosition(base, rails, t)`: t>0 → toward rail A by t×length, t<0 →
    toward rail B by |t|×length, missing rail for the sign → stays.
  - `EdgeSlideOperator`: horizontal pointer motion (or typed numeric) drives
    t∈[-1,1]; modal-GEOMETRY undo pattern (preview via setVertCo, restore + push
    MeshEditCommand.capture('Edge Slide') on confirm; restore on cancel).
- NEW `src/tools/edgeSlide.test.ts` — 9 unit tests: rail pick on a 3×1 quad
  strip, t=±0.5 hand-checked coordinates, single-rail sign matching, operator
  confirm/undo + cancel + empty-selection guard.
- `src/tools/editTransform.ts` — minimal hook (~5 lines): `EditTranslateOperator`
  gains a public `slideRequested` flag, set when 'g' is pressed mid-Move.
- `src/input/InputManager.ts` — GG swap: after the Move consumes 'g', cancel it
  (restore, push nothing) and start `EdgeSlideOperator` on the same selection.
- `e2e/edit.mjs` — appended P7-3: cube, edge mode, select a vertical edge, GG →
  status contains 'Slide', horizontal drag + confirm slides both verts (each
  stays on its top/bottom plane), Ctrl+Z restores.
