# P7-3 — Edge slide (GG)

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Edit mode: pressing **G twice quickly** (second G within the modal, i.e. G then
G as a modal key) switches the move into EDGE SLIDE: each selected vert moves
along its "rail" — factor t in [-1, 1] driven by horizontal pointer motion —
toward the neighbouring vert on either side. LMB/Enter confirm, RMB/Esc cancel,
numeric input sets t.

## Rails
For each affected vert v (from selectedVertIds): rail candidates are v's
adjacent edges NOT in the selection and not between two selected verts. Pick
the two rails most anti-parallel to each other (dot < 0 preferred); t > 0
slides toward rail A's far vert by t × edgeLength, t < 0 toward rail B's.
A vert with one rail slides only for the matching sign; zero rails → stays.
This is v1: no clamping beyond |t| ≤ 1, no even-mode.

## Acceptance criteria
1. Unit: on a 3x1 quad strip (build via fromData), the middle edge's verts
   slide toward the +X neighbours at t=0.5 by half the rail length (hand-check
   one coordinate); t=-0.5 goes the other way; cancel restores.
2. e2e: cube, edge mode, select one vertical edge, G then G → status contains
   'Slide', horizontal drag + confirm moves both verts along their top/bottom
   rails (assert both moved and stayed on the cube's top/bottom planes);
   Ctrl+Z restores.
3. All four e2e suites + npm run build + npm test pass.
