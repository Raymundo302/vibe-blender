icons: boolean; frustums: boolean; cursor3d: boolean }` with a singleton
`overlays`, `loadOverlayPrefs()` / `saveOverlayPrefs()` persisting to
localStorage key `vibe-overlays` (all default true; malformed stored JSON →
defaults — unit-test this).

## originDots.ts
Small (6px) orange dot per SELECTED, effectively-visible object at its WORLD
origin (`scene.worldTransformOf(obj).position`), only when
`overlays.originPoints` — Blender shows origins of selected objects.
`update()` every frame from main.ts; reuse one pool of divs (selection sizes
are small). Hide when behind the camera — copy the clip-w check from
cursorOverlay.ts verbatim.

## Topbar
1. **Overlays dropdown** (label `⬒ Overlays` or similar): checkbox rows Grid /
   Origin Points / Light & Camera Icons / Camera Frustums / 3D Cursor. Each
   toggles the pref + saves; 3D Cursor also sets `cursorOverlay.visible`
   (topbar receives cursorOverlay — extend the Topbar constructor opts from
   main.ts... that touches main.ts only where you already may).
2. **Pivot dropdown**: `Pivot: Median ▾` ↔ `3D Cursor` writing scene.pivotMode
   (no undo entry — viewport setting like shading mode).
3. **💡 Lights toggle button**: flips `visible` on ALL light objects (off if
   any is on, else all on). Not undoable; icon reflects state each update().
