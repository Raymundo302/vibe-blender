# UR16-1 — Shader model v2: named shaders, per-channel inputs, gradient, alpha

## Goal (Ray's redesign, 2026-07-12)
Materials get a top-level **shader** choice — simple shaders that expose only
what they need, plus the do-everything one — and every channel becomes a
SOCKET that can be driven by a value/color, an image, or a NEW object-space
gradient. All shaders gain an **alpha** channel. This task is the MODEL +
ENGINES; UR16-2 builds the UI.

## Material model
```ts
shader: 'diffuse' | 'super' | 'metal' | 'glass' | 'emit';   // default NEW materials: 'diffuse'
alpha: ChannelInput<number>;                                 // 0..1, default 1, ALL shaders
// Per-channel inputs replace the scattered texKind/texDataUrl/roughDataUrl/…:
type ChannelInput<T> =
  | { kind: 'value'; value: T }                              // scalar or [r,g,b]
  | { kind: 'image'; dataUrl: string }
  | { kind: 'gradient'; a: [r,g,b]; b: [r,g,b]; axis: 'x'|'y'|'z';
      offset: number; scale: number }                        // OBJECT-space linear
```
Channels per shader (engines consume exactly these; extras ignored):
- diffuse: color, roughness, alpha
- metal:   color, roughness, alpha            (metallic forced 1)
- glass:   color, roughness, ior(value only), alpha
- emit:    color, strength(value only), alpha (shadeless semantics)
- super:   everything current — color, roughness, metallic, emissive+strength,
           transmission, ior, subsurface, normal/bump maps, alpha, nodes/bake.

## Migration (critical — old scenes render IDENTICALLY)
- sceneJson bump (next version): parse old materials → shader 'super' with
  channel inputs synthesized from the legacy fields (texKind image →
  color:{image}, roughDataUrl → roughness:{image}, etc.). Serialize NEW format
  only. Round-trip + fixture regression (donut) must stay green.
- **Image-plane fix (Ray's confusion)**: shadeless image-plane materials
  migrate to shader 'emit' with color = {image}. Diffuse-mode image planes →
  shader 'diffuse', color = {image}. imagePlane.ts/htmlPlane.ts create the
  new shapes directly. The image now lives in the shader's COLOR socket, not
  a detached "Texture" row.

## Engines (all three must agree; the gpu-parity suite is the referee)
- CPU tracer + GPU kernel + Rendered viewport evaluate ChannelInput per
  channel: value passthrough; image = existing texture paths; **gradient** =
  object-space position → t = clamp(p[axis]·scale + offset, 0, 1) →
  lerp(a, b). GPU: gradient params pack into the material texels (no texture
  needed).
- Named shaders map onto the existing BRDF paths (metal = metallic 1 path,
  glass = transmission 1 path, emit = shadeless/emissive) — no new BSDF math.
- **Alpha**: multiplies into the UR8-3 alphaBlend/cutout machinery: alpha < 1
  → blended in raster (auto-set alphaBlend), tracer/GPU treat alpha as
  pass-through probability (stochastic transparency, matches cutout at 0/1).
- Extend e2e/gpu-parity.mjs with ONE scene using a gradient + a half-alpha
  diffuse sphere; threshold per suite convention.

## Acceptance criteria
- tsc clean; units green (migration mapping table-tested per legacy shape;
  gradient eval closed-form; alpha 0/0.5/1 semantics).
- e2e: donut + image-plane + html-plane + ur10-2/3 + gpu-tracer-2 + the new
  parity scene ALL green (both backends where suites demand). An emit image
  plane still renders pixel-equivalent to before (probe vs baseline capture).
- Free-text report, FIRST LINE STATUS:. NEVER idle-wait on long runs.
