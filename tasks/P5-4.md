# P5-4 — Fill face (F), subdivide, frame selection

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Three small modeling quality-of-life tools:
- **F**: create a face from the current selection (vert mode: 3+ selected
  verts; edge mode: a single open/closed chain).
- **Ctrl+D**: subdivide the selected faces (each quad → 4 quads, each tri →
  4 tris via edge midpoints; midpoint verts shared between adjacent selected
  faces).
- **Period (.)**: frame the selection — orbit target moves to the selection's
  centroid (selected objects in object mode, selected elements in edit mode;
  falls back to the active object, then scene origin) and distance fits its
  bounding radius (`distance = max(radius / tan(fovY/2) * 1.1, 1)`).

## Fill rules
- Vert mode: order the selected verts by angle around their centroid projected
  on their best-fit plane (Newell normal of the vert fan); reject (<3 verts or
  a selected vert already fully surrounded) with a status message.
- Edge mode: selected edges must form ONE open or closed chain → face from the
  chain's vert path. Winding: orient the new face's Newell normal to oppose
  the average normal of faces adjacent to the chain's edges (a hole-fill faces
  outward); if there are no adjacent faces, keep the path order as-is.
- MeshEditCommand.capture('Fill'); select the new face in face mode after?
  No — keep current mode, select nothing new.

## Subdivide sharing rule
Compute each edge's midpoint vert ONCE (key by edge key) so neighboring
selected faces share the new verts — no duplicate verts along shared edges;
unselected neighbors keep the old edge (T-junction, same as loop-cut policy,
document it). Quad: midpoints m01,m12,m23,m30 + center c → 4 quads keeping the
original winding. Tri: 3 midpoints → 4 tris (corner tris + center tri).

## Acceptance criteria
1. Unit: fill on a cube with one face deleted (select the 4 boundary verts) →
   6 faces again, manifold, new face normal outward; edge-mode fill on the
   same hole via its 4 boundary edges → same result; subdivide one cube face →
   9 verts... (8 + 4 midpoints + 1 center = 13 verts, 9 faces), subdivide two
   ADJACENT faces → shared edge midpoint created once (18 total verts NOT 19:
   8 + 7 midpoints + 2 centers = 17 — do the arithmetic in your test from
   actual topology, don't trust this comment blindly); Euler + manifold checks
   where applicable.
2. e2e: period key recenters the camera target onto a moved cube (assert
   camera.target ≈ cube position); delete a face, select boundary verts via
   __app, F restores 6 faces, Ctrl+Z undoes; Ctrl+D on a selected face grows
   the face count by 3 and undoes.
3. All four e2e suites + npm run build + npm test pass.
