# P7-2 — Separate selection (P)

Read `tasks/P4-CONVENTIONS.md` first.

## Goal
Edit mode, face select, ≥1 face selected: **P** moves the selected faces (and
the verts only they use) into a NEW object named `<name>.sep` (dup-name rules
apply — reuse `nextDupName` if exported, else replicate; check InputManager).
The new object copies the source's transform/color/shadeSmooth. Stays in edit
mode on the source. One undoable command restores both sides.

## Files
- NEW `src/core/mesh/ops/separate.ts` (pure split:
  `extractFaces(mesh, faceIds) → { removed: EditableMesh-shaped data }`) + test
- NEW command in `src/core/undo/separateCommand.ts` (perform-style; snapshots
  source mesh + created object)
- `src/input/InputManager.ts` — P in edit mode (face mode; else status message)
- `e2e/edit.mjs` — append checks

## Semantics
- Verts used ONLY by selected faces move; verts shared with unselected faces
  are COPIED into the new mesh (the source keeps them) — the seam duplicates,
  like Blender. Selected faces are deleted from the source. New mesh vert ids
  restart from 0 (fresh EditableMesh via addVert/addFace; deterministic order).
- Empty selection → status, no-op. Selecting ALL faces → status "can't separate
  the whole mesh", no-op (avoids leaving an empty object).

## Acceptance criteria
1. Unit: cube, separate 1 face → source 8 verts (seam verts kept)/5 faces; new
   mesh 4 verts/1 face; separate 2 adjacent faces → new mesh 6 verts/2 faces
   sharing an edge (manifold seam inside the new shell); whole-mesh rejection.
2. e2e: cube, face mode, click a face, P → 2 objects in the outliner, source
   still in edit mode with 5 faces; Ctrl+Z → 1 object, 6 faces.
3. All four e2e suites + npm run build + npm test pass.

## Result
Built. `src/core/mesh/ops/separate.ts` exposes pure `extractFaces(mesh, faceIds)
→ { removed, orphanVertIds }`: it copies every vert used by a selected face into
a fresh EditableMesh (ids restart at 0, deterministic first-appearance order,
seam verts de-duplicated so shared edges stay manifold inside the new shell) and
reports the source verts used ONLY by the separated faces so the caller can drop
them. `src/core/undo/separateCommand.ts` adds perform-style `SeparateCommand`:
snapshots the source mesh before/after (clone) + the created SceneObject and its
list index + the edit selection, builds the new object (`<name>.sep`, unique via
a local dup-name helper — nextDupName isn't exported and core mustn't import the
input layer), copies transform/color/shadeSmooth (not the modifier stack, like
Blender), deletes the separated faces + orphan verts from the source, prunes the
selection, and stays in edit mode. Returns null (no-op) outside face mode / empty
selection / whole-mesh. `P` wired in `InputManager.onEditModeKey` (face mode; else
status; empty and whole-mesh guards each get their own status message).

Acceptance: unit test (cube 1 face → source 8v/5f, new 4v/1f, no orphans; 2
adjacent faces → new 6v/2f sharing one manifold seam edge; deterministic id
restart; pyramid orphan-apex case; whole-mesh guarded in the command).
`e2e/edit.mjs` P7-2 block: reload → face mode → select one face → P → 2 objects,
still editing source with 5 faces, new object `Cube.sep` 4v/1f; Ctrl+Z → 1
object, source back to 6 faces. All green: `npm run build`, 209 unit tests
(4 new), all 4 e2e suites sequentially.
