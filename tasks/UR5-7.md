# UR5-7 — Empty objects; camera Focus Object (DoF) + Look At target

## Part A — Empty object kind
New object kind `'empty'` — a null object for rigging/targeting (Ray: DoF focus
points, look-at targets).
- `objectData.ts` / SceneObject `kind` union += 'empty'; payload
  `EmptyData { displaySize: number }` (default 1).
- **Display**: plain-axes — three world-axis-aligned line pairs through the
  origin, `displaySize` long, drawn in the objects' selection tint
  (`selectionColor`) like light aim arrows (`lightDirPass` is the pattern —
  new tiny pass or generalize it; rides the same overlays.icons toggle).
- **Picking**: reuse the billboard icon pick path lights/cameras use (grep how
  icon picking feeds `Renderer.pick`) so clicking the axes/center selects it.
- Add via Shift+A: root item **Empty** (below Camera). Spawns at the cursor like
  everything else. Duplicate (Shift+D) must handle the kind (grep the
  kind-aware duplicate from P8). Transforms/parenting/animation channels work
  as for any object (transform channelPaths are kind-agnostic — verify).
- Serialization: follow the light/camera payload precedent (optional field,
  current version tolerates absence). Outliner kind glyph: pick something
  consistent with the P8-5 glyphs.
- Join guard: join must skip empties the way it skips lights/cameras.

## Part B — Camera targets
Two new OPTIONAL CameraData fields, serialized as OBJECT INDICES not ids
(the P8 lesson — grep how activeCamera and modifier object refs serialize;
follow exactly):
1. `focusObjectId?: number` — **Focus Object** for DoF: when set, the tracer's
   `focusDistance` is overridden per render/frame by
   `|camera world position − target world origin|`. Wire it where the tracer
   builds its camera (snapshot.ts / tracer.ts cam) AND wherever Ctrl+F12 poses
   frames, so an animated target refocuses per frame. Manual focusDistance
   field stays and applies when no object is set.
2. `lookAtId?: number` — **Look At**: the camera's world ORIENTATION is replaced
   by aim-at-target (−Z toward the target's world origin, up = world +Z;
   degenerate straight-up/down: keep the previous up-projection stable — use the
   standard lookAt basis with a fallback up). Implement centrally so EVERY
   consumer agrees (through-camera view, tracer, frustum drawing, Numpad0):
   grep how many places compute a camera's world matrix / view matrix and route
   them through ONE helper (e.g. `scene.cameraWorldMatrix(obj)`); position still
   comes from transform+parenting. **Cycle guard**: if the target is a
   descendant of the camera, IGNORE the lookAt (world-matrix recursion would
   loop) and surface that in the Camera tab as a warning line.
   The object gizmo/R on such a camera still edits the underlying transform
   (harmless; lookAt wins visually) — document.
3. Camera tab UI: two object pickers (dropdown: None + scene objects by name,
   excluding the camera itself) — the modifier system's 'object' field kind
   (P9) has this select pattern; reuse its population/refresh approach. Edits
   undoable via the Camera tab's existing command path.
4. Deleting a target object: refs go stale → treat as unset (defensive lookup;
   grep how modifier object refs handle deleted targets and match).
