# P2-8 — Box select (B) + invert (Ctrl+I) in edit mode

Read `tasks/P2-CONVENTIONS.md` first.

## Goal
B starts box select: the next LMB drag draws a screen-space rectangle; on release,
elements inside are ADDED to the selection (Shift held at release: REMOVED).
Esc/RMB cancels. Ctrl+I inverts the current-mode selection.

## Files
- NEW `src/tools/boxSelect.ts` — modal Operator + a DOM rect overlay
- `src/input/InputManager.ts` — wire B and Ctrl+I in `onEditModeKey`
- `src/ui/theme.css` — one rule for the rect (dashed 1px border, transparent fill)
- `e2e/edit.mjs` — append checks

## Design
- Implement as an Operator so the existing modal routing handles keys, but note:
  unlike G, the DRAG defines the box. `start()` records nothing; first
  pointerdown anchors the rect; pointermove resizes a `position:absolute` div
  appended to the canvas's parent (`#viewport-wrap`); pointerup applies and
  self-confirms. **Check `InputManager.onPointerDown/'onPointerUp'` behavior for
  active operators before you write this** — you may need a small, guarded
  extension so pointerup can confirm (the gizmoDrag flag shows the pattern).
  If that extension turns ugly, STOP and report per the escape hatch.
- "Inside" test, per current elementMode, all in CSS pixels using the projection
  formula from the conventions doc, with the edit object's model matrix applied
  first (verts are local): vert → its projected point inside the rect;
  edge → BOTH endpoints inside; face → ALL corners inside.
- Ignore elements behind the camera (ndc.w/z handled by transformPoint already —
  skip points whose projected z is outside [-1, 1]).
- Selection changes: add to / remove from the current-mode Set only, `touch()`.
- Ctrl+I: invert the current-mode Set against the full element list
  (`elementIndexMaps(mesh)`), `touch()`.

## Out of scope
Occlusion-aware box select (Blender's "visible only" mode), lasso, circle select.

## Acceptance criteria
1. Unit test for the pure inside-rect logic (projection given a known camera).
2. e2e: edit mode, vert mode, Alt+A, B, drag a rect around the whole cube →
   8 verts selected; B + Shift-release over half the cube → fewer selected;
   Ctrl+I → complement; Esc during drag → selection unchanged.
3. `npm run build`, `npm test`, `node e2e/edit.mjs` all pass.

## Result

Built `src/tools/boxSelect.ts` exporting: `projectToScreen` (local vert → CSS px
via proj·view·model, null when ndc.z ∉ [-1,1] to skip behind-camera points), the
pure `elementsInRect(mesh, mode, mvp, w, h, rect)` (vert inside; edge = both
endpoints inside; face = all corners inside), `invertSelection(sel, mesh)`
(complement against `elementIndexMaps`, `touch()`), and the `BoxSelectOperator`
(modal; `anchor` on first LMB fixes a corner and appends a `.box-select-rect` div
to `#viewport-wrap`, pointer move resizes it, confirm adds to / Shift-removes from
the current-mode Set + `touch()`, cancel just cleans up the overlay).

Wired into `InputManager`: B and Ctrl+I in `onEditModeKey`; a guarded extension so
that while `boxSelectOp` is active the first LMB *anchors* (instead of the normal
click-confirm) with pointer capture, and pointerup confirms with the release-time
Shift state — mirrors the existing `gizmoDrag` pattern. `endOperator` clears the
flag. One CSS rule added to `theme.css` (1px dashed border, transparent fill,
`pointer-events:none`).

Verified: `npm run build` (tsc + vite) clean; `npm test` 101/101 (8 new in
`boxSelect.test.ts` covering projection with a known lookAt+perspective camera,
inside-rect membership for vert/edge/face, and invert round-trip); `node
e2e/edit.mjs` all checks pass including the 6 new P2-8 checks (B → 8 verts,
Shift-release removes, Ctrl+I complement, Esc leaves selection unchanged, overlay
removed on cancel).

Deviation: added a colocated `src/tools/boxSelect.test.ts` (not enumerated in the
Files list, but acceptance criterion 1 requires a unit test — same colocation
convention as the other tools' tests).
